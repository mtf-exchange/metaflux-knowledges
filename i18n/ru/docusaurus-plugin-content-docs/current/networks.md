## Devnet

Песочница для интеграции. Тестовые USDC — через кран (faucet); состояние эфемерное (периодически сбрасывается).

Шлюз (gateway) — единая публичная точка входа. Нативная поверхность MTF предоставляется по адресам
`/info` · `/exchange` · `/ws`; EVM JSON-RPC — по адресу `/evm`.

| Сервис | Эндпоинт |
|--------|----------|
| Точка входа шлюза | `https://api.devnet.mtf.exchange` |
| MTF-native | `POST /info` · `POST /exchange` · `GET /ws` |
| EVM JSON-RPC | `POST /evm` |
| Кран (devnet/testnet) | `POST /faucet` |
| WebSocket шлюза (native) | `wss://api.devnet.mtf.exchange/ws` |
| Обозреватель | `https://app.mtf.exchange/explorer` |
| Статус | `https://status.mtf.exchange/devnet` |

Запускаете ноду самостоятельно? Нода предоставляет те же нативные эндпоинты напрямую по адресу
`http://localhost:8080` (`/info` · `/exchange` · `/ws` · `/faucet`), а EVM RPC — по адресу
`http://localhost:8545`. Это порты для самостоятельного хостинга, а не публичные URL.

| Параметры подписи | Значение |
|-------------------|---------|
| `chainId` | `31337` |
| EIP-712 domain `name` | `"MetaFlux"` |
| EIP-712 domain `version` | `"1"` |
| EIP-712 domain `verifyingContract` | `0x0000000000000000000000000000000000000000` |

Бриджинг USDC осуществляется через **кастодиальный мост MetaBridge** ([мост](./bridge/)), а не через Circle CCTP. Депозиты в testnet используют развёртывание `MetaBridgeUSDC` в сети Base Sepolia + тестовые USDC от Circle в Base Sepolia.

### Кран (Faucet)

`POST /faucet` на точке входа шлюза зачисляет тестовые средства на указанный адрес.
Только для devnet/testnet — этот маршрут **никогда не монтируется в mainnet** (`chainId 8964`).
Выдача имеет статус **`"queued"`** — средства поставлены в очередь на следующий блок, поэтому баланс обновляется примерно через 1 блок, а не мгновенно. Полная спецификация: [`POST /faucet`](api/rest/faucet.md).

```bash
curl -X POST https://api.devnet.mtf.exchange/faucet \
  -H 'content-type: application/json' \
  -d '{"address":"0x<YOUR_ADDRESS>"}'
# -> {"address":"0x…","usdc":3000,"mtf":10,"status":"queued"}
```

- Выдаёт **3000 USDC** для кросс-обеспечения **+ 10 MTF** (спот) — **один раз на адрес**
  (повторный запрос → `429 address already funded`).
- `amount` необязателен (целые USDC); ограничивает выдачу USDC *сверху вниз* (≤ 3000). MTF фиксирован.
- Ограничение частоты запросов: 1 запрос / минуту / IP (`429` при превышении).
- `400` — некорректный адрес · `429` — адрес уже получил средства / превышен лимит по IP · `503` — очередь переполнена — тело ответа `{"error":"…"}`.

### Сброс состояния

Devnet может быть сброшен при обновлениях протокола. Периодичность: по мере необходимости в процессе разработки до запуска mainnet; при возможности — уведомление за неделю. Следите за анонсами сбросов на странице [статуса](https://status.mtf.exchange/devnet).

`chainId` testnet: `114514` (`0x1bf52`). MetaFlux — независимая сеть с собственными идентификаторами цепочек.

Отличия testnet от mainnet:
- USDC бриджируется через MetaBridge из тестовой исходной сети (тестовые USDC в Base Sepolia), а не реальные USDC.
- Набор валидаторов управляется оператором.
- Реальной экономической ценности нет.

Формат протокола testnet идентичен mainnet. Клиенты, прошедшие тестирование против testnet, должны требовать **только смены `chainId` и базового URL** для перехода на mainnet.

## Коридоры бриджинга

USDC (и другие активы) бриджируются через **кастодиальный мост MetaBridge** — совместная
подпись валидаторами с весом ⅔ от стейка, без зависимости от Circle CCTP. Исходные сети:

| Сеть | Статус |
|------|--------|
| Base | **активен в Base Sepolia** (`MetaBridgeUSDC` v3 [`0xaCF3d88013b6Bd5022cF8e8259Bd1326Ee8B73Af`](https://sepolia.basescan.org/address/0xaCF3d88013b6Bd5022cF8e8259Bd1326Ee8B73Af)); mainnet на стадии аудита |
| Solana | **активен в devnet** (`metabridge-solana` program [`Db5KYqPTFv3naxWTx83EzXQaZPMmbbAbaWHbZxK71sLB`](https://solscan.io/account/Db5KYqPTFv3naxWTx83EzXQaZPMmbbAbaWHbZxK71sLB?cluster=devnet)); mainnet на стадии аудита |
| Arbitrum | планируется |

Подробнее о процессе пополнения и вывода средств, а также таблицу развёртываний см. в разделе [мост](./bridge/).
