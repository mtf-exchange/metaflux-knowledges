# MIP-1 — Стандарт спот-токенов и развёртывание рынка

:::info
**Реализовано.** Поставляется как семейство действий `spotDeploy`; см. примечание о нумерации ниже.
:::

MIP-1 — это нативный стандарт спот-токенов MetaFlux и механизм развёртывания
спот-рынка для токена через аукцион газа в блокчейне. Он является спот-аналогом
механизма [MIP-3](./mip-3.md) для размещения **бессрочных контрактов** без разрешений —
на устоявшихся блокчейн-площадках соответствующий примитив оформлен в виде отдельного
предложения, и MetaFlux сохраняет это разделение.

## Назначение

Листинг спот-рынков, как и листинг бессрочных контрактов, является частью протокола,
а не курируемым командой решением. Любой желающий может зарегистрировать символ
токена и открыть спот-рынок, выиграв соответствующий аукцион газа и задав начальные
параметры — списков одобренных участников и комитетов по проверке не существует.

## Порядок действий

Развёртывание спот-рынка осуществляется через действие `spotDeploy`, отправляемое
через подвариант `SpotDeployKind`, который охватывает полный жизненный цикл пары:

1. **`RegisterToken`** — регистрация нового спот-токена; выделяет `AssetId`.
2. **`SetPair`** — регистрация торговой пары `(base, quote)` (например, `(BTC, USDC)`);
   выделяет `AssetId` пары.
3. **`SetFee`** — установка уровня комиссии для пары.
4. **`ActivatePair`** — активация пары (открытие для торговли).
5. **`DeactivatePair`** — деактивация пары (закрытие для новых ордеров).

Получение слота для развёртывания происходит через общий аукцион газа: строитель
(builder) вызывает **`submitGasAuctionBid`** для потока `register_token_gas_auction`
(чтобы занять символ токена) или для потока `spot_pair_deploy_gas_auction` (чтобы
развернуть пару). Каждая ставка блокирует сумму в USDC (возвращается при проигрыше
за вычетом небольшой комиссии) и содержит спецификацию рынка. Параметры аукциона
(затухание, окно возврата, интервал слотов) настраиваются через управление и являются
общими с механизмом MIP-3.

## Примечание о нумерации

В текущей реализации действия `spotDeploy` находятся в том же модуле, что и действия
`perpDeploy`, и исторически обозначались как «MIP-3». Согласно [реестру MIP](./index.md)
развёртывание спот-рынков правомерно относится к **MIP-1**, а развёртывание бессрочных
контрактов — к **MIP-3** (что отражает разделение спот/бессрочные контракты на устоявшихся
площадках). Поведение не изменилось; пересматривается только метка.
