# EVM

:::tip
**Работает в devnet.** Исполнение EVM и действия CoreWriter активны, как и статeless деривативные прекомпайлы MTF (`0x0900`–`0x0904`). Прекомпайлы для чтения состояния Core (прямые запросы позиций и книги ордеров цепочки) и кросс-чейн прекомпайлы находятся в разработке. [Мост](../bridge/) уже работает.
:::

MetaFlux EVM — это **сайдчейн** на основе [revm](https://github.com/bluealloy/revm),
который исполняет обычные Solidity-контракты и предоставляет им доступ к MetaFlux **Core** — клиринговому дому L1 для бессрочных контрактов и ончейн CLOB: слой исполнения EVM, напрямую подключённый к L1, против которого производятся расчёты.

## Чем отличается от стандартного EVM

- **Единый блок, параллельные страты** — один блок на раунд консенсуса (доли секунды); транзакции разбиваются на параллельные страты конфликтов, поэтому пропускная способность масштабируется с числом ядер, а развёртывание контрактов подтверждается уже в следующем блоке (без медленной очереди с 60-секундным ожиданием). См. [Модель исполнения](execution-model.md).
- **Встроенный доступ к Core** — контракты читают состояние Core через **системные прекомпайлы** и записывают в Core через системный контракт **CoreWriter**. См. [Взаимодействие с Core](interacting-with-core.md).
- **Детерминированность** — временны́е метки инжектируются консенсусом, числа с плавающей точкой не используются, параллельное исполнение с последовательно-эквивалентным зафиксированным состоянием.
- **Сжигание базовой комиссии EIP-1559** на адрес-сжигатель coinbase.

## Системные адреса (краткий обзор)

| Адрес | Роль |
|-------|------|
| `0x3333…3333` | **CoreWriter** — отправка действий L1 (`sendRawAction`) |
| `0x0900`–`0x0904` | прекомпайлы для чтения деривативов (маржа, NAV, ADL, отметка/расчёт, RFQ) |
| `0x0906`–`0x0908` | прекомпайлы для чтения рыночных данных (BBO, глубина L2, инвентарный риск) |
| `0x0a01`–`0x0a02` | кросс-чейн прекомпайлы (отправка / верификация) |

## JSON-RPC

Стандартный `eth_*` JSON-RPC доступен через `POST /evm` на шлюзе; сеть возвращает собственный
идентификатор через `eth_chainId` (см. [Сети и идентификаторы цепочек](../networks.md)). Развёртываемые
контракты находятся в публичном репозитории
[`metaflux-contracts`](https://github.com/mtf-exchange/metaflux-contracts).
