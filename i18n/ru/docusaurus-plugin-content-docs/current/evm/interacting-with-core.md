# Взаимодействие с Core

:::tip
**Работает в devnet.** Действия CoreWriter активны, как и детерминированные MTF-препомпилы деривативов (`0x0900`–`0x0904`). Препомпилы чтения с живым состоянием Core — прямой запрос позиций и стакана цепи — а также кросс-чейн-препомпилы готовятся к запуску. Мост ([Bridge](../bridge/)) уже работает.
:::

Контракт в MetaFlux EVM общается с **Core** (L1-клиринговая палата бессрочных контрактов + on-chain CLOB) в двух направлениях:

- **Чтение** — `staticcall` к системному **препомпилу** для получения значения, производного от Core.
- **Запись** — вызов системного контракта **CoreWriter** для отправки L1-действия.

Такое разделение на препомпилы чтения и контракт записи позволяет EVM-контракту напрямую взаимодействовать с живым состоянием L1 — получать котировки по собственным формулам цепи и затем совершать действия в клиринговой палате — не покидая VM.

## Запись в Core — CoreWriter

Отправьте L1-действие, вызвав **CoreWriter** по адресу
`0x3333333333333333333333333333333333333333`:

```solidity
interface ICoreWriter {
    /// Emitted on every successful call; the L1 scanner consumes this log.
    event RawAction(address indexed user, bytes data);

    /// selector = keccak256("sendRawAction(bytes)")[0..4] = 0x17938e13
    function sendRawAction(bytes calldata data) external;
}
```

`data` — это полезная нагрузка с префиксом версии и идентификатора:

```
data = abi.encodePacked(
    uint8(1),            // version (currently 1)
    uint24(actionId),    // action id, big-endian (1..=20)
    abi.encode(params)   // the action's ABI-encoded parameters
);
```

Действующий аккаунт — это `msg.sender` (вызывающий контракт). После короткой задержки L1 обрабатывает декодированное действие.

:::info
**Атомарность.** Вызов `sendRawAction` только сжигает газ и генерирует событие `RawAction`. Любой сбой на стороне L1 **после** этого происходит без уведомления — **откат EVM не производится**. Контракт должен самостоятельно восстанавливаться и рассматривать событие `RawAction` как единственную причинно-следственную связь между вызовом EVM и результатом на L1.
:::

### Действия

CoreWriter поддерживает 20 L1-действий (идентификатор в формате big-endian, в слоте `uint24`):

| id | Действие | Назначение |
|---:|--------|---------|
| 1 | `LimitOrder` | Выставить лимитный ордер на рынке бессрочных контрактов или спота |
| 2 | `VaultTransfer` | Внести или вывести средства из вол та |
| 3 | `TokenDelegate` | Делегировать стейк валидатору |
| 4 | `StakingDeposit` | Перевести токены на стейкинговый баланс |
| 5 | `StakingWithdraw` | Вывести токены со стейкингового баланса |
| 6 | `SpotSend` | Перевести спотовый токен на другой аккаунт |
| 7 | `UsdClassTransfer` | Переместить USDC между аккаунтами класса бессрочных контрактов и спота |
| 8 | `FinalizeEvmContract` | Привязать EVM-контракт к его Core-токену / идентификатору контракта |
| 9 | `AddApiWallet` | Авторизовать суб-ключ (агентский кошелёк) для торговли |
| 10 | `CancelByOid` | Отменить ордер по серверному идентификатору ордера |
| 11 | `CancelByCloid` | Отменить ордер по клиентскому идентификатору ордера |
| 12 | `ApproveBuilderFee` | Авторизовать билдера для взимания (ограниченной) комиссии |
| 13 | `SendAsset` | Универсальный перевод актива (бессрочные контракты / спот / волт) |
| 14 | `ReflectEvmSupplyChange` | Синхронизировать изменение эмиссии ERC-20 на стороне EVM с Core |
| 15 | `BorrowLend` | Открыть или закрыть позицию кредитования |
| 16 | `PortfolioMarginEnroll` | Включить или отключить для отправителя кросс-активную маржу портфеля |
| 17 | `RfqSubmit` | Подать котировку RFQ (id, рынок, сторона, объём, лимитная цена) |
| 18 | `FbaConfigure` | Конфигурация аукциона частых пакетов (FBA) на уровне рынка |
| 19 | `CrossChainSend` | Чейн-агностический кросс-чейн-перевод (ставится в очередь в [MetaBridge](../bridge/)) |
| 20 | `EncryptedOrderSubmit` | Ордер с пороговым шифрованием (обязательство + шифротекст) |

Типизированные структуры параметров и готовый к использованию Solidity-вызывающий код находятся в публичном репозитории
[`metaflux-contracts`](https://github.com/mtf-exchange/metaflux-contracts);
CoreWriter по адресу `0x3333…` является производственной целью (в тестах детерминированная Solidity-заглушка генерирует такую же полезную нагрузку `RawAction`).

## Чтение Core — препомпилы

Каждый препомпил вызывается через `staticcall` по фиксированному адресу с вручную сформированными **упакованными** входными данными в формате big-endian (не Solidity ABI). Размеры и цены используют плоскость с фиксированной точкой **1e8** (`px_e8`, `size_e8`); маржа USDC — **1e6**.

| Адрес | Препомпил | Возвращает |
|---------|------------|---------|
| `0x0900` | `portfolio_margin_eval` | Требуемая поддерживающая маржа по методу SPAN, индекс наихудшего сценария, штраф за концентрацию |
| `0x0901` | `vault_nav` | Общий NAV волта, общее количество долей, NAV на долю, нереализованный PnL |
| `0x0902` | `adl_pro_rata_price` | VWAP, по которому проводится ADL заданного размера, с обходом очереди по приоритету стороны |
| `0x0903` | `mark_settle` | Дельта PnL по позиции, новое накопленное финансирование, нереализованный PnL по маркировочной цене |
| `0x0904` | `rfq_book_depth` | Глубина стакана RFQ (с фильтром по стороне и ограничением глубины) |
| `0x0906` | `clob_bbo` | Лучшая цена покупки / лучшая цена продажи + объём (топ стакана) |
| `0x0907` | `clob_l2_depth` | Топ-N агрегированных уровней `(price, size)` по каждой стороне |
| `0x0908` | `inventory_risk` | Нетто/гросс-условный объём, концентрация, ограничение риска |

На сегодняшний день это **детерминированные** препомпилы котирования: вызывающая сторона передаёт входные данные (позиции, уровни очереди, котировки и т. д.), а препомпил возвращает вычисленный результат — таким образом, контракт может воспроизвести вычисление Core по собственным формулам цепи. **Чтение с живым состоянием Core** (прямой запрос позиций и стакана цепи) готовится к запуску.

## Переводы стоимости Core ↔ EVM

- **В Core** из EVM-контракта: `SpotSend` / `SendAsset` / `UsdClassTransfer`
  / `VaultTransfer` через CoreWriter (см. выше).
- **Между цепями**: `CrossChainSend` ставится в очередь в
  [кастодиальный мост MetaBridge](../bridge/), который высвобождает средства в целевой цепи при наличии подписи ⅔ валидаторов.
