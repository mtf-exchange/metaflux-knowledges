# Справочник API

Три семейства протоколов, доступных через единый шлюз
(`https://<net>-gateway.mtf.exchange`) — выбор формата взаимодействия с клиентом сводится
к выбору пути.

| Семейство | Где | Когда использовать |
|-----------|-----|-------------------|
| **MTF-native** | Путь шлюза **по умолчанию**: `POST /exchange`, `POST /info`, `GET /ws`, `POST /faucet` | Новые клиенты. Компактный формат snake_case. Открывает полный доступ, включая уникальные возможности MTF (RFQ, FBA, подключение PM, кросс-чейн). |
| **HL-compat** | Шлюз по пути `/hl/*`: `POST /hl/exchange`, `POST /hl/info`, `GET /hl/ws` | Перенос существующего HL-клиента. Формат JSON совпадает с HL точно. Нулевые изменения кода для `order`, `cancel` (поддержка дополнительных вариантов добавляется со временем). |
| **CCXT-compat** | Шлюз по пути `/ccxt/*` | Quant-фреймворки, уже работающие с CCXT. Базовое подмножество REST доступно; CCXT Pro WS — в разработке. |

> Шлюз — единая точка входа: MTF-native является путём по умолчанию
> (`/info`, `/exchange`), HL-compat вынесен в пространство имён `/hl/*`, CCXT — в `/ccxt/*`.
> Запускаете узел самостоятельно? Он предоставляет тот же нативный интерфейс
> напрямую по адресу `http://localhost:8080`.
