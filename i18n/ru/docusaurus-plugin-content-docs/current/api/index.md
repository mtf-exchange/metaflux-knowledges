# Справочник API

Три семейства протоколов, обслуживаемых единым шлюзом
(`https://<net>-gateway.mtf.exchange`) — выбор клиентского формата сводится лишь
к выбору пути.

| Семейство | Где | Когда использовать |
|-----------|-----|-------------------|
| **MTF-native** | Gateway **по умолчанию**: `POST /exchange`, `POST /info`, `GET /ws`, `POST /faucet` | Новые клиенты. Компактный формат snake_case. Открывает полный доступ, включая расширенные функции MTF (RFQ, FBA, подключение к PM, кросс-чейн). |
| **HL-compat** | Gateway по пути `/hl/*`: `POST /hl/exchange`, `POST /hl/info`, `GET /hl/ws` | Перенос существующего HL-клиента. Формат JSON совпадает с HL в точности. Нулевые изменения кода для `order`, `cancel` (дополнительные варианты добавляются со временем). |
| **CCXT-compat** | Gateway по пути `/ccxt/*` | Фреймворки квантового трейдинга, уже работающие с CCXT. Минимальный REST-набор активен; CCXT Pro WS в разработке. |

> Шлюз — единая точка входа: MTF-native — путь по умолчанию
> (`/info`, `/exchange`), HL-compat вынесен в пространство имён `/hl/*`, CCXT — в
> `/ccxt/*`. Запускаете ноду самостоятельно? Она предоставляет тот же нативный интерфейс
> напрямую по адресу `http://localhost:8080`.
