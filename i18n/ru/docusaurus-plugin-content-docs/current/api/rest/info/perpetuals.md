# `POST /info` — запросы по бессрочным контрактам

Запросы на чтение для рынков **бессрочных контрактов**. Тот же эндпоинт `POST /info`, конверт (envelope) и соглашения, что и на [базовой странице](../info.md), — здесь описаны специфичные для рынков бессрочных контрактов `type`.

:::info
**Рынки идентифицируются по `coin` (символу).** Каждый запрос на чтение в
разрезе рынка (`market_info`, `l2_book`, `recent_trades`, `trades_by_time`,
`funding_history`, `oracle_sources`, `active_asset_data`, `fba_batch_state`, …)
определяет рынок по его **символу `coin`** (`"BTC"`, `"ETH"`, …). Устаревшие
числовые аргументы запроса `asset_id` / `market_id` были **удалены** — запрос,
передающий их (и не передающий `coin`), отклоняется с
`400 {"error":"missing field coin"}`. Эти запросы по рынкам отражают символ
`coin` в своих ответах. (Только подписанный путь записи `/exchange` всё ещё
адресует рынки по числовому `asset` — это поле заморожено на уровне
консенсуса; см. [`POST /exchange`](../exchange.md).)
:::

## Типы запросов к бессрочным контрактам {#perpetual-query-types}

### Получение метаданных по отдельному рынку {#market_info}

Метаданные по отдельному рынку. Рынок определяется по символу `coin`.

```json
{ "type": "market_info", "coin": "BTC" }
```

| Аргумент | Тип | Обязателен |
|-----|------|----------|
| `coin` | symbol | да |

Отсутствие `coin` → `400 {"error":"missing field coin"}`; неизвестный символ → `404 {"error":"market not found"}`.

Ответ:

```json
{
  "type": "market_info",
  "data": {
    "coin":               "BTC",
    "kind":               "perp",
    "sz_decimals":        5,
    "mark_px":            "61550.2",
    "oracle_px":          "61501.7",
    "mid_px":             "61669.4",
    "premium":            "0.00209225",
    "tick_size":          "0.1",
    "step_size":          "0.00001",
    "min_order":          "0.00001",
    "max_leverage":       50,
    "maint_margin_ratio": "1320",
    "init_margin_ratio":  "200",
    "margin_tiers": [
      { "max_open_interest": "100000",  "max_leverage": 50, "maint_margin_ratio": "100" },
      { "max_open_interest": "500000",  "max_leverage": 20, "maint_margin_ratio": "250" },
      { "max_open_interest": "2000000", "max_leverage": 10, "maint_margin_ratio": "500" },
      { "max_open_interest": null,      "max_leverage": 5,  "maint_margin_ratio": "1000" }
    ],
    "funding": {
      "rate_per_hr":     "21",
      "cap_per_hr":      "1120",
      "interval_ms":     3600000,
      "next_payment_ts": 1783011600000
    },
    "mark_source":   "oracle_median",
    "fba_enabled":   false,
    "open_interest": "0.02346",
    "day_ntl_vlm":   "3772.890084",
    "change_24h":    "-0.00274143",
    "prev_day_px":   "61719.4",
    "disable_open":  false,
    "disable_close": false,
    "halted":        false,
    "strict_isolated": false,
    "asset_id":      0
  }
}
```

:::warning
**`asset_id` УСТАРЕЛО.** Поле сохранено временно исключительно как удобство
для indexer-шима — **не** стройте логику на его основе и **не** используйте
его в качестве аргумента запроса (он больше не принимается). Везде
адресуйте рынки по `coin`. Поле может быть удалено без повышения версии
протокола.
:::

:::info
**Плоскость отображения цен.** `mark_px`, `oracle_px`, `mid_px`, `tick_size`,
`step_size` и `min_order` представлены в **человеко-читаемой десятичной
плоскости** (`"61550.2"`, `"0.1"`, `"0.00001"`) — в тех же единицах, что и
метка позиций аккаунта. `mark_px` — это внутрибиржевая (on-book) метка с
откатом на цену оракула, если у книги пока нет метки; `oracle_px` — последняя
зафиксированная индексная цена; каждое из полей равно `"0"`, если значение не
задано. **Плоскость отправки ордеров/книги — отдельная плоскость с
фиксированной точкой 1e8** — `px` уровней `l2_book` и `limit_px` ордера
представлены в сырых величинах 1e8, а НЕ в человеко-читаемых десятичных
числах; MTF сохраняет эти две шкалы раздельными.
:::

:::info
**`margin_tiers` — встроенная лестница кредитного плеча по диапазонам
номинала.** `market_info` (и каждая строка [`markets`](#markets)) несёт
лестницу поддерживающей маржи рынка **встроенно**, как `margin_tiers`, —
восходящий список диапазонов с верхней границей:

- `max_open_interest` — **верхняя граница** диапазона (строка-число, в
  единицах размера рынка); `null` обозначает **неограниченный верхний
  уровень**.
- `max_leverage` — максимальное кредитное плечо, допустимое, пока открытый
  интерес находится в этом диапазоне (`u8`).
- `maint_margin_ratio` — коэффициент поддерживающей маржи для диапазона,
  **строка десятичных б.п.** (`"100"` = 1,00 %).

Уровень позиции — это первый диапазон, чью границу `max_open_interest` не
превышает её открытый интерес (верхний диапазон с `null` покрывает всё выше
последней конечной границы). По мере роста открытого интереса кредитное
плечо снижается, а коэффициент поддерживающей маржи растёт. Это заменяет
удалённый отдельный запрос `margin_table` — теперь лестница переносится
прямо на запись о рынке.
:::

:::info
**Точность цены vs `sz_decimals`.** `sz_decimals` — это точность **РАЗМЕРА**
(гранулярность количества в ордере — `5` ⇒ `0.00001` единиц); она **не**
определяет знаки после запятой в цене, которые задаются ценовым тиком
(`tick_size`). Это две независимые оси.
:::

`market_info` возвращает **полную** запись — объединение **динамических**
полей, которые отдаёт [`markets`](#markets) (`mark_px`, `oracle_px`, `mid_px`,
`premium`, `funding`, `open_interest`, `day_ntl_vlm`, `prev_day_px`,
`change_24h`, `halted`), и **статических** полей, которые отдаёт
[`markets_meta`](#markets_meta) (`sz_decimals`, `tick_size`, `step_size`,
`min_order`, `max_leverage`, коэффициенты маржи, `margin_tiers`,
`strict_isolated`, `disable_open` / `disable_close`, `mark_source`,
`fba_enabled`, `asset_id`). Семантику отдельных полей см. в этих двух
запросах.

### Получение актуального состояния всех рынков {#markets}

**Актуальное (динамическое)** состояние каждого зарегистрированного рынка —
поля, меняющиеся на каждом коммите (метка / цена оракула / средняя цена,
премия финансирования, открытый интерес, скользящий 24-часовой тикер,
`halted`), плюс ключи соединения `(coin, kind)`, — вместе с реестром
спот-пар и токенов. Долгоживущие **статические** метаданные (сетки
точности, лестницы плеча/маржи, источник метки, флаги контроля торговли)
отдаются отдельно запросом [`markets_meta`](#markets_meta);
[`market_info`](#market_info) возвращает обе половины для одного рынка.

```json
{ "type": "markets" }
```

Фильтрация по одному продукту через `kind` (отсутствует ⇒ обе секции):

```json
{ "type": "markets", "kind": "perp" }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `kind` | `"perp"` \| `"spot"` | нет | Фильтр по секции — отсутствует = обе; `"perp"` = только массив перпов; `"spot"` = только спот-секция |

Полезная нагрузка `data` — это **объект** с массивом `perp` (каждая строка —
**динамическая**) и объектом `spot` `{pairs, tokens}`. Строки `perp`
упорядочены детерминированно по возрастанию идентификатора рынка;
`spot.pairs` / `spot.tokens` — в порядке идентификаторов пары/токена.

Ответ (усечён до одной записи на список):

```json
{
  "type": "markets",
  "data": {
    "perp": [
      {
        "coin":            "BTC",
        "kind":            "perp",
        "mark_px":         "61521.1",
        "oracle_px":       "61529.3",
        "mid_px":          "61669.4",
        "premium":         "0.0018587",
        "funding": {
          "rate_per_hr":     "20",
          "cap_per_hr":      "1120",
          "interval_ms":     3600000,
          "next_payment_ts": 1783011600000
        },
        "open_interest":   "0.02346",
        "day_ntl_vlm":     "3772.890084",
        "prev_day_px":     "61719.4",
        "change_24h":      "-0.00300293",
        "halted":          false
      }
    ],
    "spot": {
      "pairs": [
        {
          "id": 110, "name": "BTC/USDC", "base": 101, "quote": 100,
          "active": true, "mark_px": "50000", "mid_px": "50000", "prev_day_px": null,
          "day_ntl_vlm": "0", "min_notional": "1", "taker_fee_bps": "5",
          "circulating_supply": "0"
        }
      ],
      "tokens": [
        {
          "id": 100, "name": "USDC", "sz_decimals": 2, "wei_decimals": 6,
          "is_canonical": true, "evm_contract": null,
          "system_address": "0x80abd3bd8c42d2a279e4fa00f20bb30637734371",
          "token_id": "0xf23ea17597e324c04f842e6d8bfffe75636f0af88e7c7ab93ea755d9056396bc"
        }
      ]
    }
  }

```

Каждая строка `perp` — это **динамическая** половина комплекта
[`market_info`](#market_info) — обе строятся одним и тем же построителем,
поэтому они никогда не расходятся; **статический** аналог хранится в
[`markets_meta`](#markets_meta) и соединяется по `(coin, kind)`. `mid_px`
**опускается** из строки, если книга односторонняя (никогда не передаётся
как `null`). Живой WS-канал [`markets`](../../ws/subscriptions.md#markets)
транслирует те же динамические строки (полный снимок при подписке, затем
дельты изменённых строк).

| Поле | Тип | Описание |
|-------|------|-------------|
| `perp[*].coin` | string | Символ рынка, например `"BTC"` (ключ соединения) |
| `perp[*].kind` | `"perp"` | Тип рынка (в нижнем регистре, ключ соединения) |
| `perp[*].mark_px` | Decimal string | Внутрибиржевая метка, **человеко-читаемая десятичная плоскость**, привязана к тику (откат на оракул; `"0"`, если не задана) |
| `perp[*].oracle_px` | Decimal string | Индексная цена, человеко-читаемая десятичная плоскость, привязана к тику (`"0"`, если не задана) |
| `perp[*].mid_px` | Decimal string | Средняя цена книги ордеров `(best_bid + best_ask) / 2`, человеко-читаемая, привязана к тику; **опускается**, если книга односторонняя / пустая |
| `perp[*].premium` | Decimal string \| null | Последний зафиксированный образец премии финансирования (со знаком), строка с **8 знаками** после запятой (усечена к нулю); `null`, если образца нет |
| `perp[*].funding.rate_per_hr` | bps string | Последний образец почасовой ставки финансирования (до ограничения), десятичные б.п. |
| `perp[*].funding.cap_per_hr` | bps string | Ограничение ставки финансирования в час, десятичные б.п. |
| `perp[*].funding.interval_ms` | uint64 | Периодичность финансирования по активу (1 ч = `3600000`) |
| `perp[*].funding.next_payment_ts` | uint64 | Следующая выровненная граница расчёта финансирования (epoch-мс); `0` до первого образца |
| `perp[*].open_interest` | Decimal string | Текущий открытый интерес (в единицах размера) |
| `perp[*].day_ntl_vlm` | Decimal string | 24-часовой номинальный объём |
| `perp[*].prev_day_px` | Decimal string \| null | Цена 24 часа назад; `null`, если неизвестна |
| `perp[*].change_24h` | Decimal string \| null | Изменение цены за 24 часа (доля, со знаком); `null`, если нет предыдущей цены |
| `perp[*].halted` | bool | Рынок приостановлен |
| `spot.pairs` | array | Реестр спот-пар (те же строки, что и `pairs` в [`spot_meta`](./spot.md#spot_meta), плюс актуальные `mark_px` / `mid_px` / `day_ntl_vlm`) |
| `spot.tokens` | array | Реестр спот-токенов (те же строки, что и `tokens` в [`spot_meta`](./spot.md#spot_meta)) |

**Статические** поля по каждому рынку (`sz_decimals`, `tick_size`,
`step_size`, `min_order`, `max_leverage`, `maint_margin_ratio`,
`init_margin_ratio`, `margin_tiers`, `strict_isolated`, `disable_open` /
`disable_close`, `mark_source`, `fba_enabled`, `asset_id`) в этом запросе
**отсутствуют** — получайте их из [`markets_meta`](#markets_meta).
Семантику полей спот-пар / токенов см. в [`spot_meta`](./spot.md#spot_meta).

### Получение статических метаданных по всем рынкам {#markets_meta}

**Статические** метаданные каждого зарегистрированного рынка —
долгоживущие поля, которые рынок публикует один раз и редко меняет (сетки
точности, лестницы плеча/маржи, флаги контроля торговли, источник метки),
плюс ключи соединения `(coin, kind)`, — вместе с реестром спот-пар и
токенов. Статический аналог [`markets`](#markets): вместе обе половины
покрывают все поля, которые возвращает [`market_info`](#market_info),
поэтому клиент может кэшировать статическую половину и опрашивать только
динамическую половину [`markets`](#markets). Тот же необязательный фильтр
`kind`.

```json
{ "type": "markets_meta" }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `kind` | `"perp"` \| `"spot"` | нет | Фильтр по секции — отсутствует = обе; `"perp"` = только массив перпов; `"spot"` = только спот-секция |

Полезная нагрузка `data` — это **объект** с массивом `perp` (каждая строка —
**статическая**) и тем же объектом `spot` `{pairs, tokens}`, что возвращает
[`markets`](#markets). Строки `perp` упорядочены по возрастанию
идентификатора рынка.

Ответ (перп усечён до одной записи; секция `spot` идентична
[`markets`](#markets)):

```json
{
  "type": "markets_meta",
  "data": {
    "perp": [
      {
        "coin":               "BTC",
        "kind":               "perp",
        "sz_decimals":        5,
        "tick_size":          "0.1",
        "step_size":          "0.00001",
        "min_order":          "0.00001",
        "max_leverage":       50,
        "maint_margin_ratio": "1320",
        "init_margin_ratio":  "200",
        "margin_tiers": [
          { "max_open_interest": "100000",  "max_leverage": 50, "maint_margin_ratio": "100" },
          { "max_open_interest": "500000",  "max_leverage": 20, "maint_margin_ratio": "250" },
          { "max_open_interest": "2000000", "max_leverage": 10, "maint_margin_ratio": "500" },
          { "max_open_interest": null,      "max_leverage": 5,  "maint_margin_ratio": "1000" }
        ],
        "strict_isolated": false,
        "disable_open":    false,
        "disable_close":   false,
        "mark_source":     "oracle_median",
        "fba_enabled":     false,
        "asset_id":        0
      }
    ],
    "spot": { "pairs": [ /* … same as `markets` */ ], "tokens": [ /* … */ ] }
  }
}
```

Каждая строка `perp` — это **статическая** половина комплекта
[`market_info`](#market_info), соединённая со своей динамической строкой
[`markets`](#markets) по `(coin, kind)`. Ни одно из динамических полей
(`mark_px`, `oracle_px`, `mid_px`, `premium`, `funding`, `open_interest`,
`day_ntl_vlm`, `prev_day_px`, `change_24h`, `halted`) здесь не встречается.

| Поле | Тип | Описание |
|-------|------|-------------|
| `perp[*].coin` | string | Символ рынка (ключ соединения) |
| `perp[*].kind` | `"perp"` | Тип рынка (в нижнем регистре, ключ соединения) |
| `perp[*].sz_decimals` | uint8 | Знаки после запятой для отображения размера |
| `perp[*].tick_size` | Decimal string | Минимальный шаг цены (человеко-читаемая десятичная, например `"0.1"`) |
| `perp[*].step_size` | Decimal string | Минимальный шаг размера / размер лота (человеко-читаемая десятичная) |
| `perp[*].min_order` | Decimal string | Минимальный размер ордера (человеко-читаемая десятичная) |
| `perp[*].max_leverage` | uint8 | Максимальное кредитное плечо (верхняя ступень лестницы маржи) |
| `perp[*].maint_margin_ratio` | bps string | Базовый коэффициент поддерживающей маржи, десятичные б.п. |
| `perp[*].init_margin_ratio` | bps string | Базовый коэффициент начальной маржи, десятичные б.п. |
| `perp[*].margin_tiers` | array | Лестница кредитного плеча по диапазонам номинала (см. [`market_info`](#market_info)); каждый элемент `{max_open_interest: string\|null, max_leverage: u8, maint_margin_ratio: bps-string}`, восходящие диапазоны с верхней границей, `null` = неограниченный верхний уровень |
| `perp[*].strict_isolated` | bool | Рынок принудительно использует режим строго изолированной маржи |
| `perp[*].disable_open` / `disable_close` | bool | Открытие / закрытие отключено для этого рынка |
| `perp[*].mark_source` | string | Дескриптор источника метки (например, `"oracle_median"`) |
| `perp[*].fba_enabled` | bool | Включён ли режим частых пакетных аукционов (FBA) для этого рынка |
| `perp[*].asset_id` | uint32 | **УСТАРЕВШЕЕ** поле indexer-шима — не стройте логику на его основе |
| `spot.pairs` / `spot.tokens` | array | Реестр спот-пар / токенов, идентичный [`markets`](#markets) (см. [`spot_meta`](./spot.md#spot_meta)) |

Семантику полей спот-пар / токенов см. в [`spot_meta`](./spot.md#spot_meta).

### Получение агрегированных уровней книги ордеров {#l2_book}

Агрегированные уровни заявок и предложений в разрезе рынка.

```json
{ "type": "l2_book", "coin": "BTC" }
```

| Аргумент | Тип | Обязателен |
|-----|------|----------|
| `coin` | symbol | да |

Отсутствие `coin` → `400 {"error":"missing field coin"}`.

Ответ:

```json
{
  "type": "l2_book",
  "data": {
    "coin": "BTC",
    "bids": [ { "px": "61663.1", "size": "0.04862", "n_orders": 1 } ],
    "asks": [ { "px": "61675.7", "size": "0.04862", "n_orders": 1 } ]
  }
}
```

Заявки на покупку отсортированы от лучшей (по убыванию цены), заявки на
продажу — по возрастанию. Каждый уровень агрегирует суммарный `size` и
количество выставленных ордеров `n_orders`. Неизвестный / пустой рынок
возвращает пустые массивы `bids` / `asks`.

| Поле | Тип | Описание |
|-------|------|-------------|
| `coin` | string | Отражённый символ рынка |
| `bids[*].px` / `asks[*].px` | i128 string | Цена уровня, строка с фиксированной точкой (плоскость ордеров/книги 1e8) |
| `bids[*].size` / `asks[*].size` | u128 string | Суммарный размер на уровне |
| `bids[*].n_orders` / `asks[*].n_orders` | uint64 | Количество выставленных ордеров на уровне |

### Получение недавних публичных сделок {#recent_trades}

Публичная лента сделок в разрезе рынка, обслуживаемая непосредственно из
зафиксированного состояния на ноде (ограниченное кольцо сделок по каждому
рынку, свёрнутое в AppHash — без внешнего индексатора).

```json
{ "type": "recent_trades", "coin": "BTC" }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `coin` | symbol | да | Символ рынка |
| `limit` | uint32 | нет | Ограничение числа возвращаемых **самых свежих** записей; отсутствие / `0` ⇒ полное кольцо |

Ответ:

```json
{
  "type": "recent_trades",
  "data": {
    "coin":           "BTC",
    "last_trade_ms":  1783001424768,
    "trades": [
      {
        "coin":  "BTC",
        "side":  "A",
        "px":    "61643.70000000",
        "sz":    "0.00024",
        "time":  1783001424768,
        "tid":   17691615279761551171,
        "block": 38997,
        "hash":  "0x4660d9ccf52ef1abde5e03d1b3f1c110b948d2f71331f086239666781dbde91c"
      }
    ]
  }
}
```

Записи упорядочены от старых к новым (самые новые — в конце). Кольцо
ограничено, поэтому это недавнее окно, а не вся история. Неизвестный / ни
разу не торговавшийся рынок возвращает `"trades": []` и `last_trade_ms: 0`.

| Поле | Тип | Описание |
|-------|------|-------------|
| `coin` | string | Отражённый символ рынка |
| `last_trade_ms` | uint64 | Метка времени последней сделки (`0`, если сделок нет) |
| `trades[*].coin` | string | Символ рынка, на котором исполнена сделка |
| `trades[*].side` | `"B"` / `"A"` | Токен стороны тейкера (агрессора) — `"B"` = покупка, `"A"` = продажа |
| `trades[*].px` | Decimal string | Цена исполнения, **десятичные USDC** (человеко-читаемая) |
| `trades[*].sz` | Decimal string | Исполненный размер, **базовые единицы** (целые единицы) |
| `trades[*].time` | uint64 | Метка времени сделки (мс консенсуса) |
| `trades[*].tid` | uint64 | Детерминированный идентификатор сделки (общий для обеих сторон исполнения); может превышать 2⁵³ — разбирайте как 64-битное / big integer значение, а не как число JS |
| `trades[*].block` | uint64 | Высота зафиксированного блока, в котором исполнена сделка (on-chain локатор) |
| `trades[*].hash` | hex string | Хеш транзакции исходного ордера, шестнадцатеричная строка с префиксом `0x` — позволяет отследить исполнение в блокчейне |

### Получение сделок за временное окно {#trades_by_time}

Аналогично [`recent_trades`](#recent_trades), но с фильтрацией по окну

ограниченного недавнего окна. Для глубокой истории за пределами кольца
используйте архивные типы запросов шлюза.

```json
{ "type": "trades_by_time", "coin": "BTC", "start_time": 1783000000000, "end_time": 1783011600000 }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `coin` | symbol | да | Символ рынка |
| `start_time` | uint64 | нет | Начало окна (мс, включительно); фильтрует по `time` сделки. Отсутствует ⇒ нижняя граница не ограничена |
| `end_time` | uint64 | нет | Конец окна (мс, включительно). Отсутствует ⇒ верхняя граница не ограничена |

Ответ:

```json
{
  "type": "trades_by_time",
  "data": {
    "coin":       "BTC",
    "start_time": 1783000000000,
    "end_time":   1783011600000,
    "trades": [
      {
        "coin":  "BTC",
        "side":  "A",
        "px":    "61643.70000000",
        "sz":    "0.00024",
        "time":  1783000781368,
        "tid":   4898317237641214538,
        "block": 37692,
        "hash":  "0x4660d9ccf52ef1abde5e03d1b3f1c110b948d2f71331f086239666781dbde91c"
      }
    ]
  }
}
```

`trades` использует ту же структуру записи о сделке, что и
[`recent_trades`](#recent_trades), от старых к новым. `start_time` /
`end_time` возвращаются эхом (каждое — `null`, если не задано). Рынок вне
окна / ни разу не торговавшийся возвращает `"trades": []`.

### Получение исторических OHLCV-свечей {#candle_snapshot}

Исторические OHLCV-бары для `(coin, interval)`. Единственный запрос свечей
(отдельный тип `candle` был **удалён**): в первую очередь обслуживается из
архива, если он подключён, иначе — с откатом на бары, свёрнутые из
публичного потока сделок. REST-аналог живого WS-канала
[`candles`](../../ws/subscriptions.md#candles).

```json
{ "type": "candle_snapshot", "coin": "BTC", "interval": "1m", "start_time": 1783000000000, "end_time": 1783011600000 }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `coin` | symbol | да | Символ рынка, например `"BTC"` |
| `interval` | string | да | Токен временного интервала — один из `1m`, `5m`, `15m`, `1h`, `4h`, `1d` |
| `start_time` | uint64 | нет | Начало окна (мс); фильтрует по времени открытия бара. По умолчанию `0` |
| `end_time` | uint64 | нет | Конец окна (мс); фильтрует по времени открытия бара. По умолчанию не ограничено |

Отсутствие `coin` → `400 {"error":"missing field coin"}`; отсутствие
`interval` → `400 {"error":"missing field interval"}`.

Ответ:

```json
{
  "type": "candle_snapshot",
  "data": {
    "candles": [
      {
        "t": 1783000020000,
        "T": 1783000080000,
        "i": "1m",
        "o": "6164610000000",
        "c": "6165270000000",
        "h": "6165270000000",
        "l": "6164610000000",
        "v": "576",
        "n": 24
      }
    ]
  }
}
```

Бары упорядочены от старых к новым по `t` (время открытия); последний
элемент — формирующийся бар. Пустой массив `candles` — честный пустой
ответ для рынка без истории (или без подключённого архива/источника
свёртки).

| Поле | Тип | Описание |
|-------|------|-------------|
| `t` | uint64 | Метка времени **открытия** бара (мс, выровнена по бакету) |
| `T` | uint64 | Метка времени **закрытия** бара (мс) |
| `i` | string | Токен временного интервала бакета |
| `o` / `c` / `h` / `l` | Decimal string | Цена **открытия** / **закрытия** / **максимума** / **минимума**, строка с **фиксированной точкой 1e8** (например, `"6165270000000"` = `61652.7`) |
| `v` | Decimal string | **Объём в базовом активе** — Σ торгуемого размера за бар (в единицах размера, НЕ в номинале) |
| `n` | uint64 | Количество сделок (исполнений) за бар |

### Получение истории премии финансирования {#funding_history}

Образцы премии финансирования в разрезе рынка (кольцо премии).

```json
{ "type": "funding_history", "coin": "BTC" }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `coin` | symbol | да | Символ рынка |
| `start_time` | uint64 | нет | Начало окна (мс); фильтрует по `ts_ms` образца |
| `end_time` | uint64 | нет | Конец окна (мс) |

Отсутствие `coin` → `400 {"error":"missing field coin"}`.

Ответ:

```json
{
  "type": "funding_history",
  "data": {
    "coin": "BTC",
    "samples": [
      { "ts_ms": 1783008579269, "premium": "0.00027179", "funding_rate": "0.00027179" },
      { "ts_ms": 1783008587316, "premium": "0.0005469",  "funding_rate": "0.0005469" }
    ]
  }
}
```

Образцы — это упорядоченное кольцо снимков премии из трекера
финансирования. `premium` — точное значение `Decimal` до ограничения,
представленное в виде строки (со знаком, полная точность); `funding_rate` —
та же премия, пропущенная через ограничение финансирования для конкретного
актива, — фактическая ставка, которая была бы реально начислена. Если
премия находится в пределах ограничения, `funding_rate == premium`; при
превышении `funding_rate` ограничивается знаковым пределом. Неизвестный /
пустой рынок возвращает `"samples": []`.

| Поле | Тип | Описание |
|-------|------|-------------|
| `coin` | string | Отражённый символ рынка |
| `samples[*].ts_ms` | uint64 | Метка времени образца (мс консенсуса) |
| `samples[*].premium` | decimal string | Необработанный образец премии финансирования, до ограничения (со знаком) |
| `samples[*].funding_rate` | decimal string | Реализованная ставка = `premium`, ограниченная пределом для актива (со знаком) |

### Получение прогнозируемых ставок финансирования {#predicted_fundings}

Прогнозируемая ставка финансирования по каждому рынку + время следующего
расчёта, по всем зарегистрированным рынкам бессрочных контрактов. Параметры
отсутствуют.

```json
{ "type": "predicted_fundings" }
```

Полезная нагрузка `data` — это **массив**, по одной записи на каждый
зарегистрированный рынок бессрочных контрактов, в порядке возрастания
рынков. Пустая вселенная возвращает `"data": []`.

Ответ:

```json
{
  "type": "predicted_fundings",
  "data": [
    { "coin": "BTC", "predicted_rate": "0.0020702132945825193491902456", "next_funding_time": 1783011600000 },
    { "coin": "ETH", "predicted_rate": "0.0091563951859402408793685995", "next_funding_time": 1783011600000 }
  ]
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `coin` | string | Символ рынка |
| `predicted_rate` | decimal string | **Ограниченная** ставка, которая реально была бы начислена на предстоящей границе, — премия, пропущенная через предел `±cap` для актива, со знаком (`"0"` до первого образца) |
| `next_funding_time` | uint64 | **Следующая выровненная граница расчёта** для актива (epoch-мс); `0` до первого образца |

:::info
**`predicted_rate` — это начисляемая ставка, а не сырая премия.** Она
отражает применённый предел финансирования для актива — число, которое
было бы списано/зачислено по позиции, если бы финансирование
рассчитывалось прямо сейчас. Финансирование рассчитывается **дискретно** на
границе для актива (`next_funding_time`), с периодичностью `interval_ms`
для актива (по умолчанию 1 ч). Сырой ряд премии до ограничения см. в
[`funding_history`](#funding_history); периодичность / границу см. в
[`market_info`](#market_info) `funding.interval_ms` /
`funding.next_payment_ts`.
:::
