# `POST /info` — запросы по бессрочным контрактам

Запросы на чтение для рынков **бессрочных контрактов**. Тот же эндпоинт `POST /info`, оболочка и соглашения, что на [базовой странице](../info.md) — это `type`-специфичные запросы для рынков бессрочных контрактов. (Запросы книги ордеров / сделок / свечей также обслуживают спот-пары по идентификатору `pair`.)

## Типы запросов к бессрочным контрактам

### `market_info`

Метаданные по отдельному рынку.

```json
{ "type": "market_info", "asset_id": 0 }
```

Или по имени:

```json
{ "type": "market_info", "coin": "BTC" }
```

Ответ:

```json
{
  "type": "market_info",
  "data": {
    "asset_id":        0,
    "name":            "BTC",
    "kind":            "perp",
    "sz_decimals":     5,
    "mark_px":         "67079.265",
    "oracle_px":       "67073.35",
    "mid_px":          "67079.27",
    "premium":         "0.0015",
    "tick_size":       "1000000",
    "step_size":       "1",
    "min_order":       "1",
    "max_leverage":    50,
    "maint_margin_ratio": "300",
    "init_margin_ratio":  "200",
    "funding": {
      "rate_per_hr":  "0",
      "cap_per_hr":   "400",
      "interval_ms":     3600000,
      "next_payment_ts": 0
    },
    "mark_source": "MedianOfOraclesAndMid",
    "fba_enabled": false,
    "open_interest": "0"
  }
}
```

:::info
**Плоскость отображения цен.** В этом запросе как `mark_px`, так и `oracle_px` представлены в **плоскости целых USDC** (человеко-читаемые доллары — `"67079.265"` / `"67073.35"`), в тех же единицах, что и метки позиций аккаунта. `mark_px` — это внутренняя рыночная цена, масштабированная из 1e8 с фиксированной точкой, с откатом на цену оракула, если у книги пока нет метки; `oracle_px` — последняя зафиксированная индексная цена. Каждое из полей равно `"0"`, если значение не задано. Обратите внимание: **плоскость отправки ордеров/книги остаётся в 1e8 с фиксированной точкой** — цены уровней `l2_book` и `limit_px` ордеров НЕ являются целыми USDC; MTF сохраняет эти две шкалы раздельными, и только человеко-ориентированные запросы (`market_info`, `markets`, позиции) возвращают цены в целых USDC. Семантика полей остальной части записи приведена в таблице [`markets`](#markets) ниже.
:::

:::info
**Точность цены и `sz_decimals`.** `mark_px` и `oracle_px` **привязаны к ценовому тику рынка** (`tick_size`, усечение к нулю), поэтому запрос никогда не возвращает шум ниже тика — при тике `$0.01` (`tick_size: "1000000"` в плоскости 1e8) значение `66735.255` отображается как `"66735.25"`. Обратите внимание: `sz_decimals` — это точность **РАЗМЕРА** (гранулярность количества в ордере — `5` ⇒ `0.00001` единиц), она **не** определяет знаки после запятой в цене; это делает ценовой тик. Обе величины — независимые оси (аналогично разделению в HL).
:::

### `markets`

Все зарегистрированные рынки бессрочных контрактов MIP-3 в одном вызове. Параметры не требуются.

```json
{ "type": "markets" }
```

Полезная нагрузка `data` представляет собой **массив** тех же подробных записей по отдельному рынку, которые возвращает [`market_info`](#market_info) для одного актива. Записи упорядочены детерминированно по возрастанию `asset_id` (нода обходит `BTreeMap` `mip3_market_specs`). Пустая вселенная возвращает `"data": []`.

Ответ:

```json
{
  "type": "markets",
  "data": [
    {
      "asset_id":        0,
      "name":            "BTC",
      "kind":            "perp",
      "sz_decimals":     5,
      "mark_px":         "67042.335",
      "oracle_px":       "67042.335",
      "mid_px":          "67042.33",
      "premium":         "0.0015",
      "tick_size":       "1000000",
      "step_size":       "1",
      "min_order":       "1",
      "max_leverage":    50,
      "maint_margin_ratio": "300",
      "init_margin_ratio":  "200",
      "funding": {
        "rate_per_hr":  "0",
        "cap_per_hr":   "400",
        "interval_ms":     3600000,
        "next_payment_ts": 0
      },
      "mark_source": "MedianOfOraclesAndMid",
      "fba_enabled": false,
      "open_interest": "0"
    }
  ]
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `asset_id` | uint32 | Канонический идентификатор актива (ключ сортировки) |
| `name` | string | Символ рынка, например `"BTC"` |
| `kind` | `"perp"` | Тип рынка (строчными буквами) |
| `sz_decimals` | uint8 | Знаки после запятой для отображения размера (из реестра базовых спот-токенов; `0`, если спецификация токена отсутствует) |
| `mark_px` | Decimal string | Рыночная цена по книге, **плоскость целых USDC** (метка из книги, масштабированная из 1e8, с откатом на оракул; `"0"`, если не задана) |
| `oracle_px` | Decimal string | Индексная цена, **плоскость целых USDC** (`"0"`, если не задана) |
| `mid_px` | Decimal string \| null | Реальный мид реальной книги ордеров `(лучшая_заявка + лучшее_предложение) / 2`, **плоскость целых USDC** (привязан к тику); `null`, если книга односторонняя / пустая |
| `premium` | Decimal string \| null | Последний зафиксированный образец премии финансирования (со знаком); `null`, если образец отсутствует |
| `tick_size` | i128 string | Минимальный шаг цены, **1e8 с фиксированной точкой** (плоскость отправки ордеров/книги) |
| `step_size` | u128 string | Минимальный шаг размера (лот), фиксированная точка |
| `min_order` | u128 string | Минимальный размер ордера |
| `max_leverage` | uint8 | Максимальное кредитное плечо |
| `maint_margin_ratio` | bps string | Коэффициент поддерживающей маржи, десятичные б.п. |
| `init_margin_ratio` | bps string | Коэффициент начальной маржи (`1 / max_leverage`), десятичные б.п. |
| `funding.rate_per_hr` | bps string | Последний образец премии финансирования, десятичные б.п. |
| `funding.cap_per_hr` | bps string | Ограничение ставки финансирования в час, десятичные б.п. |
| `funding.interval_ms` | uint64 | Период финансирования (1 ч = `3600000`) |
| `funding.next_payment_ts` | uint64 | Метка времени следующей выплаты финансирования (`0` до появления образца) |
| `mark_source` | string | Дескриптор рыночной цены (`"MedianOfOraclesAndMid"`) |
| `fba_enabled` | bool | Включён ли режим частых пакетных аукционов (FBA) для данного рынка |
| `open_interest` | u128 string | Текущий открытый интерес, фиксированная точка |

Каждый элемент побайтово идентичен полю `data` соответствующего ответа `market_info` для одного актива — оба строятся одним и тем же построителем записей по рынку, поэтому структуры одиночного и массового запросов никогда не расходятся. Семантику полей и примечания к прокси-значениям (`mark_source`, `next_payment_ts`) см. в [`market_info`](#market_info).

### `l2_book`

Агрегированные уровни заявок и предложений по рынку.

```json
{ "type": "l2_book", "market_id": 0 }
```

| Аргумент | Тип | Обязателен |
|-----|------|----------|
| `market_id` | uint32 | да |

Ответ:

```json
{
  "type": "l2_book",
  "data": {
    "market_id": 0,
    "bids": [ { "px": "99000", "size": "700", "n_orders": 1 } ],
    "asks": [ { "px": "101000", "size": "750", "n_orders": 2 } ]
  }
}
```

Заявки отсортированы по убыванию цены (лучшие первыми), предложения — по возрастанию. Каждый уровень агрегирует суммарный `size` и количество выставленных ордеров `n_orders`. Неизвестный / пустой рынок возвращает пустые массивы `bids` / `asks`.

| Поле | Тип | Описание |
|-------|------|-------------|
| `market_id` | uint32 | Отражённый идентификатор рынка |
| `bids[*].px` / `asks[*].px` | i128 string | Цена уровня, десятичная строка с фиксированной точкой |
| `bids[*].size` / `asks[*].size` | u128 string | Суммарный размер на уровне |
| `bids[*].n_orders` / `asks[*].n_orders` | uint64 | Количество выставленных ордеров на уровне |

### `recent_trades`

Публичная лента сделок по рынку, обслуживаемая непосредственно из зафиксированного состояния на ноде (ограниченное кольцо сделок по каждому рынку, свёрнутое в AppHash — без внешнего индексатора).

```json
{ "type": "recent_trades", "market_id": 0 }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `market_id` | uint32 | да | Идентификатор актива / рынка |
| `limit` | uint32 | нет | Ограничение числа возвращаемых **последних** записей; отсутствие / `0` ⇒ полное кольцо |

Ответ:

```json
{
  "type": "recent_trades",
  "data": {
    "market_id":      0,
    "last_trade_ms":  1700000000555,
    "trades": [
      {
        "coin":  0,
        "side":  "B",
        "px":    "67042.50",
        "sz":    "0.125",
        "time":  1700000000555,
        "tid":   90123,
        "block": 562,
        "hash":  "0x2315b79b9e82c2deb279a59448bf7841f3767d30d874e5b544d75bb9fd1e9b0c"
      }
    ]
  }
}
```

Записи упорядочены от старых к новым (новейшие в конце). Кольцо ограничено, поэтому это скользящее окно последних сделок, а не вся история. Неизвестный / ни разу не торговавшийся рынок возвращает `"trades": []` и `last_trade_ms: 0`.

| Поле | Тип | Описание |
|-------|------|-------------|
| `market_id` | uint32 | Отражённый идентификатор рынка |
| `last_trade_ms` | uint64 | Метка времени последней сделки (`0`, если сделок нет) |
| `trades[*].coin` | uint32 | Идентификатор актива / рынка, на котором исполнена сделка |
| `trades[*].side` | `"B"` / `"A"` | Сторона тейкера (агрессора) — `"B"` = покупка, `"A"` = продажа |
| `trades[*].px` | Decimal string | Цена исполнения, **десятичные USDC** (человеко-читаемая) |
| `trades[*].sz` | Decimal string | Исполненный размер, **базовые единицы** (целые единицы) |
| `trades[*].time` | uint64 | Метка времени сделки (мс консенсуса) |
| `trades[*].tid` | uint64 | Детерминированный идентификатор сделки (общий для обеих ног исполнения) |
| `trades[*].block` | uint64 | Высота зафиксированного блока, в котором исполнена сделка (on-chain локатор) |
| `trades[*].hash` | hex string | Хеш транзакции исходного ордера, шестнадцатеричная строка с префиксом `0x` — позволяет отследить исполнение в блокчейне |

### `candle`

Исторические OHLCV-бары для `(coin, interval)` за временной диапазон. REST-аналог живого WS-канала [`candles`](../../ws/subscriptions.md#candles) — WS передаёт формирующийся бар по мере поступления сделок, этот запрос возвращает закрытую историю.

```json
{ "type": "candle", "coin": "BTC", "interval": "1m" }
```

| Аргумент | Тип | Обязателен | Описание |
|-----|------|----------|-------------|
| `coin` | string | да | Символ рынка, например `"BTC"` |
| `interval` | string | да | Токен временного интервала — один из `1m`, `5m`, `15m`, `1h`, `4h`, `1d` |
| `start_time` | uint64 | нет | Начало окна (мс); фильтрует по времени открытия бара. По умолчанию `0` |
| `end_time` | uint64 | нет | Конец окна (мс); фильтрует по времени открытия бара. По умолчанию не ограничено |

Аргументы можно передавать плоско (как выше) или вложить в объект `req`; `start_time` / `end_time` также принимают camelCase-написание `startTime` / `endTime`. Отсутствие `coin` или `interval` → `400 {"error":"missing field <name>"}`.

Ответ:

```json
{
  "type": "candle",
  "data": [
    {
      "t": 1700000040000,
      "T": 1700000099999,
      "s": "BTC",
      "i": "1m",
      "o": "67000.00",
      "c": "67042.50",
      "h": "67080.00",
      "l": "66990.00",
      "v": "12.5",
      "q": "837843.75",
      "n": 37
    }
  ]
}
```

Бары упорядочены от старых к новым по `t` (время открытия); последний элемент — формирующийся бар. Пустой массив — корректный ответ при неподдерживаемом токене `interval`, рынке без проиндексированных сделок или деплое без подключённого индексатора.

| Поле | Тип | Описание |
|-------|------|-------------|
| `t` | uint64 | Метка времени **открытия** бара (мс, выровнена по интервалу) |
| `T` | uint64 | Метка времени **закрытия** бара (мс) — `t + interval − 1` |
| `s` | string | Символ монеты / рынка |
| `i` | string | Токен временного интервала |
| `o` / `c` / `h` / `l` | Decimal string | Цена **открытия** / **закрытия** / **максимума** / **минимума**, **десятичные USDC** (человеко-читаемые доллары, например `"67042.50"`) |
| `v` | Decimal string | **Объём в базовом активе** — Σ торгуемого размера за бар (в единицах монеты, НЕ в номинале) |
| `q` | Decimal string | **Объём в котировочной валюте (USD)** — `Σ цена × размер` по заявкам за бар |
| `n` | uint64 | Количество сделок (заявок) за бар |

:::info
**Ряд непрерывен.** Интервал **без сделок** всё равно формирует плоский бар, переносящий цену закрытия предыдущего бара вперёд: `o = h = l = c = предыдущее закрытие`, при `v = q = 0`, `n = 0`. Потребители получают непрерывный ряд баров без пропусков для интерполяции. **Бар не формируется до первой сделки на рынке** — ряд начинается с бакета первого исполнения, поэтому пустой массив означает, что рынок никогда не торговался (или история не подключена), а не то, что ранние бакеты были отброшены.
:::

:::info
**Этот тип обслуживается шлюзом, а не нодой.** Свечи — это производные отображаемые данные, свёрнутые из публичного потока сделок; они **не** являются зафиксированным состоянием блокчейна, не затрагивают app-hash и не несут гарантий консенсуса. Шлюз отвечает на запрос `candle` из собственного скользящего хранилища; прямой запрос к отдельной ноде вернёт `unknown info type: candle`. Честный пустой ответ (`"data": []`) возвращается, если у шлюза ещё нет истории сделок по данному рынку.
:::

### `funding_history`

Выборки фандинговой премии в разрезе рынка.

```json
{ "type": "funding_history", "market_id": 0 }
```

| Аргумент | Тип | Обязателен |
|-----|------|----------|
| `market_id` | uint32 | да |

Ответ:

```json
{
  "type": "funding_history",
  "data": {
    "market_id": 0,
    "samples": [
      { "ts_ms": 1700000000000, "premium": "0.0015", "funding_rate": "0.0015" },
      { "ts_ms": 1700000008000, "premium": "-0.0007", "funding_rate": "-0.0007" }
    ]
  }
}
```

Выборки представляют собой упорядоченное кольцо снимков премии из трекера фандинга.
`premium` — точное значение `Decimal` до применения ограничения, переданное в виде строки (со знаком, полная
точность); `funding_rate` — та же премия, пропущенная через лимит фандинга для конкретного актива
(`±funding_rate_cap` — динамическое переопределение риска, либо базовый лимит `0.04`/ч)
— т. е. фактическая ставка, которая будет применена. Если премия не превышает лимит,
`funding_rate == premium`; при превышении `funding_rate` ограничивается знаковым лимитом.
Неизвестный или пустой рынок возвращает `"samples": []`.

| Поле | Тип | Описание |
|-------|------|-------------|
| `market_id` | uint32 | Идентификатор рынка (эхо) |
| `samples[*].ts_ms` | uint64 | Временная метка выборки (консенсусное время, мс) |
| `samples[*].premium` | decimal string | Необработанная выборка фандинговой премии до ограничения (со знаком) |
| `samples[*].funding_rate` | decimal string | Реализованная ставка = `premium` с применением лимита для актива (со знаком) |

### `predicted_fundings`

Прогнозируемая ставка фандинга и время следующей выплаты по каждому рынку — для всех зарегистрированных
бессрочных рынков. Параметры отсутствуют.

```json
{ "type": "predicted_fundings" }
```

Полезная нагрузка `data` представляет собой **массив**, упорядоченный детерминированно по возрастанию
`asset` (узел перебирает `BTreeMap` спецификаций рынков). Пустая вселенная возвращает
`"data": []`.

Ответ:

```json
{
  "type": "predicted_fundings",
  "data": [
    { "asset": 0, "predicted_rate": "0.0015", "next_funding_time": 1700003600000 }
  ]
}
```

`predicted_rate` — последняя выборка премии (прокси почасовой ставки, строка decimal)
— `"0"` до первой выборки. `next_funding_time` — производная временная метка следующей
выплаты (`last_sample_ts + 1ч`), `0` до первой выборки.

| Поле | Тип | Описание |
|-------|------|-------------|
| `asset` | uint32 | Идентификатор актива / рынка |
| `predicted_rate` | decimal string | Последняя выборка премии (прокси почасовой ставки); `"0"` до первой выборки |
| `next_funding_time` | uint64 | Временная метка следующей выплаты фандинга (консенсусное время, мс); `0` до первой выборки |

### `mip3_active_bids`

Снимок газового аукциона для бессрочного деплоя без разрешений по MIP-3. Параметры отсутствуют.

```json
{ "type": "mip3_active_bids" }
```

Ответ:

```json
{
  "type": "mip3_active_bids",
  "data": {
    "auction_round":   2,
    "current_bid":     "12345",
    "current_winner":  "0x<bidder>",
    "auction_end_ms":  1700086400000,
    "started_at_ms":   1700000000000,
    "bids": [
      {
        "bidder":          "0x<bidder>",
        "amount":          "12345",
        "submitted_at_ms": 1700000000500,
        "tag":             "ETH-PERP"
      }
    ]
  }
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `auction_round` | uint64 | Текущий раунд аукциона |
| `current_bid` | decimal string | Размер лидирующей ставки |
| `current_winner` | hex address \| null | Текущий победитель аукциона; `null`, если победителя нет |
| `auction_end_ms` | uint64 | Временная метка закрытия аукциона (консенсусное время, мс) |
| `started_at_ms` | uint64 | Временная метка начала аукциона (консенсусное время, мс) |
| `bids[*].bidder` | hex address | Адрес участника аукциона |
| `bids[*].amount` | decimal string | Размер ставки |
| `bids[*].submitted_at_ms` | uint64 | Временная метка подачи ставки (консенсусное время, мс) |
| `bids[*].tag` | string | Метка ставки (например, предлагаемое название рынка) |

### `liquidatable`

Аккаунты, в настоящее время помеченные для ликвидации. Параметры отсутствуют.

```json
{ "type": "liquidatable" }
```

Ответ:

```json
{
  "type": "liquidatable",
  "data": { "accounts": [ { "address": "0x<addr>", "tier": "PartialMarket50" } ] }
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `accounts[*].address` | hex address | Аккаунт, требующий действий |
| `accounts[*].tier` | `"YellowCard" \| "PartialMarket50" \| "FullMarket" \| "BackstopTakeover"` | Уровень BOLE |

Источник состояния: `Exchange.bole_index.tier` (индекс аккаунтов BOLE, требующих действий — **не** полное сканирование аккаунтов).

> **ПОМЕТКА.** `bole_index` помечен `#[serde(skip)]` — производное, неканоническое состояние, восстанавливаемое полным сканированием при первом обращении / после загрузки снимка. В только что опубликованном снимке он пуст, пока среда выполнения не выполнит хотя бы один проход BOLE.

### `active_asset_data`

Кредитное плечо / режим маржи / максимальный размер сделки пользователя по каждому активу. Обязательно: `address` (0x hex) + `asset_id` (u32).

```json
{ "type": "active_asset_data", "address": "0x<addr>", "asset_id": 0 }
```

Ответ:

```json
{
  "type": "active_asset_data",
  "data": {
    "address": "0x<addr>", "asset_id": 0, "leverage": 7,
    "margin_mode": "isolated", "max_trade_size": "5000000000", "has_position": true
  }
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `leverage` | uint32 | Кредитное плечо открытой позиции; если позиции нет — плечо по умолчанию для аккаунта; если не задано — максимальное по рынку |
| `margin_mode` | `"cross" \| "isolated" \| "strict_iso"` | Действующий режим маржи |
| `max_trade_size` | decimal string | Максимальный потолок ордера по активу (см. `max_market_order_ntls`) |
| `has_position` | bool | Наличие у пользователя ненулевой позиции по данному активу |

Источник состояния: `locus.clearinghouses[asset].positions[addr]`, `locus.user_account_configs[addr]`, спецификация рынка / динамический риск.

### `max_market_order_ntls`

Максимальный условный объём маркет-ордера по каждому активу. Параметры отсутствуют.

```json
{ "type": "max_market_order_ntls" }
```

Ответ:

```json
{
  "type": "max_market_order_ntls",
  "data": { "ntls": [ { "asset_id": 0, "max_market_order_ntl": "5000000000" } ] }
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `ntls[*].asset_id` | uint32 | Идентификатор актива |
| `ntls[*].max_market_order_ntl` | decimal string | Потолок размера, производный от лимита открытого интереса |

Источник состояния: `PerpAnnotation.oi_cap` для каждого рынка, иначе `default_mip3_limits.max_oi_per_market`.

> **ПОМЕТКА.** В закоммиченном состоянии отдельное поле «максимальный условный объём маркет-ордера» для каждого актива отсутствует; лимит открытого интереса является ближайшим закоммиченным потолком риска и выражается в единицах **размера** (слой матчинга переводит в условный объём по текущей расчётной цене).

### `perps_at_open_interest_cap`

Активы, открытый интерес по которым достиг или превысил лимит. Параметры отсутствуют.

```json
{ "type": "perps_at_open_interest_cap" }
```

Ответ:

```json
{ "type": "perps_at_open_interest_cap", "data": { "assets": [0] } }
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `assets` | uint32[] | Идентификаторы активов, достигших или превысивших `oi_cap`, в порядке возрастания |

Источник состояния: `open_interest` каждого стакана vs `PerpAnnotation.oi_cap` (стаканы без положительного лимита пропускаются).

### `margin_table`

Таблица уровней маржи (кредитное плечо → коэффициенты поддерживающей / начальной маржи). Параметры отсутствуют.

```json
{ "type": "margin_table" }
```

Ответ:

```json
{
  "type": "margin_table",
  "data": { "tiers": [ { "asset_id": 0, "max_leverage": 50, "maint_margin_ratio": "300", "init_margin_ratio": "200" } ] }
}
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `tiers[*].asset_id` | uint32 | Идентификатор актива |
| `tiers[*].max_leverage` | uint8 | Действующее максимальное кредитное плечо (переопределение или статическое) |
| `tiers[*].maint_margin_ratio` | bps string | Коэффициент поддерживающей маржи (переопределение или статический нижний порог 3%) |
| `tiers[*].init_margin_ratio` | bps string | `1 / max_leverage` |

Источник состояния: `dynamic_risk_overrides[asset]`, иначе статический базис.

> **ПОМЕТКА.** В закоммиченном состоянии хранится один действующий уровень риска на рынок (переопределение или статический), а не многострочная лестница кредитного плеча, как в HL. Прокси — один уровень на рынок: именно тот, который движок применяет сегодня.

### `perp_dexs`

Список бессрочных DEX. Параметры отсутствуют.

Ответ:

```json
{ "type": "perp_dexs", "data": { "dexs": [ { "index": 0, "n_assets": 1, "assets": [0] } ] } }
```

| Поле | Тип | Описание |
|-------|------|-------------|
| `dexs[*].index` | uint64 | Индекс DEX в `Exchange.perp_dexs` |

| `dexs[*].assets` | uint32[] | Идентификаторы активов в DEX |

Источник состояния: `Exchange.perp_dexs`.
