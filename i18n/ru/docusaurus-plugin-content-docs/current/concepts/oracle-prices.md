# Цены оракула

:::tip
**Стабильно.**
:::

## Кратко

**Цена оракула** — это эталонная цена базового актива в протоколе, формируемая один раз за блок как **взвешенная медиана внешних спот-площадок**. Это внешний якорь, на котором строятся как [маркировочная цена](./mark-prices.md) (её компонент C1), так и [финансирование](./funding-rates.md) (расчётный ориентир). Оракул намеренно *основан на споте и реагирует медленно* — это не цена книги заявок MetaFlux и не цена последней сделки.

В протокол поступают два различных набора данных, которые легко перепутать:

| Набор данных | Площадки | Агрегация | Используется для |
|---|---|---|---|
| **Спот-оракул** | до **10 спот-площадок** | взвешенная медиана | `oracle_px`; якорь C1 маркировочной цены; расчётный номинал финансирования |
| **Внешние миды перпов** | **5 перп-площадок** (Binance, OKX, Bybit, Gate, MEXC) | медиана (≥ 2 источников) | только компонент **C3** маркировочной цены |

## Почему оракул (а не книга заявок)?

Маржа, ликвидация и финансирование — все они требуют цены, которую злоумышленник **не сможет** сдвинуть одной сделкой в тонкой книге MetaFlux. Взвешенная рыночная медиана по глубоким внешним спот-площадкам именно это и обеспечивает: чтобы её сдвинуть, нужно одновременно двигать спот на многих площадках, что дорого и самоарбитражируется. Внутренняя книга *действительно* влияет на маркировочную цену (через компоненты C2 и базис C1) — но всегда в смеси с этим внешним якорем.

## Формирование

`oracle_px` — это **взвешенная медиана** актуальных спот-площадок по данному активу.

### Таблица весов спот-площадок по умолчанию (сумма = 15)

| Площадка | Вес | | Площадка | Вес |
|---|---:|-|---|---:|
| Binance | 3 | | Kraken | 1 |
| OKX | 2 | | KuCoin | 1 |
| Bybit | 2 | | Gate | 1 |
| Coinbase | 2 | | MEXC | 1 |
| Bitget | 1 | | MetaFlux spot | 1 |

Используется **взвешенная медиана** (а не взвешенное среднее), чтобы единственная площадка с аномальным тиком не могла сместить результат — она лишь смещает то, какой отсчёт оказывается в точке взвешенной середины.

### Веса по символам через управление

Таблица по умолчанию — это запасной вариант. Действие `SetOracleWeights { asset_id, weights }` (`ActionId 148`), доступное только управлению, **заменяет** (не дополняет) таблицу для одного актива — это необходимо, поскольку длиннохвостые и безразрешительные ([MIP-3](../mip/mip-3.md)) рынки нередко не листингованы на Binance / Coinbase, и веса по умолчанию ни к чему пригодному не приведут. Деплойеры рынков **не могут** самостоятельно задавать веса (выбор источников оракула равнозначен выбору маркировочной цены); новые рынки стартуют с таблицей по умолчанию, и только управление может её переопределить.

Зафиксированный набор источников для каждого рынка доступен для запроса — см. [`oracle_sources`](#querying) — как подмножество-маска над списком площадок.

## Правила надёжности

Агрегатор спроектирован так, чтобы деградировать, а не врать. На каждом тике, по порядку:

- **Устаревание по площадке.** Площадка, не выдавшая свежий тик в течение `feed_staleness_ms` (по умолчанию **60 с**), считается отсутствующей для данного тика.
- **Отброс выбросов между площадками.** Площадка, отклоняющаяся от кросс-рыночной медианы более чем на `feed_deviation_pct` (по умолчанию **5 %**), исключается до взятия взвешенной медианы — защита от одного застывшего/нулевого/ошибочного тика.
- **Перенормировка по выжившим.** Отсутствующим площадкам присваивается вес 0, а оставшиеся веса перенормируются.
- **Порог минимального покрытия.** Если в тике присутствует **менее 50 %** суммарного настроенного веса, слот оракула **не обновляется** — сохраняется предыдущее достоверное значение. Это жёсткий нижний порог, не позволяющий одной-двум оставшимся площадкам определять цену при массовом сбое источников.

Площадка с весом 0 (например, делистингованная по данному символу) просто никогда не запрашивается.

## Публикация

Сформированное значение `oracle_px` публикуется **один раз за блок**, на основе временной метки блока консенсуса (никогда — системных часов), и подписывается валидаторами оракула активного набора. Поскольку медиана, фильтры устаревания/выбросов и временная метка — всё это производные консенсуса, каждый добросовестный валидатор вычисляет **побайтово идентичный** снимок оракула для блока.

## Связь с маркировочной ценой и финансированием

- **Маркировочная цена.** Оракул — это **якорь C1** маркировочной цены: `C1 = oracle + EMA(book_mid − oracle)`. При отсутствии внутренней книги и внешних перпов маркировочная цена деградирует до значения оракула. См. [маркировочные цены](./mark-prices.md).
- **Финансирование.** Финансирование — это разрыв между **ценой воздействия** (взвешенная по глубине цена книги) и **оракулом**, и оно **рассчитывается против оракула**. Важно: если оракул для рынка устарел или ненадёжен, финансирование по этому рынку *блокируется* и стремится к 0, вместо того чтобы рассчитываться против цены, которой никто не доверяет. См. [ставки финансирования](./funding-rates.md#gating-when-the-oracle-is-untrusted).

## Запросы

Сформированное значение `oracle_px` отображается в **плоскости целых USDC** (например, `"67042.335"`) запросом [`market_info`](../api/rest/info/perpetuals.md#market_info) вместе с `mark_px`:

```bash
curl -X POST https://api.devnet.mtf.exchange/info \
  -H 'content-type: application/json' \
  -d '{"type":"market_info","asset_id":0}'
```

```json
{
  "type": "market_info",
  "data": {
    "asset_id":  0,
    "mark_px":   "67042.335",
    "oracle_px": "67042.335"
  }
}
```

Зафиксированный набор источников для каждого рынка доступен через `oracle_sources` (подмножество активных площадок для рынка):

```bash
curl -X POST https://api.devnet.mtf.exchange/info \
  -H 'content-type: application/json' \
  -d '{"type":"oracle_sources","asset_id":0}'
```

Необработанные данные по каждой площадке и точные веса, использованные в тике, хранятся в зафиксированном состоянии; на текущий момент они не выделены как отдельные поля протокола помимо подмножества источников.

## Граничные случаи

<details>
<summary>Показать граничные случаи</summary>

- **Одна площадка застряла на устаревшей цене.** Удаляется фильтром устаревания (> 60 с) или фильтром выбросов (> 5 % от кросс-рыночной медианы) — в зависимости от того, что сработает первым; медиана берётся по выжившим.
- **Массовый сбой рынка (< 50 % веса присутствует).** Слот оракула удерживает последнее достоверное значение. C1 маркировочной цены продолжает использовать это значение, поэтому маржа/ликвидация остаются заякоренными вместо заморозки или привязки к тонкому тику.
- **Длиннохвостый рынок отсутствует на мейджорах.** Стартует с таблицей по умолчанию (которая по большей части ни к чему не приводит) до тех пор, пока управление не установит переопределение `SetOracleWeights` по символу, указывающее на площадки, которые его реально листингуют.
- **Спот-оракул здоров, но перпы расходятся.** Это нормально — перп может торговаться с устойчивой премией/дисконтом к споту. Оракул (спот) остаётся на месте; маркировочная цена движется вместе с перпом через C2/C3 и базисную EMA C1. См. [mark vs oracle](./mark-prices.md#mark-vs-oracle--why-they-diverge).

</details>

## FAQ

<details>
<summary>Показать FAQ</summary>

**В: Цена оракула — это то же самое, что маркировочная цена?**
О: Нет. Оракул — это чисто внешний спот-ориентир. Маркировочная цена — это устойчивая к манипуляциям *композиция*, смешивающая оракул с книгой MetaFlux и внешними мидами перпов. Они совпадают, когда перп следует за спотом, и расходятся, когда перп несёт базис. См. [маркировочные цены](./mark-prices.md).

**В: Могут ли операторы оракула сдвинуть мою цену ликвидации?**
О: Чисто-оракульная маркировочная цена позволила бы им это. Именно поэтому маркировочная цена — медиана из трёх: оракул — лишь один из трёх компонентов, так что манипулируемый источник остаётся в меньшинстве, если только книга и внешние перпы не движутся вместе с ним.

**В: Какие площадки определяют цену конкретного рынка?**
О: Таблица из 10 площадок по умолчанию, если только управление не установило переопределение по символу. Запросите `oracle_sources`, чтобы узнать зафиксированное подмножество для конкретного рынка.

</details>
