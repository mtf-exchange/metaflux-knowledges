# Глоссарий

:::tip
**Стабильный.** Новые термины добавляются с каждым расширением протокола.
:::

Определения терминов, используемых во всей документации. Там, где у темы есть отдельная страница, приведены перекрёстные ссылки.

## A

**ADL — Авто-делевередж (Auto-deleverage).** Механизм взаимного распределения убытков, который изымает нереализованный PnL у прибыльных контрагентов, когда страховой пул не может покрыть дефицит ликвидации T3. См. [ADL](./adl.md).

**Агентский кошелёк (Agent wallet).** Ключ подписи, авторизованный мастер-аккаунтом для действий от его имени, **без** права на вывод средств. См. [агентские кошельки](./agent-wallets.md).

**ALO — Только добавление лимита (Add-Limit-Only).** Параметр TIF, при котором ордер полностью отклоняется, если любая его часть пересекла бы стакан. Гарантированно исполняется как мейкер. См. [типы ордеров](./order-types.md#time-in-force).

**Asset ID — Идентификатор актива.** Канонический целочисленный идентификатор рынка. Различается в зависимости от сети; можно получить через запрос `meta`.

**Action — Действие.** Вызов, изменяющий состояние, к `POST /exchange`. Тегированное объединение вариантов с примерно 30 типами. См. [exchange.md](../api/rest/exchange.md#action-catalog).

## B

**Бэкстоп (T3) (Backstop).** Уровень ликвидации, при котором протокол изымает позицию аккаунта, вышедшего за пороговое значение, в страховой пул. См. [многоуровневая ликвидация](./tiered-liquidation.md#t3-backstop--netting-at-mark).

**Диапазон марк-цены (Band, mark-price).** Ограничение на допустимое изменение марк-цены за один блок. Защита от манипуляций оракулом и серединой стакана. См. [марк-цены](./mark-prices.md#sanity-bands).

**Batch ID — Идентификатор партии.** Идентификатор аукционной партии для рынков FBA. См. [FBA](./fba.md).

**bps — Базисный пункт (Basis point).** 0.01% (= `1e-4`). Ставки комиссий номинированы в bps; `5 bps` = 0.05%.

**Строительный рибейт (Builder rebate).** Доля комиссии, выплачиваемая адресу, который инициировал ордер (фронтенд, агрегатор, сервис автоматизации). См. [комиссии](./fees.md#builder-rebate).

## C

**CCTP — Протокол межсетевой передачи (Cross-Chain Transfer Protocol).** Протокол межсетевой передачи Circle. MetaFlux **не** использует CCTP; вместо этого USDC переводится через [MetaBridge](../bridge/) (мост под управлением валидаторов с кастодиальным хранением).

**chainId.** Поле домена EIP-712, определяющее сеть. `31337` — devnet, `114514` — testnet, `8964` — mainnet. См. [сети](../networks.md).

**Cloid — Клиентский идентификатор ордера (Client Order ID).** 16-байтный идентификатор, задаваемый клиентом; обеспечивает `CancelByCloid` и идемпотентность ордеров. См. [exchange.md `submit_order`](../api/rest/exchange.md#submit_order).

**Цена клиринга (Clearing price, FBA).** Единая унифицированная цена, по которой рассчитывается партия FBA. См. [FBA](./fba.md).

**Кросс-маржа (Cross margin).** Режим маржи, при котором все позиции используют общее обеспечение аккаунта. Эффективное использование капитала; не является изолированным. См. [режимы маржи](./margin-modes.md).

## D

**Делегирование (стейкинг) (Delegation).** Ставка MTF делегатора, назначенная в пул валидатора. Приносит вознаграждения, подвергается слэшингу. См. [стейкинг](./staking.md).

**Разделитель домена (Domain separator).** 32-байтовая константа EIP-712 для каждой сети; один из входных параметров подписанного хеша. См. [подписание](../integration/signing.md).

## E

**EIP-712.** Стандарт Ethereum для типизированных структурированных подписанных данных. Подписание в MetaFlux использует конверт EIP-712 (`0x1901 || domain || hash`). См. [подписание](../integration/signing.md).

**EMA — Экспоненциальное скользящее среднее (Exponential Moving Average).** Используется при сглаживании средней цены для вычисления марк-цены. См. [марк-цены](./mark-prices.md).

## F

**FBA — Частый пакетный аукцион (Frequent Batch Auction).** Дискретная по времени альтернатива непрерывному CLOB для матчинга ордеров. См. [FBA](./fba.md).

**FIFO — Первый вошёл — первый вышел (First-In-First-Out).** Приоритет исполнения ордеров на одном ценовом уровне в непрерывном CLOB.

**FOK — Исполнить или отменить (Fill-or-Kill).** Параметр TIF, при котором ордер исполняется целиком или полностью отменяется. См. [типы ордеров](./order-types.md#time-in-force).

**Ставка финансирования (Funding rate).** Ежечасный платёж между пользователями, привязывающий цену бессрочного контракта к базовому оракулу. См. [ставки финансирования](./funding-rates.md).

## G

**Группировка (Grouping).** Параметр `Order`, связывающий ноги в семейство OCO (`NormalTpsl`) или скобки, привязанные к позиции (`PositionTpsl`). См. [типы ордеров](./order-types.md#grouping).

**GTC — Действует до отмены (Good-Till-Cancelled).** Параметр TIF по умолчанию; ордер остаётся в стакане до тех пор, пока не будет отменён. См. [типы ордеров](./order-types.md#time-in-force).

## H

**Коэффициент здоровья (Health ratio).** `account_value / maint_margin`. Определяет положение на лестнице [многоуровневой ликвидации](./tiered-liquidation.md).

**Максимальная отметка (High-water mark).** Исторически наивысшая цена доли в хранилище, используемая для расчёта комиссии за результат. См. [хранилища](./vaults.md).

**HL-compat.** Поверхность протокола шлюза, зеркально повторяющая форму передачи данных HL (URL, JSON, подписи). См. [hl-compat](../api/rest/hl-compat.md).

## I

**IOC — Немедленно или отменить (Immediate-Or-Cancel).** Параметр TIF; исполняет доступный объём, остаток отменяет. См. [типы ордеров](./order-types.md#time-in-force).

**Идемпотентность (Idempotency).** Свойство, при котором повторная отправка запроса даёт тот же наблюдаемый результат. См. [идемпотентность](../integration/idempotency.md).

**Страховой пул (Insurance pool).** Часть MFlux Vault, зарезервированная для покрытия бэкстопа T3. См. [хранилища](./vaults.md#insurance-pool).

**Изолированная маржа (Isolated margin).** Режим маржи, при котором отдельный лимит по каждому активу ограничивает убыток по данному активу. См. [режимы маржи](./margin-modes.md).

## L

**Стакан L2 (L2 book).** Книга ордеров на заданную глубину (топ-N уровней с каждой стороны). См. [`l2_book` info](../api/rest/info.md#l2_book).

**Уровень ликвидации (Liquidation tier).** Ступень на [многоуровневой лестнице](./tiered-liquidation.md): T0 — жёлтая карточка, T1 — частичная, T2 — полная, T3 — бэкстоп, T4 — ADL.

**Период блокировки (Lock-up, стейкинг / хранилище).** Время, необходимое между сигналом на вывод из стейкинга или хранилища и фактической доступностью средств. См. [стейкинг](./staking.md), [хранилища](./vaults.md).

## M

**Поддерживающая маржа (Maintenance margin).** Минимальное обеспечение, необходимое для удержания открытой позиции. Здоровье = `account_value / maint_margin`. См. [режимы маржи](./margin-modes.md).

**Мейкер / Тейкер (Maker / Taker).** Мейкер обеспечивает ликвидность (пассивный ордер в стакане); тейкер забирает её (ордер, пересекающий стакан). Разные ставки комиссий. См. [комиссии](./fees.md).

**Марк-цена (Mark price).** Авторитетная цена протокола для расчёта маржи и ликвидации. Медианная композиция из средней цены, оракула и EMA. См. [марк-цены](./mark-prices.md).

**Мастер-аккаунт (Master account).** Аккаунт, состояние которого изменяется в результате действий; может подписываться самостоятельно или авторизованным агентом. См. [агентские кошельки](./agent-wallets.md).

**MFlux Vault.** Страховой пул и пул маркет-мейкинга, управляемый протоколом. См. [хранилища](./vaults.md#mflux-vault).

**MIP — Предложение по улучшению рынка (Market Improvement Proposal).** Нумерованное предложение по улучшению протокола (аналог схем предложений по улучшению, принятых в известных on-chain протоколах бессрочных контрактов). См. [MIP](../mip/).

**msgpack.** Формат бинарной сериализации. Подписанная полезная нагрузка действия представляет собой байты msgpack. См. [подписание](../integration/signing.md).

**MTF.** Токен протокола MetaFlux. Используется для стейкинга, управления и сжигания комиссий.

**Мультиподпись (Multi-sig).** Требование M-из-N подписей для аккаунта. См. [мультиподпись](./multi-sig.md).

## N

**Nonce.** Строго монотонный uint64 на отправителя, включаемый в каждое действие; защита от повторного использования. См. [идемпотентность](../integration/idempotency.md).

## O

**Oid — Идентификатор ордера (Order ID).** uint64, присваиваемый сервером; возвращается в ответе `Order` и в событиях `userEvents`/`orderEvents`. См. [exchange.md](../api/rest/exchange.md).

**Оракул (Oracle).** Внешний ценовой поток, формируемый из цен CEX через средневзвешенное по времени значение (TWA). Входные данные для марк-цены и ставки финансирования. См. [марк-цены](./mark-prices.md#the-oracle-c1-anchor).

## P

**PnL.** Прибыль и убытки. Нереализованные (по рыночной стоимости открытой позиции) в сравнении с реализованными (закрытыми по цене выхода из сделки).

**Портфельная маржа (Portfolio margin, PM).** Кросс-активная маржинальная модель на основе сценариев; капиталоэффективна для хеджированных книг. См. [портфельная маржа](./portfolio-margin.md).

**Индекс премии (Premium index).** EMA значения `mid - oracle`; входные данные для ставки финансирования. См. [ставки финансирования](./funding-rates.md).

## R

**Только на сокращение (Reduce-only).** Флаг ордера, отклоняющий ордер при поступлении, если он увеличивает размер позиции. См. [типы ордеров](./order-types.md#reduce-only).

**RFQ — Запрос котировки (Request for Quote).** Рабочий процесс с котировками мейкера для крупных объёмов, которые нежелательно выставлять в публичный стакан. См. [RFQ](./rfq.md).

## S

**Отправитель (Sender).** Адрес, состояние которого изменяется при запросе `POST /exchange`. Может подписываться самостоятельно или авторизованным агентом.

**Доля (Share, хранилище).** Единица участия в хранилище; выпускается при пополнении по текущей `share_price`, погашается при выводе по текущей `share_price`. См. [хранилища](./vaults.md).

**Слэшинг (Slashing).** Наказание валидатора за двойную подпись или простой; уменьшает ставку валидатора (и делегаторов). См. [стейкинг](./staking.md#slashing).

**STP — Предотвращение самоторговли (Self-Trade Prevention).** Параметр ордера, определяющий действие при совпадении нового ордера с собственным пассивным ордером. См. [типы ордеров](./order-types.md#self-trade-prevention).

**Strict-Iso.** Режим маржи, аналогичный изолированному, с дополнительным свойством: позиция исключается из любого неттинга по портфельной марже. См. [режимы маржи](./margin-modes.md).

**Суб-аккаунт (Sub-account).** Производный аккаунт под мастер-аккаунтом; изолированные позиции и ордера, пополнение/вывод средств только через мастер-аккаунт. См. [суб-аккаунты](./sub-accounts.md).

## T

**Тейкер (Taker).** Сторона, забирающая ликвидность; сторона сделки, пересекающей стакан.

**Размер тика (Tick size).** Минимальный ценовой шаг для рынка. Ордера должны ему соответствовать.

**TIF — Время действия ордера (Time-In-Force).** Параметр ордера: GTC / IOC / ALO / FOK. См. [типы ордеров](./order-types.md#time-in-force).

**TPSL — Тейк-профит / Стоп-лосс (Take-Profit / Stop-Loss).** Группировка триггерных ордеров в защитные скобки. См. [типы ордеров](./order-types.md#triggers).

**TVL — Общая заблокированная стоимость (Total Value Locked).** Сумма NAV хранилища по всем депозиторам.

**TWAP — Средневзвешенная по времени цена (Time-Weighted Average Price).** Примитив ордера, разбивающий крупный ордер на части во времени. См. [типы ордеров](./order-types.md#twap).

## U

**Universe — Вселенная.** Активный список рынков (бессрочные контракты + спот) на протоколе. Возвращается запросом `meta`.

**Нереализованный PnL (Unrealised PnL).** Прибыль/убыток по рыночной стоимости открытых позиций. Ещё не реализован закрытием.

**USDC.** Валюта котировки для рынков MetaFlux; переводится через [MetaBridge](../bridge/).

## V

**Валидатор (Validator).** Участник консенсуса; предлагает блоки и голосует. Получает комиссию с вознаграждений делегаторов; подвергается слэшингу.

**Хранилище (Vault).** Пул USDC под управлением подписанта-менеджера, с семантикой выпуска/погашения долей. См. [хранилища](./vaults.md).

## W

**Доступно для вывода (Withdrawable).** Свободный баланс, который может покинуть аккаунт (не удерживается в качестве маржи под открытые позиции, не в изолированном лимите, не заблокирован в хранилище).

## Y

**Жёлтая карточка (T0) (Yellow card).** Первый уровень ликвидации. Ордера ALO отменяются; позиции не затрагиваются; клиент уведомлён. См. [многоуровневая ликвидация](./tiered-liquidation.md#why-a-yellow-card).
