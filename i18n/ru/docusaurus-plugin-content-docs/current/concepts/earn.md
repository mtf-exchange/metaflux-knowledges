# Earn

:::info
**Доступно на devnet (предварительная версия).** Пул кредитования USDC, генерирующий доходность за счёт заёмщиков [спот-маржи](../products/spot-margin.md). Пополнение пула, ценообразование долей, погашение в пределах свободной ликвидности, а также автоматический ликвидатор спот-маржи, защищающий пул, — всё это работает сквозным образом на **devnet уже сегодня** (см. [раздел действий](#deposit--withdraw) ниже). Воспринимайте это как **предварительную версию**: коэффициенты поддерживающей маржи по парам всё ещё проходят калибровку.
:::

## Кратко

Внесите USDC в **пул Earn** и зарабатывайте доходность. Пул выдаёт USDC в кредит заёмщикам [спот-маржи](../products/spot-margin.md), которые платят проценты; эти проценты накапливаются в пуле и повышают стоимость ваших **долей**. **Шаг получения вознаграждения отсутствует** — доходность непрерывно реинвестируется в стоимость доли, и вы фиксируете её при выводе средств.

## Принцип работы — модель долей / NAV

При внесении средств вы получаете **доли**, оценённые по текущей чистой стоимости активов пула на одну долю (NAV). Проценты, уплаченные заёмщиками, увеличивают общую стоимость пула, поэтому каждая доля постепенно становится дороже в USDC.

```
share_price        = pool_value / total_shares           # NAV per share
deposit D USDC  →  mint  D / share_price  shares
withdraw S shares → receive  S × share_price  USDC
```

- `pool_value` изначально равна общей сумме депозитов и **растёт с каждым блоком** по мере накопления процентов по займам.
- `total_shares` изменяется только при пополнениях (выпуск) и выводах (погашение).
- Первый депозит устанавливает `share_price = 1.0` (1 доля = 1 USDC).

Поскольку проценты увеличивают `pool_value` (а не количество долей), `share_price` монотонно растёт пока займы обслуживаются — доли каждого участника дорожают с одинаковой скоростью, без гонки за вознаграждением и без индивидуального учёта по пользователям.

## Расчёт доходности

Ваш заработок — это прирост стоимости ваших долей между депозитом и выводом:

```
your_yield = your_shares × (share_price_now − share_price_at_deposit)
```

За каждый блок пул увеличивается на сумму процентов, начисленных по действующим займам:

```
interest_this_block = total_borrowed × borrow_rate_per_ms × Δms
pool_value         += interest_this_block
share_price         = pool_value / total_shares          # recomputed
```

### Эффективная APY

Не весь внесённый USDC выдаётся в кредит сразу — ставку заимствования зарабатывает лишь **задействованная** часть. Поэтому доходность вкладчика равна ставке заимствования, умноженной на коэффициент утилизации:

```
utilisation     = total_borrowed / pool_value            # 0 … 1
depositor_APY  ≈ borrow_APR × utilisation × (1 − protocol_fee)
```

| | Значение |
|---|---|
| `borrow_APR` | фиксированная ставка заимствования спот-маржи (по паре) |
| `utilisation` | доля пула, выданная в кредит в данный момент |
| `protocol_fee` | опциональная доля протокола с процентов, если настроена |

Пример: ставка заимствования 12% при утилизации 50%, без комиссии протокола → APY вкладчика ≈ 6%. Все вычисления ведутся с фиксированной точностью (`Decimal`), без плавающей запятой.

## Пополнение / вывод

Оба действия авторизуются отправителем через публичный путь
[`/exchange`](../api/rest/exchange.md#spot-margin--earn); `asset` — это
**идентификатор котируемого актива, допускающего кредитование** (ключ пула — котировка зарегистрированной спотовой пары),
а `amount` / `shares` — десятичные числа, передаваемые как строки JSON. Пул
**создаётся автоматически при первом депозите** для любого допустимого актива. Проверьте выпущенные /
оставшиеся доли и итоги пула через
[`/info` `earn_state`](../api/rest/info/spot.md#earn_state).

```json
// supply 5,000 USDC into the Earn pool for asset 100
{ "type": "earn_deposit", "params": { "asset": 100, "amount": "5000" } }
```

```json
// redeem shares (receive shares × share_value), idle-bounded
{ "type": "earn_withdraw", "params": { "asset": 100, "shares": "1234.5" } }
```

| Действие | Эффект |
|---|---|
| [`earn_deposit`](../api/rest/exchange.md#earn_deposit) | Внести котируемый актив → доли пула (1:1 для нового пула, иначе по NAV) |
| [`earn_withdraw`](../api/rest/exchange.md#earn_withdraw) | Погасить доли → котируемый актив, **ограничено свободной ликвидностью** |

**Ограничение свободной ликвидностью.** Вывод происходит мгновенно, но **ограничен свободной ликвидностью**
(`total_supplied − total_borrowed`): погашение, превышающее свободный остаток, выплачивает ровно свободный остаток и сжигает пропорционально меньше долей; пул с **нулевой свободной ликвидностью** (полностью выданный в кредит) отклоняет запрос на вывод до момента погашения займов заёмщиками. Это гарантирует, что вкладчик всегда может вывести до суммы, не выданной в кредит, и не подрывает обеспеченность книги займов.

## Риски

Earn **не является безрисковым инструментом**. Если позиция [спот-маржи](../products/spot-margin.md) закрывается с убытком, который не покрывается залогом заёмщика, **недостача социализируется между вкладчиками**: `total_supplied` пула уменьшается (но не ниже нуля), что снижает `share_value`. Защита пула обеспечивается **автоматическим ликвидатором** (работает на devnet): каждый блок маржинальные счета с отрицательным капиталом [принудительно закрываются](../products/spot-margin.md#liquidation) на уровне поддерживающей маржи, поэтому позиция разворачивается, как правило, пока стоимости ещё достаточно для погашения займа. Консервативный коэффициент поддерживающей маржи по паре (всё ещё проходит калибровку) задаёт этот буфер; страховой каскадный буфер перед вкладчиками запланирован, но пока не подключён. Существует также **риск ликвидности**: погашения ограничены свободной ликвидностью, поэтому из полностью задействованного пула нельзя выйти до погашения займов заёмщиками.

## Часто задаваемые вопросы

<details>
<summary>Показать FAQ</summary>

**В: Нужно ли вручную получать доходность?**
О: Нет. Доходность непрерывно реинвестируется в стоимость доли; вы фиксируете её при выводе.

**В: Почему моя APY ниже ставки заимствования?**
О: Только выданная в кредит (задействованная) часть пула зарабатывает проценты. APY ≈ ставка заимствования × утилизация.

**В: Могу ли я потерять основную сумму?**
О: Да, если убыток по спот-марже превышает залог заёмщика — непокрытая недостача социализируется между вкладчиками и снижает стоимость доли (страховой буфер перед вкладчиками запланирован, но пока не подключён). Такой сценарий задуман как редкий: автоматический ликвидатор принудительно закрывает убыточные позиции на уровне поддерживающей маржи, а коэффициент по паре задан консервативно. Earn менее рискован, чем торговое [хранилище](./vaults.md), но не является безрисковым.

**В: Почему я не могу вывести весь баланс прямо сейчас?**
О: Погашения ограничены **свободной ликвидностью** (`supplied − borrowed`). Если пул полностью выдан в кредит заёмщикам спот-маржи, вы можете вывести только свободный остаток; остальное разблокируется по мере погашения займов заёмщиками.

**В: Чем Earn отличается от хранилища MetaLiquidity?**
О: Earn — это пассивный **кредитный** пул USDC (доходность = проценты по займам). [Хранилище](./vaults.md) — это **торговый** LP-капитал (доходность/убыток = PnL стратегии). Разные профили риска.

</details>
