# Спот с маржой

:::info
**Доступно на devnet (предварительная версия).** Торговля спотом с кредитным плечом, финансируемая из кредитного пула [Earn](../concepts/earn.md). [Обычный спот](./spot.md) работает только в рамках баланса (без кредитного плеча); спот с маржой — надстройка, добавляющая заём и кредитное плечо поверх стандартного спота.
Полный цикл «депозит → заём → покупка с плечом → закрытие» И автоматическая [принудительная ликвидация](#liquidation) полностью функционируют **на devnet уже сейчас** (см. раздел [доступных действий](#action-surface) ниже). Считайте это **предварительной версией**: **коэффициенты поддерживающей маржи на уровне каждой пары являются параметрами управления и всё ещё калибруются**.
Кредитное плечо работает на devnet; не считайте это гарантией безопасности в продуктовой среде при промышленных нагрузках.
:::

## Кратко

Спот с маржой позволяет **занимать котировочную валюту (USDC) под залог для покупки спота с кредитным плечом**, вместо того чтобы платить 100% стоимости авансом. Заёмные средства в USDC поступают из пула [Earn](../concepts/earn.md), вы платите **проценты** по займу, и позиция имеет **поддерживающую маржу** и **цену ликвидации** — как у бессрочного контракта.

В первом релизе спот с маржой **изолирован на уровне каждой пары** — каждая позиция с кредитным плечом выставляет собственную маржу и ликвидируется независимо, отдельно от вашего кросс-счёта по бессрочным контрактам.

## Как это работает

```
1. Внесите залог (USDC) для пары — чистый буфер потерь.
2. Возьмите займ в котировочной валюте из пула Earn; заём финансирует покупку на 100%.
3. IOC-покупка базового актива на споте с использованием заёмных средств.
   Купленный базовый актив хранится ОБОСОБЛЕННО на маржинальном счёте.
4. Платите проценты по займу непрерывно, пока он открыт.
5. Закрытие: продайте базовый актив, погасите заём + начисленные проценты, оставьте остаток.
```

Залог **не** финансирует покупку — её финансирует заём. Залог — буфер потерь, обеспечивающий сверхзалоговость позиции; таким образом, кредитное плечо ≈
`notional / collateral`. Купленный базовый актив хранится **обособленно** на маржинальном счёте и никогда не смешивается с вашими доступными спотовыми балансами — при закрытии (или последующей ликвидации) затрагивается только эта конкретная позиция. Первый релиз допускает **одну открытую позицию на `(account, pair)`** (без добавления к существующей); открытый IOC-ордер **мгновенно погашает любой незатраченный заём**, поэтому непогашенный кредит равен только фактически потраченным на покупку средствам.

### Доступные действия {#action-surface}

Шесть действий [`/exchange`](../api/rest/exchange.md#spot-margin--earn) (каждое авторизуется отправителем) управляют полным циклом. Подтвердите зафиксированное состояние через
[`/info` `spot_margin_state`](../api/rest/info/spot.md#spot_margin_state).

| Действие | Эффект |
|---|---|
| [`spot_margin_deposit`](../api/rest/exchange.md#spot_margin_deposit) | Внесение залога в котировочной валюте для пары (маржа должна быть включена) |
| [`spot_margin_open`](../api/rest/exchange.md#spot_margin_open) | Заём + IOC-покупка базового актива с кредитным плечом; требует выполнения условий начальной маржи |
| [`spot_margin_close`](../api/rest/exchange.md#spot_margin_close) | IOC-продажа удерживаемого базового актива, погашение основного долга + процентов, возврат остатка |
| [`spot_margin_withdraw`](../api/rest/exchange.md#spot_margin_withdraw) | Вывод свободного залога (полный при нулевой позиции; ограничен начальной маржой при открытой позиции) |

### Маржа

```
position_value   = base_held × mark_px
debt             = borrowed + accrued_interest
equity           = position_value − debt
init_required    = position_value × spot_margin_initial_bps / 10000
maint_required   = position_value × spot_margin_maintenance_bps / 10000
health           = equity / maint_required
```

Открытие отклоняется, если это оставит `equity < init_required`. Позиция ликвидируется при `health < 1` (собственный капитал опускается до уровня поддерживающей маржи).

**Коэффициент поддерживающей маржи на споте является параметром каждой конкретной пары и устанавливается консервативно** — как правило, выше, чем у бессрочного контракта с сопоставимой ликвидностью. Причина механическая: ликвидация спота с маржой **продаёт базовый актив на спотовом стакане**, поэтому буфер поддерживающей маржи должен покрывать реализованное **проскальзывание** при раскрытии позиции по пороговой цене, иначе кредитный пул поглощает недостачу. Более тонкие (низколиквидные) стаканы дают больше проскальзывания и потому требуют более высокого коэффициента. Точное значение для каждой пары **калибруется на основе глубины стакана и волатильности пары относительно целевого предела проскальзывания при ликвидации** — это устанавливаемый управлением параметр риска, а не фиксированная константа; спот с маржой не включается для пары до завершения калибровки.
**На devnet коэффициенты на уровне пар всё ещё калибруются** — пара без откалиброванных параметров риска отклоняет каждое действие по споту с маржой
(`spot margin not enabled for pair`).

### Проценты

Заёмные средства в USDC начисляют проценты по ставке каждой пары (`spot_borrow_rate_bps`, годовая, начисляется каждый блок). Проценты поступают в пул [Earn](../concepts/earn.md), повышая его стоимость на долю — это и есть доходность кредиторов. В первом релизе ставка **фиксированная**; кривая на основе коэффициента использования появится в более поздних версиях.

### Ликвидация {#liquidation}

**Работает на devnet.** В каждом блоке цепь переоценивает каждый маржинальный счёт по спотовой отметке пары (цена последней сделки в стакане) и принудительно закрывает любой счёт, чей собственный капитал опустился ниже уровня поддерживающей маржи:

```
liquidate when   collateral + base_held × mark − debt  <  base_held × mark × maint_bps / 10⁴
```

Принудительное закрытие выполняется по **тому же урегулированному пути, что и добровольное закрытие**
— удерживаемый базовый актив IOC-продаётся на спотовом стакане, пул Earn получает обратно основной долг + проценты, остаток (за вычетом небольшой **комиссии за ликвидацию**, пополняющей страховой фонд протокола) возвращается, и счёт закрывается.
Два свойства защиты от каскадных ликвидаций соответствуют [принудительному закрытию бессрочных контрактов](../concepts/tiered-liquidation.md#how-a-forced-close-executes-the-price-floor):

- **Ценовой пол.** Принудительная продажа является ЛИМИТНЫМ ордером, ограниченным снизу значением
  `mark × (1 − floor)` (по умолчанию: половина коэффициента поддерживающей маржи, настраивается на уровне пары). Тонкий стакан никогда не «сметается» — то, что не продаётся выше пола, остаётся на счёте и переоценивается в следующем блоке.
- **Частичные заполнения сохраняют счёт открытым.** Реализованные поступления немедленно погашают долг; непроданный базовый актив повторно выставляется по мере возврата ликвидности.

Полный убыток урегулируется за счёт **собственного изолированного залога** позиции и пула Earn — на ваш кросс-счёт по бессрочным контрактам это не распространяется.

**Обработка недостачи.** Когда полное раскрытие не покрывает долг (выручка + залог оказываются меньше долга), весь основной долг по займу всё равно списывается из книги заёмных средств пула, а **недостача распределяется между поставщиками Earn** — совокупный объём средств в пуле уменьшается (с нулём в качестве нижней границы), что снижает стоимость доли.
Консервативный коэффициент поддерживающей маржи на уровне пары и автоматический ликвидатор призваны сделать подобные недостачи редкостью.

## Комиссии

Позиция по споту с маржой несёт три отдельных вида расходов:

| Вид расходов | Когда | Размер |
|---|---|---|
| **Торговая комиссия** | при IOC-заполнениях при открытии и закрытии | [ставка мейкера/тейкера спота](./spot.md#matching-fills-and-fees) для данной пары (спот с маржой торгует на спотовом стакане) |
| **Проценты по займу** | непрерывно, на непогашенный заём в USDC | `spot_borrow_rate_bps` — годовая ставка на уровне пары, начисляется каждый блок; поступает в пул [Earn](../concepts/earn.md) как доходность кредиторов |
| **Комиссия за ликвидацию** | только при принудительном закрытии | небольшая ставка на уровне пары, пополняющая страховой фонд протокола |

Открытие и закрытие являются обычными IOC-заполнениями спота, поэтому они оплачиваются по **спотовому** тарифному расписанию, а не по уровням бессрочных контрактов. Проценты по займу — это специфические для спота с маржой издержки, которые в точности равны доходности поставщиков [Earn](../concepts/earn.md). Все ставки являются параметрами управления на уровне пары; запросите их через
[`/info spot_margin_state`](../api/rest/info/spot.md#spot_margin_state) и
[`fee_schedule`](../api/rest/info.md#fee_schedule) спота.

## Область применения залога

| Релиз | Залог | Радиус поражения при ликвидации |
|---|---|---|
| V1 | **Изолировано на уровне пары** — каждая позиция вносит собственный USDC | Только данная позиция |
| Позднее | Кросс-приемлемый (использует общий залог счёта) | В масштабах всего счёта |

Изоляция на уровне пары обеспечивает ограниченность первого релиза: полный убыток по споту с маржой не затрагивает ваш кросс-баланс по бессрочным контрактам.

## Связь с Earn

Заёмщики по споту с маржой формируют **сторону спроса**; вкладчики [Earn](../concepts/earn.md) — **сторону предложения**. Проценты, уплачиваемые трейдерами по споту с маржой, в точности равны доходности вкладчиков Earn. Подробнее о расчёте доходности см. в разделе [Earn](../concepts/earn.md).

## Часто задаваемые вопросы

<details>
<summary>Показать FAQ</summary>

**Q: Затрагивает ли это обычный (без кредитного плеча) спот?**
A: Нет. Покупка спота на 100% за счёт собственного баланса работает в точности как прежде — спот с маржой является опциональной надстройкой.

**Q: Может ли убыток по споту с маржой затронуть мой счёт по бессрочным контрактам?**
A: В первом релизе — нет; спот с маржой изолирован на уровне пары. Кросс-залог — это более поздний опциональный апгрейд.

**Q: Откуда берутся заёмные средства в USDC?**
A: Из кредитного пула [Earn](../concepts/earn.md). Займы ограничены доступной (незаёмной) ликвидностью пула.

**Q: Какую ставку я плачу?**
A: Фиксированную годовую ставку на уровне пары в первом релизе, начисляемую каждый блок. Ценообразование на основе коэффициента использования появится позднее.

</details>
