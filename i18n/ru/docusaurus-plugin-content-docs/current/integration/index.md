## SDK

| Язык | Статус | Пакет |
|----------|--------|---------|
| TypeScript / JavaScript | предварительный просмотр | [`@metaflux/sdk`](./typescript-sdk.md) |
| Rust | предварительный просмотр | [`metaflux-client`](./rust-sdk.md) |

Для других языков (Python, Go, Java, C++ …) реализуйте подпись типизированных данных EIP-712 согласно [руководству по подписи](./typed-data-signing.md) — каждый шаг задокументирован с проработанными примерами. Протокол достаточно компактен, поэтому для нишевых стеков написать клиент вручную — вполне оправданное решение.

## Конечные точки сети

См. [сети](../networks.md) для полного справочника по каждой сети.

Шлюз (`https://api.<net>.mtf.exchange`) — единственная публичная точка входа.

| Путь | Обслуживает | Назначение |
|------|--------|---------|
| `POST /info` · `POST /exchange` · `GET /ws` | MTF-native | Нативный интерфейс в стиле snake_case |
| `POST /evm` | EVM JSON-RPC | RPC EVM-сайдчейна |
| `POST /faucet` | Кран | Тестовый кран для devnet/testnet |

В продуктивных развёртываниях TLS завершается на шлюзе, который прикрыт CDN; нода намеренно не имеет прямого выхода в интернет — она располагается за шлюзом. При самостоятельном запуске ноды тот же нативный интерфейс доступен напрямую по адресу `http://localhost:8080` (сырой EVM RPC по адресу `http://localhost:8545`).

## Типичные паттерны

- **Мейкер-бот** — подпись агентом, постоянное выставление котировок, сайдкар мониторинга рисков, ордера ALO для гарантированного мейкерского уровня
- **Мониторинг ликвидаций** — подписчик WebSocket на `marginEvents` + `userEvents` (`yellowCard`); инициирует пополнение маржи до наступления уровня T1
- **TWAP-обёртка** — выставляет `TwapOrder`, отслеживает `twapEvents` для телеметрии по срезам, опциональная ручная отмена в ходе выполнения
- **Управление хранилищем** — однократный `VaultDeploy`, затем ордера с подписью агента от адреса хранилища при перебалансировке
- **Институциональное хранение** — мастер-кошелёк с мультиподписью + агенты на каждый хост + обёртка с мультиподписью для крупных транзакций
