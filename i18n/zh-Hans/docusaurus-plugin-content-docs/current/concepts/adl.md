# 自动去杠杆 (ADL)

:::info
**预览功能**。T4 仅在保险基金无法完全弥补 T3 亏损时触发——在正常运营中很少见，但设计为确定性的。
:::

## 摘要

当保险基金无法吸收剩余清算亏损时，协议从同一工具上的盈利对手方回收资金，按其未实现盈亏的比例分配。MetaFlux 的分配使用在线学习排名，旨在最小化**超额减值**（超过亏损所需的减值）。

## ADL 何时触发

[分层阶梯](./tiered-liquidation.md)：

```
T0 黄牌  →  T1 部分  →  T2 全额  →  T3 后备  →  T4 ADL
```

T3 将清算账户的头寸交给后备多签，从保险基金中支付。如果 `保险基金 < 亏损`，协议移至 T4：未资助的剩余亏损被分配为有利可图对手方的头寸减值。

```
亏损 = 清算亏损 - 保险基金提取
如果 亏损 > 0:
    fire_adl(资产, 亏损)
```

保险基金提取本身受到速率限制——见 [分层清算](./tiered-liquidation.md#t3-backstop--netting-at-mark)。

## ADL 如何计算

> **深入阅读**："Autodeleveraging as Online Learning" (arXiv:2602.15182)。

MTF **不**使用单一排名分数。ADL 分为两个独立的子问题：1-D **严重程度**决策（本轮减值多少）在线学习，和确定性的**按比例分配**（谁，按盈亏能力）。

> ⚠️ **更正与之前的文本**。之前的文档描述了单一的在线学习*排名* `score = α·pnl% + β·leverage + γ·age`。这**不是**实现的算法。真实控制器是 `θ ∈ [0,1]` 通过投影 OGD 的严重程度 + 能力按比例分配。`α/β/γ` 排名公式被拒绝（"2D OGD——维度爆炸"）。经典的 `pnl% × leverage` 队列是 HL 基准，MTF 替代，不是 MTF 运行的。

### 1. 严重程度 — θ 上的 1-D 在线梯度下降

每一轮选择一个标量 `θ_t ∈ [0,1]` = 本轮亏损需要减值的分数：

```
B_t          = θ_t · D_t                                  # 本轮预算
θ_needed_t   = clamp(B̂_needed / D_t, 0, 1)                 # 实际所需的事前估计
grad         = D_t · sign(θ_t − θ_needed_t)
θ_{t+1}      = clamp(θ_t − η · grad, 0, 1)                 # 投影 OGD 步骤
```

`D_t` = 轮亏损，`B̂_needed` = 执行价格估计器对真实需要的猜测。

**步长 η** (`AdlController::current_eta`)：
- 默认模式是**自适应**（论文推论 1）：`η* = sqrt( (1 + 2·P_T^θ) / Σ D_t² )`，从运行遥测数据（`path_variation`、`cumulative_squared_deficit`）每轮重新计算。
- 在第一轮（`Σ D_t² == 0`）时，它回退到治理可调的 `η₀ = 0.01`（`default_eta`）。
- `Fixed(c)` 模式将 `η = c` 固定（治理 kill-switch / 可重现性）。

控制器有一个**动态遗憾界**（命题 1）：

```
Reg_T^dyn  ≤  sqrt( (1 + 2·P_T^θ) · Σ D_t² )
```

作为 `analytical_bound()` 公开；`check_bound(slack)` 断言经验遗憾 ≤ `slack · bound`（默认 slack 4）在混沌测试中。

所有分数字段（`θ`、`η`、`path_variation` 等）是 `rust_decimal::Decimal`；`sqrt` 是整数牛顿 `Decimal` sqrt（无 `f64`）；每个累加器使用 `saturating_*`（`Decimal::from(u128 > MAX)` 否则会 panic 并停止内核）。状态在 BOLE 累加器**槽 5** 中持久化。

### 2. 分配 — 确定性能力按比例

给定预算 `B_t`，跨有利可图的对手方 `W_t` 分配（每个都有减值能力 `u_i` = 可减值的未实现盈亏，`u128`）：

```
total_u = Σ u_i
x_i     = floor( u_i · B_t / total_u )      限制在 u_i 处        # 128 位乘法然后 128 位除法
```

整数除法**尘埃** `B_t − Σ x_i` 按**升序 AccountId 顺序**逐个单位重新分配给任何有剩余能力的赢家（`BTreeMap` 按键顺序迭代→字节在节点间相同）。如果 `B_t > total_u`，该轮受能力限制，`Σ x_i = total_u`。

这替代了被拒绝的矢量镜像下降和 ILP 分配器（ILP 是最优的但是非确定性求解器——不能上链）。

**为什么按比例**（HL 2025 年 10 月 10 日回放）：

| 算法 | 10 月 10 日总目标（低=更好） |
|-----------|-----------------------------------------|
| HL 生产（ROE 启发式） | ~$4500 万溢出 |
| **MTF 按比例** | **$340 万** (~13 倍更好） |
| 矢量镜像下降 | $441 万 |
| 最小-最大 ILP（最优，仅离线） | $10.6 万 |

按比例也给出**0 %** 单调性违规（vs HL 的 11.4 %）和排名稳定性 ≈ 1.0（vs HL 的 0.34）。

### 报价 ADL 价格（读方）

EVM 预编译 `0x0902 adl_pro_rata_price` 让 Solidity 帮助程序*报价* 大小为 N 的 ADL 会清算的 VWAP 填充，按照侧面合适的优先级遍历队列（多头 ADL：最高价优先；空头 ADL：最低价优先）——**所有价格都在 1e8 定点平面上**（`price_e8`、`capacity_e8`）。它仅支持按比例；严重程度 OGD 位于核心状态，不在无状态预编译中（严重程度是每轮一次决策；价格报价是每秒多个调用）。

## "减值"的机制含义

减值不是头寸转移——对手方的头寸规模**缩小**，他们的未实现盈亏被转换为实现亏损。垂直账户的相反方向头寸也按相同数量消失。

具体来说：假设账户 A 在入场价 100 处持多 1 BTC，账户 B 在入场价 100 处持空 1 BTC，标记价格 = 110。

- A 盈利（+10 USDC 未实现）。
- B 是垂直账户，被清算；头寸在标记价格 110 处结清但 B 只有 5 USDC 权益。5 USDC 亏损。
- 保险基金：0（已耗尽）。
- ADL 对 A 触发：
  - A 的多头减少到 0.5 BTC。
  - A 实现 +5 USDC 盈亏（被减值的部分）。
  - A 的剩余 0.5 BTC 多头在入场价 100、标记价格 110、+5 USDC 未实现。
  - B 的空头完全平仓。

A 保留其剩余头寸上的未实现盈亏；A 仅损失*平仓*部分的盈亏。

## 通知

ADL 事件在 [`userEvents` WS 通道](../api/ws/subscriptions.md#userevents)触发：

```json
{
  "kind":        "adl",
  "asset":       0,
  "haircut_sz": "50000000",
  "realised_pnl": "5000000",
  "block":       12345
}
```

加上账户范围通知：

```json
{ "kind": "marginChange", "free": "...", ... }
```

对于自动化机器人，将 `adl` 事件视为强制成交——你的头寸改变了，协议给了你成交价格（减值区块处的标记价格），你通常会重新评估你的策略。

## 预测 ADL 敞口

对于风险监控，[`/info`](../api/rest/info.md) 账户状态包括 `adl_rank_estimate` 字段：

```json
"adl_rank_estimate": {
  "asset": 0,
  "percentile": 95,
  "score": 1.2
}
```

`percentile: 95` 意味着你在该资产风险最高的前 5% 账户中。前十分之一的账户面临最大的 ADL 触发敞口。

这是一个估计——实际排名发生在 ADL 触发时刻，针对当时状态。对于运行大型账簿的做市商，主要风险是集中（一个大头寸主导资产的未平仓利息）；跨资产多元化可减少 ADL 敞口。

## 边界情况

<details>
<summary>显示边界情况</summary>

- **一个区块中的多个亏损**。每个都独立地针对当时的对手方集合分配。排名可以在事件之间移动。
- **空对手方集合**。如果字面上同一工具上不存在任何盈利对手方，亏损社会化到保险基金的"未覆盖亏损"寄存器，在下一个基金补充时支付。对于流动资产不应发生；在长尾 MIP-3 市场上理论上可能发生。
- **PM 注册的对手方**。ADL 仍然针对同一工具上的未实现盈亏——PM 注册不改变 ADL 的按资产粒度。PM 情景引擎在下一个区块看到减值后状态。
- **现货市场**。现货在 perp 意义上没有未实现盈亏。现货 ADL 对 V1 未定义；现货头寸从 ADL 排名中排除。

</details>

## 序列 — 在薄尾资产上的 ADL

```
区块 T：账户 X 在资产 42（MIP-3 市场）上清算，亏损 = 100 USDC
        资产 42 上的保险基金 = 60 USDC
        D_t（亏损）= 40 USDC
        严重程度控制器：θ_t 解析为 1.0（需要全部亏损）
        B_t = θ_t · D_t = 40                  # 本轮预算
        资产 42 上的赢家 W_t（按减值能力 u_i）：
          A: u_A = 30
          B: u_B = 50    →    total_u = 80
        按比例：x_i = floor(u_i · B_t / total_u)
          x_A = floor(30·40/80) = 15
          x_B = floor(50·40/80) = 25          # Σ = 40，无尘埃
        结果：
          A 的头寸减值 15 USDC 盈亏实现，15 保留
          B 的头寸减值 25 USDC 盈亏实现，25 保留
```

（分配是**能力按比例**，不是分数排名走：每个赢家放弃相同*分数*的能力——这里 50 %——这正是按比例购买的最小-最大公平性属性。比较这个与旧的"按分数排名、优先耗尽顶层"模型，这不是代码做的。）

## 常见问题

<details>
<summary>显示常见问题</summary>

**问：我可以选择退出 ADL 吗？**
答：不能。ADL 是协议级亏损互相化机制；选择退出只会将亏损推向他人。最小化超额减值目标提供保护。

**问：为什么按盈亏能力按比例分配而不是分数排名队列？**
答：按比例减值每个赢家相同*分数*的可减值盈亏——内置最小-最大公平性，无单调性违规（两个相同能力的账户遇到相同结果），排名稳定性 ≈ 1.0。它在 2025 年 10 月 10 日回放时测量表现比 HL 的 ROE 启发式队列好 ~13 倍，在离线 ILP 最优值的 ~30 % 以内，同时保持完全确定性和链上。*严重程度*（总共减值多少）是在线学习的部分；*谁支付*是纯按比例。

**问：ADL 是否尊重 Strict-Iso？**
答：是的。ADL 按构造是按资产的；Strict-Iso 头寸是对手方候选人当且仅当它们持有相同资产。

**问：排名在验证器间是确定性的吗？**
答：是的——所有输入（每个赢家的盈亏能力 `u_i`、轮亏损 `D_t`）从已提交状态读取，严重程度控制器的状态（`θ`、`path_variation`、`Σ D_t²`、`η`）位于 BOLE 累加器槽 5，折叠到 LtHash 中，所以每个节点验证字节相同。按比例使用 128 位整数乘/除，升序 AccountId 尘埃处理——无浮点，无 `HashMap`。

</details>
