# 对冲模式（双向仓位）

:::info
**已上线。** 可选的切换开关、显式的每侧订单路由、**独立的单腿保证金** 和 **双腿仓位报告** 已发布：账户可以切换到对冲模式（在平仓时），通过 `position_side` 将每个订单路由到显式的腿，每条腿都会单独提交保证金并作为自己的仓位对象报告。**单腿清算** 部分已到位：当账户被标记为强制平仓时，两条腿都会被扫描和评分，确定性的平仓顺序（较大的维持保证金腿优先）在验证者之间的计算方式相同——对于账簿的实际单腿平仓发出仍在推出中。默认且推荐的行为仍然是单向（每个市场单个净头寸）。
:::

## 总结

默认情况下，账户在每个市场持有 **一个净头寸**（单向）：买入而空头会减少，然后翻转，同一个头寸。**对冲模式** 让账户可以在同一市场同时持有 **多头腿和空头腿**，分别追踪。

对冲模式是 **每个账户可选的**，**没有余额阈值** —— 任何账户都可以启用它。它只能在账户 **在每个市场都平仓时** 进行切换。

## 单向对比对冲

| | 单向（默认） | 对冲 |
|---|---|---|
| 每个市场的仓位 | 1 个净头寸（有符号） | 最多 2 个（一个多头腿 + 一个空头腿） |
| "空头时买入" | 减少，然后翻转净头寸 | 只减少 **空头** 腿（或打开/扩展 **多头** 腿 —— 您选择） |
| 订单方向选择 | 从买入/卖出推断 | **显式** `position_side`（`long` / `short`）必需 |
| 保证金 | 一个净需求 | 每条腿独立保证金 |
| 清算 | 一个清算价格 | 每条腿根据其自己的维持保证金评分；确定性平仓顺序已到位，单腿平仓发出正在推出中 |
| 报告 | 一个净头寸对象 | 每个非零腿一个对象（每个标记有 `position_side`） |

切换、每侧路由、独立单腿保证金和双腿报告已上线；单腿清算选择已到位，单腿平仓发出仍在推出中（请参阅上面的状态说明）。

## 启用它

切换头寸模式是一个签名操作；只有当账户 **在每个市场上都平仓时** 才是合法的。

```json
// enable hedge mode (only legal when flat on ALL markets)
{ "type": "set_position_mode", "params": { "hedge": true } }
```

```json
// back to one-way (also only when flat)
{ "type": "set_position_mode", "params": { "hedge": false } }
```

`hedge` 是一个布尔值：`true` = 对冲（双向），`false` = 单向（默认）。
**在所有市场平仓** 的前提条件是安全规则 —— 在持有任何未平仓头寸时切换会被拒绝（一个清晰的无操作，不会改变任何东西），所以净头寸永远不会被默默地重新解释为一个困顿的腿。将模式设置为它已经具有的值（在平仓时）是一个无操作的成功。

有关请求/响应详细信息，请参阅 `/exchange` 引用中的 [`set_position_mode`](../api/rest/exchange.md#set_position_mode)。

## 在对冲模式中放置订单

在对冲模式中，每个订单 **必须携带显式的 `position_side`**（`long` / `short`）。单向账户 **不应该** 发送 `position_side`；对冲账户必须。该字段在 `submit_order` 订单正文中，与 `side`、`reduce_only` 等相邻。

```json
{
  "type": "submit_order",
  "order": {
    "owner": "0x...aa", "market": 0, "side": "bid",
    "kind": "limit", "size": 100000000, "limit_px": 5000000000,
    "tif": "gtc", "stp_mode": "cancel_oldest",
    "reduce_only": false,
    "position_side": "long"
  }
}
```

| 意图 | `side` | `position_side` | `reduce_only` |
|---|---|---|---|
| 打开/添加到多头 | `bid` | `long` | false |
| 减少/平仓多头 | `ask` | `long` | true |
| 打开/添加到空头 | `ask` | `short` | false |
| 减少/平仓空头 | `bid` | `short` | true |

`position_side` 是 **显式的，永远不会被推断** —— 否则，一个打算 *减少空头* 的买单可能会错误地打开或增加多头。`reduce_only` 是 **仅针对命名腿进行评估**：空头腿上的 `reduce_only` 订单永远不会接触多头腿。

对冲模式中 **没有"翻转"**。平仓多头腿永远不会打开空头 —— 这是针对空头腿的单独订单。

## 保证金

每条腿 **独立保证金** —— 多头腿和空头腿各自提交其自己的初始和维持保证金，汇总为账户需求：

```
required_margin = init_margin(long_leg) + init_margin(short_leg)
```

这是故意保守的：一个市场中的多头+空头在价格条款上是无风险的，但每条腿仍然会占用保证金。（未来的升级可能会在 [投资组合保证金](./portfolio-margin.md) 下为抵销腿提供净额信用；在此之前，仅在您想要真正独立的敞口时才持有两条腿，例如，您打算独立管理的不同入场价格。）

每条腿保持其自己的 [保证金模式](./margin-modes.md) —— 例如，您可以运行多头腿隔离，空头腿交叉。

## 清算

当账户被标记为强制平仓时，两条腿都会被扫描并根据其自己的维持保证金贡献进行评分，确定性的平仓顺序 —— **较大的维持保证金腿优先**（平手：多头优先于空头），在验证者之间计算相同 —— 通过标准 [分级清算](./tiered-liquidation.md) 梯形计算。对于账簿的实际单腿平仓发出仍在推出中；当它到达时，清算一条腿将不会接触另一条腿。

## 报告

当对冲模式开启时，账户状态和 `/info` 仓位读取为有两条腿的市场返回 **每个非零腿一个仓位对象**，每个都标记有其 `position_side`（`"long"` / `"short"`）。单向账户返回单个 *净* 头寸，**没有** `position_side` 字段，完全如同今天。市场级别的未平仓利息保持单个净数字。请参阅 [`account_state` 仓位](../api/rest/info.md#account_state)。

## 常见问题

<details>
<summary>显示常见问题</summary>

**问：我必须使用对冲模式吗？**
答：不是。单向（净）是默认值，其行为与之前完全相同；对冲模式纯粹是可选的。

**问：启用它有最低余额要求吗？**
答：没有阈值 —— 任何账户都可以在平仓时切换到对冲模式。

**问：为什么我在有未平仓头寸时不能切换？**
答：为了防止现有净头寸变成模糊的、困顿的腿。平仓，然后切换。

**问：等量的多头 + 空头需要双倍的保证金吗？**
答：是的 —— 腿是独立保证金的，所以抵销的多头+空头会占用两条腿的保证金。抵销腿的净额信用是后期的增强功能（在 [投资组合保证金](./portfolio-margin.md) 下）。

</details>
