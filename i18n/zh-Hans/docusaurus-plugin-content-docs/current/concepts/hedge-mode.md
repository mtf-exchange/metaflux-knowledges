# 对冲模式（双向持仓）

:::info
**已上线。** 账户自选开关、每笔订单的明确持仓方向路由、**各腿独立保证金**以及**双腿持仓报告**均已上线：账户可在空仓状态下切换至对冲模式，通过 `position_side` 将每笔订单路由至指定腿，各腿各自占用保证金并以独立持仓对象呈现。**各腿清算**功能已部分就位：账户被标记为强制平仓时，两条腿均会被扫描评分，确定性平仓顺序（维持保证金占用较大的腿优先平仓）已在各验证节点间保持一致 —— 对应腿实际挂单平仓的执行功能仍在陆续上线中。默认及推荐行为仍为单向模式（每个市场仅持有一个净头寸）。
:::

## 概述

默认情况下，账户在每个市场持有**一个净头寸**（单向模式）：做空时买入会减少乃至反转该头寸。**对冲模式**允许账户在**同一市场同时持有多头腿和空头腿**，两者独立跟踪。

对冲模式**由账户自主开启，无最低余额门槛** —— 任意账户均可启用。切换前，账户必须在**所有市场均处于空仓状态**。

## 单向模式与对冲模式对比

| | 单向模式（默认） | 对冲模式 |
|---|---|---|
| 每市场持仓数 | 1 个净头寸（带符号） | 最多 2 个（多头腿 + 空头腿） |
| "做空时买入" | 减少净头寸，直至反向 | 仅减少**空头**腿（或开仓/加仓**多头**腿 —— 由您指定） |
| 订单方向选择 | 由买/卖自动推断 | 必须**明确**指定 `position_side`（`long` / `short`） |
| 保证金 | 单一净头寸需求 | 各腿独立计算保证金 |
| 清算 | 单一清算价格 | 各腿按自身维持保证金独立评分；确定性平仓顺序已就位，各腿挂单平仓执行仍在上线中 |
| 报告 | 单一净头寸对象 | 每个非零腿各一个对象（均标注 `position_side`） |

开关、各方向路由、各腿独立保证金以及双腿报告均已上线；各腿清算选择已就位，各腿挂单平仓执行仍在上线中（详见上方状态说明）。

## 启用方式

切换持仓模式是一项需要签名的操作；仅在账户**所有市场均处于空仓状态**时方可执行。

```json
// enable hedge mode (only legal when flat on ALL markets)
{ "type": "set_position_mode", "params": { "hedge": true } }
```

```json
// back to one-way (also only when flat)
{ "type": "set_position_mode", "params": { "hedge": false } }
```

`hedge` 为布尔值：`true` = 对冲模式（双向），`false` = 单向模式（默认）。**全市场空仓**前提条件是一项安全规则 —— 持有任何未平仓头寸时发起切换请求将被拒绝（无任何副作用，不修改任何状态），因此净头寸永远不会被悄悄转化为孤立的腿。在空仓状态下将模式设置为当前已有值，属于无操作成功返回。

请参阅 `/exchange` 参考文档中的 [`set_position_mode`](../api/rest/exchange.md#set_position_mode) 了解请求/响应详情。

## 在对冲模式下下单

对冲模式下，每笔订单**必须明确携带 `position_side`**（`long` / `short`）。单向模式账户**不得**发送 `position_side`；对冲模式账户则必须发送。该字段位于 `submit_order` 订单体中，与 `side`、`reduce_only` 等字段并列。

```json
{
  "type": "submit_order",
  "order": {
    "owner": "0x...aa", "market": 0, "side": "bid",
    "kind": "limit", "size": 100000000, "limit_px": 5000000000,
    "tif": "gtc", "stp_mode": "cancel_oldest",
    "reduce_only": false,
    "position_side": "long"
  }
}
```

| 操作意图 | `side` | `position_side` | `reduce_only` |
|---|---|---|---|
| 开仓 / 加仓多头 | `bid` | `long` | false |
| 减仓 / 平仓多头 | `ask` | `long` | true |
| 开仓 / 加仓空头 | `ask` | `short` | false |
| 减仓 / 平仓空头 | `bid` | `short` | true |

`position_side` **必须明确指定，系统不会自动推断** —— 否则原本用于*减少空头*的买单可能会误开或加仓多头。`reduce_only` **仅针对指定腿**进行评估：空头腿上的 `reduce_only` 订单永远不会影响多头腿。

对冲模式下**不存在"反转"**。平仓多头腿不会自动开空 —— 开空是对空头腿单独下达的订单。

## 保证金

各腿**独立**计算保证金 —— 多头腿和空头腿各自占用独立的初始保证金和维持保证金，并汇总计入账户总需求：

```
required_margin = init_margin(long_leg) + init_margin(short_leg)
```

这是有意为之的保守设计：同一市场的多空头寸在价格上呈 delta 中性，但每条腿仍需单独占用保证金。（未来版本可能会在[组合保证金](./portfolio-margin.md)框架下提供对冲腿的保证金轧差减免；在此之前，仅在您确实需要独立管理不同持仓敞口时（例如不同入场价且计划分别管理）才建议同时持有两腿。）

每条腿保留各自的[保证金模式](./margin-modes.md) —— 例如，您可以将多头腿设为逐仓模式，将空头腿设为全仓模式。

## 清算

账户被标记为强制平仓时，两条腿均会按各自的维持保证金贡献进行扫描和评分，确定性平仓顺序 —— **维持保证金占用较大的腿优先平仓**（占用相同时多头先于空头），在各验证节点间保持一致 —— 通过标准[分级清算](./tiered-liquidation.md)阶梯计算得出。实际的各腿挂单平仓执行仍在上线中；上线后，清算某一条腿将不会影响另一条腿。

## 报告

开启对冲模式后，账户状态及 `/info` 持仓查询将为同时持有两腿的市场**每个非零腿各返回一个持仓对象**，每个对象均标注其 `position_side`（`"long"` / `"short"`）。单向模式账户返回单一*净*头寸，**不含** `position_side` 字段，与现有行为完全一致。市场层面的未平仓量仍以单一净值呈现。请参阅 [`account_state` 持仓](../api/rest/info.md#account_state)。

## 常见问题

<details>
<summary>展开常见问题</summary>

**问：必须使用对冲模式吗？**
答：不必要。单向（净头寸）模式是默认模式，行为与以前完全相同；对冲模式完全由用户自主选择开启。

**问：启用对冲模式有最低余额要求吗？**
答：没有门槛 —— 任意账户在空仓状态下均可开启对冲模式。

**问：为什么持有未平仓头寸时无法切换模式？**
答：这是为了防止已有的净头寸变成状态不明的孤立腿。请先平仓，再切换模式。

**问：等量多空头寸需要双倍保证金吗？**
答：是的 —— 各腿独立计算保证金，因此等量多空头寸会同时占用两条腿的保证金。对冲腿的保证金轧差减免是后续增强功能（归属于[组合保证金](./portfolio-margin.md)）。

</details>
