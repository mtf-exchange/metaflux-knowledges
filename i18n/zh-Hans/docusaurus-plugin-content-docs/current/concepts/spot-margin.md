# 现货保证金

:::info
**在开发网上可用（预览版）。** 由[收益](./earn.md)借贷池资助的杠杆现货交易。[普通现货](./spot-trading.md)仅支持余额（无杠杆）；现货保证金是在其之上叠加借贷 + 杠杆的层。完整的存入 → 借贷 → 杠杆买入 → 平仓循环AND自动[强制清算](#liquidation)完整运行在**开发网上**（见下面的[操作表面](#action-surface)）。将其视为**预览版**：按交易对**维持保证金比率是正在校准的治理参数**。杠杆在开发网上有效；请勿假设大规模生产安全性。
:::

## 摘要

现货保证金让你**借入报价（USDC）以抵押品购买现货，使用杠杆**，而不是预先支付100%。借入的USDC来自[收益](./earn.md)池，你需要支付**利息**，该头寸带有**维持保证金**和**清算价格**，类似于永续合约。

在首个版本中，现货保证金是**按交易对隔离的** — 每个杠杆现货头寸单独发布其保证金并单独清算，与你的永续合约交叉账户分离。

## 工作原理

```
1. 为交易对发布抵押品（USDC） — 纯粹的亏损缓冲。
2. 从收益池借入报价；借款资金100%资助购买。
3. 使用借入的报价在现货簿上IOC买入基础资产。
   购买的基础资产在保证金账户上隔离持有。
4. 在贷款开放期间持续支付借款利息。
5. 平仓：卖出基础资产，偿还借款 + 累计利息，保留余额。
```

抵押品不资助购买 — 借款资金才是。抵押品是亏损缓冲，使头寸过度抵押化，因此杠杆 ≈ `notional / collateral`。购买的基础资产在保证金账户上**隔离**持有，永远不与你的可花费现货余额混合，因此平仓（或稍后清算）仅触及该特定头寸。首个版本允许**每个`(account, pair)`最多一个开仓**（不支持加仓）；开放的IOC**立即偿还任何未花费的借款**，因此未偿还的贷款只等于购买实际花费的金额。

### 操作表面

六个[`/exchange`](../api/rest/exchange.md#spot-margin--earn)操作（均为发送方授权）驱动循环。通过[`/info` `spot_margin_state`](../api/rest/info.md#spot_margin_state)确认提交状态。

| 操作 | 效果 |
|---|---|
| [`spot_margin_deposit`](../api/rest/exchange.md#spot_margin_deposit) | 为交易对发布报价抵押品（必须启用保证金） |
| [`spot_margin_open`](../api/rest/exchange.md#spot_margin_open) | 借贷 + IOC买入基础资产（受初始保证金要求限制）杠杆； |
| [`spot_margin_close`](../api/rest/exchange.md#spot_margin_close) | IOC卖出持有的基础资产，偿还本金 + 利息，返回余额 |
| [`spot_margin_withdraw`](../api/rest/exchange.md#spot_margin_withdraw) | 提取自由抵押品（平仓时为全部；开仓时受初始保证金限制） |

### 保证金

```
position_value   = base_held × mark_px
debt             = borrowed + accrued_interest
equity           = position_value − debt
init_required    = position_value × spot_margin_initial_bps / 10000
maint_required   = position_value × spot_margin_maintenance_bps / 10000
health           = equity / maint_required
```

如果开仓会导致`equity < init_required`，则开仓被拒绝。当`health < 1`时头寸被清算（权益下跌到维持保证金水平）。

现货**维持比率是按交易对参数，保守设置** — 通常高于同等流动性的永续合约。原因是机械性的：现货保证金清算**将基础资产卖入现货簿**，因此维持缓冲必须覆盖在阈值处平仓头寸所产生的**滑点**，否则借贷池承担不足。较薄（长尾）的簿面会产生更多滑点，因此需要更高的比率。每个交易对的确切值**根据该交易对的簿面深度和波动性相对于目标清算滑点界限进行校准** — 它是治理设置的风险参数，而非固定常数，交易对在其比率被校准前不启用现货保证金。**在开发网上这些按交易对的比率仍在校准中** — 未校准风险参数的交易对会拒绝其所有现货保证金操作（`spot margin not enabled for pair`）。

### 利息

借入的USDC按每交易对利率（`spot_borrow_rate_bps`，年化，每块累计）累计利息。利息流向[收益](./earn.md)池，提升其每份价值 — 这是贷方的收益。在首个版本中该利率是**固定的**；基于利用率的曲线是后期升级。

### 清算

**在开发网上实时运行。** 每个块链在交易对现货标记（簿的最后交易价格）处重新评估每个保证金账户，并强制平仓任何权益跌破维持保证金水平的账户：

```
liquidate when   collateral + base_held × mark − debt  <  base_held × mark × maint_bps / 10⁴
```

强制平仓通过**与自愿平仓相同的结算路径运行**
— 持有的基础资产在现货簿上IOC卖出，收益池获得本金 + 利息偿还，余额（减去小额**清算费用**，该费用为协议保险基金提供资本）被返回，账户关闭。两个反级联属性反映[永续合约强制平仓](./tiered-liquidation.md#how-a-forced-close-executes-the-price-floor)：

- **价格下限。** 强制卖出是一个LIMIT，界限为
  `mark × (1 − floor)`（默认：维持比率的一半，按交易对可配置）。薄簿面永远不会被扫平 — 任何无法以高于下限价格卖出的部分保持持有并在下一块重新评估。
- **部分成交保持账户开放。** 已实现收益立即偿还债务；未卖出的基础资产在流动性返回时重试。

爆仓针对头寸的**自有隔离抵押品**和收益池进行结算 — 它永远不会到达你的永续合约交叉账户。

**不足处理。** 当完全平仓无法覆盖债务（收益 + 抵押品不足）时，整个贷款本金仍然离开池的借出簿，**不足被分摊给收益供应商** — 池的供应总额减少（下限为零），这会降低份额价值。保守的按交易对维持比率和自动清算器的存在是为了使不足情况稀少。

## 抵押品范围

| 版本 | 抵押品 | 清算影响范围 |
|---|---|---|
| V1 | **按交易对隔离** — 每个头寸发布其自有USDC | 仅该头寸 |
| 后期 | 交叉合格（共享账户抵押品） | 账户范围 |

按交易对隔离将首个版本保持在受控范围内：杠杆现货爆仓无法到达你的永续合约交叉余额。

## 与收益的关系

现货保证金借方是**需求方**；[收益](./earn.md)存款人是**供应方**。现货保证金交易者支付的借款利息正是收益存款人收到的收益。见[收益](./earn.md)了解收益计算。

## 常见问题

<details>
<summary>显示常见问题</summary>

**Q: 普通（无杠杆）现货会受影响吗？**
A: 不会。使用你自有余额的100%购买现货工作方式完全相同 — 现货保证金是可选叠加。

**Q: 我的现货保证金亏损会接触我的永续合约账户吗？**
A: 不会（在首个版本中） — 现货保证金按交易对隔离。交叉抵押品是后期可选升级。

**Q: 借入的USDC来自哪里？**
A: [收益](./earn.md)借贷池。借贷以池的可用（未借出）流动性为上限。

**Q: 我支付什么利率？**
A: 首个版本中固定的按交易对年化利率，每块累计。基于利用率的定价稍后推出。

</details>
