# 现货交易

:::tip
**已上线。** 纯现货交易已部署 — 代币兑代币订单簿，独立于永续合约，无杠杆无仓位。（杠杆现货是独立的计划性功能 [现货保证金](./spot-margin.md) 路线。）
:::

## 简明总结

现货是一个**代币兑代币中央限价订单簿**：你以自选价格将一种代币兑换为另一种代币。它完全独立于永续合约 — 独立的订单簿、独立的余额、**无杠杆无仓位**。你只能交易自己拥有的资产。挂单的现货订单会将填单时应支付的资金锁定在**预留余额**（托管）中；这些资金在填单时支付给交易对手方，或在撤销时退回给你。

现货订单只是另一个 [`/exchange`](../api/rest/exchange.md) 操作 — 用 [`spot_order`](../api/rest/exchange.md#spot_order) 下单，用 [`spot_cancel`](../api/rest/exchange.md#spot_cancel) 撤销。两者都是**发送人授权**的（恢复的签名者是交易者；没有 `owner` 字段），可由主账户或活跃的 [代理钱包](./agent-wallets.md) 签名。

## 什么是现货交易对

现货交易对用**基础代币**与**计价代币**进行交易（例如 `B/Q`）。订单方向确定交易方向：

| `side` | 你给出 | 你收到 | 挂单时托管锁定 |
|--------|----------|-------------|------------------------------|
| `bid`（买入） | 计价代币 | 基础代币 | **计价代币** — 以你的限价计算的名义价值 |
| `ask`（卖出） | 基础代币 | 计价代币 | **基础代币** — 你提供的基础代币 |

订单字段是**现货交易对 id**（`pair`），它不同于永续合约的 `market` id 和代币 id。交易对部署在 [MIP-1](../mip/mip-1.md) 下（现货代币标准+市场部署）；每个交易对都有自己的基础代币/计价代币、大小精度、可选的最小名义价值和费用覆盖。

## 预留余额托管

这是现货在没有杠杆情况下保持溶剂性的核心。当 `gtc` / `alo` 订单（或其未匹配部分）**挂单**在簿上时，协议将其在完全填单时应支付的资金从你的可用余额转移到**预留余额**：

- 挂单的**买入**预留**计价代币**，其金额等于限价时的名义价值（`size × limit_px`）。
- 挂单的**卖出**预留提供的**基础代币**。

预留资金不可用。它们会：

- **在订单成交时支付给交易对手方**，
- **在 [撤销](#生命周期--撤销退回托管) 时、自交易预防时或市场停用时退回到你的可用余额**。

单个代币的余额在每次挂单、成交、撤销和 STP 事件中精确守恒 — 可用加预留的金额在每个账户每个代币上是不变的（跨随机化挂单/成交/撤销流进行了模糊测试验证）。

## 可承受性夹紧

你永远无法挂单或成交超出你余额能够覆盖的数量。在接纳时，订单大小会被**夹紧**到你的余额能覆盖的范围：

- **买入**订单被夹紧为 `quote_balance ÷ limit_px`，
- **卖出**订单被夹紧为你实际拥有的基础代币。

完全不可承受的订单是一个**已接受的空操作** — 无任何成交、无挂单、无订单 id 消耗。部分可承受的订单交易/挂单可承受的部分。因为夹紧在**匹配之前**运行，所以每个成交和每个托管预留都是有资金支持的；没有填单后放弃。

## 匹配、成交和费用

现货匹配使用与 MetaFlux 其他部分相同的价格-时间 CLOB。成交在**挂单者的挂单价格**处交换基础代币和计价代币。

费用从每一方**收到的部分**中收取：

- **接受方**费用从接受方收到的部分中收取，
- **挂单方**费用从挂单方收到的部分中收取。

所以买方（收到基础代币）以基础代币支付费用；卖方（收到计价代币）以计价代币支付费用。费用计入专门的现货费用账户，与永续合约费用池分离。交易对可设置自己的 `taker_fee_bps` / `maker_fee_bps`；当交易对未设置时，应用全局现货默认值。详见 [费用](./fees.md#现货费用) 中的费率表。

## 有效期

现货订单携带与永续合约相同的有效期集合，外加一条现货特定规则：

| `tif` | 现货行为 |
|-------|------------------|
| `gtc` | 匹配能匹配的部分；任何未匹配部分**挂单**（有托管支持）直到成交或撤销 |
| `alo` | 仅增加流动性；成交的 `alo` 被**拒绝**（从不接受）。非成交的 `alo` 挂单 |
| `ioc` | 立即匹配能匹配的部分；未匹配部分被丢弃 — **从不挂单**、从不托管 |

`aon` 被拒绝（无核心等效）。自交易预防使用与永续合约相同的 [`stp_mode`](./order-types.md) 集合（`cancel_oldest` / `cancel_newest` / `cancel_both`）；不支持 `reject`。

:::info
**仅限价（暂时）。** 每个现货订单必须携带正的 `limit_px`。市价订单（`limit_px = 0`）**尚不支持** — 无限制的市价买入需要在路线图中的订单簿行走可承受性夹紧。发送限价订单；成本由 `limit_px × size` 界定。
:::

## 生命周期 — 撤销退回托管

[`spot_cancel`](../api/rest/exchange.md#spot_cancel) 在交易对上按 `oid` 撤销**你的**一个挂单，并将其锁定的托管退回到你的可用余额。

- **仅所有者。** 仅订单的所有者可以撤销它；第三方被拒绝（`not the order owner`）。
- **类型化缺失。** 未知或已消失的 `oid` 返回 `order not found`（无害的）。
- **始终可用。** 撤销**不**受现货停用限制 — 即使在禁用新订单时，你也可以始终退出挂单并回收其托管。

## 限制和治理

- **挂单上限。** 每个账户在每个现货交易对上最多可挂 **1000** 个订单；超过上限的新挂单被拒绝（`spot resting-order cap reached — cancel some orders first`）。已认可的做市商账户除外。`ioc` 订单从不挂单，所以从不受上限限制。
- **最小名义价值。** 交易对可设置最小名义价值；低于此值的订单被拒绝。
- **现货停用（治理）。** 现货交易可由治理全局启用或禁用。禁用时，**新**订单被拒绝（`spot trading disabled`），但撤销仍可用，所以挂单托管永不被困。

## 读取现货状态

现货余额和开放现货订单可通过 [`POST /info`](../api/rest/info.md) 查询。`spot_order` 返回一个**同步**的每订单状态一旦它提交 — 真实分配的 `oid` 伴随 `resting` 或 `filled` 条目（或 `error`），或 `pending` 如果没有提交在订单等待窗口内落地 — 与永续合约 [`submit_order`](../api/rest/exchange.md#submit_order) 的相同状态联合。

## 与现货保证金和 Earn 的关系

纯现货是**基线**：交易你拥有的资产，无杠杆、无清算。两个计划中的覆盖层建立在其上：

- [**现货保证金**](./spot-margin.md)（计划中）— 借计价代币对抗担保品以杠杆买入现货，带有维持保证金和清算价格。
- [**Earn**](./earn.md)（计划中）— 一个 USDC 借贷池，为现货保证金借贷融资并赚取借贷利息作为收益。

两者都是**可选覆盖层**；纯现货不受它们影响。

## 常见问题

<details>
<summary>显示常见问题</summary>

**Q: 现货交易需要担保品或保证金吗？**
A: 不需要。现货只涉及余额 — 你交易你拥有的资产。没有保证金、没有杠杆、没有清算。（杠杆是独立的计划性 [现货保证金](./spot-margin.md) 路线。）

**Q: 我的订单挂单时，我的资金会怎样？**
A: 它们被保存在预留余额（托管）中 — 不可用，但属于你。它们在成交时支付给交易对手方，或在撤销时返回到你的可用余额。

**Q: 为什么我的大额买单只部分成交/挂单了？**
A: 可承受性夹紧。订单大小被减少到你的计价代币余额以限价计算能覆盖的范围。完全不可承受的订单是一个已接受的空操作。

**Q: 我可以下现货市价单吗？**
A: 不可以 — 始终发送正的 `limit_px`。市价现货订单在路线图中。

**Q: 现货成交和永续合约成交在同一本簿上吗？**
A: 不。现货有自己的簿、余额和费用账户，完全独立于永续合约。

</details>
