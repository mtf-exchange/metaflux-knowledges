# 预言机价格

:::tip
**稳定版。**
:::

## 简介

**预言机价格**是协议为每个标的资产计算的参考价格，每个区块生成一次，取自**外部现货交易所的加权中位数**。它是外部锚定价格，[标记价格](./mark-prices.md)（其 C1 分量）和[资金费率](./funding-rates.md)（结算基准）均以此为基础。预言机的设计刻意采用**现货衍生、缓慢推送**的方式——它既不是 MetaFlux 订单簿价格，也不是最新成交价。

协议使用两组不同的价格来源，容易混淆：

| 来源组 | 交易所 | 聚合方式 | 用途 |
|----------|--------|-------------|--------|
| **现货预言机** | 最多 **10 个现货交易所** | 加权中位数 | `oracle_px`；标记价格 C1 锚点；资金费率结算名义价格 |
| **外部永续合约中间价** | **5 个永续合约交易所**（Binance、OKX、Bybit、Gate、MEXC） | 中位数（至少 2 个有效） | 仅用于标记价格 **C3** 分量 |

## 为什么用预言机而不用订单簿？

保证金、清算和资金费率都需要一个对手方**无法**通过在 MetaFlux 薄订单簿上单笔交易就能操纵的价格。对多个深度现货交易所取加权中位数正是如此：要移动它，你必须同时在多个交易所推动现货价格，成本极高且会自我套利。内部订单簿*确实*参与标记价格计算（通过 C2 和 C1 基差项），但始终与这个外部锚点混合使用。

## 构成方式

`oracle_px` 是该资产所有有效现货交易所的**加权中位数**。

### 默认现货权重表（总权重 = 15）

| 交易所 | 权重 | | 交易所 | 权重 |
|-------|-------:|-|-------|-------:|
| Binance | 3 | | Kraken | 1 |
| OKX | 2 | | KuCoin | 1 |
| Bybit | 2 | | Gate | 1 |
| Coinbase | 2 | | MEXC | 1 |
| Bitget | 1 | | MetaFlux spot | 1 |

这里使用**加权中位数**（而非加权平均数），确保单个交易所的异常报价无法拉偏结果——它只会改变落在加权中点的那个样本。

### 按交易对的治理权重

上述默认表是兜底配置。治理专属操作 `SetOracleWeights { asset_id, weights }`（`ActionId 148`）可**替换**（而非合并）某资产的权重表——这是必要设计，因为长尾市场和无许可市场（[MIP-3](../mip/mip-3.md)）通常未在 Binance / Coinbase 上架，使用默认权重将得不到任何有效价格。市场部署者**不能**自行设置权重（自选预言机来源等于自定标记价格）；新市场冷启动时使用默认表，只有治理可以覆盖。

已生效的每市场来源集合可通过 [`oracle_sources`](#查询) 查询，以交易所列表的子集掩码形式呈现。

## 可靠性规则

聚合器的设计原则是**优雅降级而非给出错误价格**。每次 tick 按顺序执行以下逻辑：

- **单来源过期检测。** 某个交易所在 `feed_staleness_ms`（默认 **60 秒**）内未产生新报价，则本次 tick 将其视为缺失。
- **跨来源异常值剔除。** 距离跨交易所中位数超过 `feed_deviation_pct`（默认 **5 %**）的交易所，在取加权中位数之前会被剔除——防范单个卡住/零值/胖手指报价的干扰。
- **对存活来源重新归一化。** 缺失的交易所权重置为 0，剩余权重重新归一化。
- **最低覆盖率保护。** 若本次 tick 中有效权重**低于总配置权重的 50 %**，预言机槽位**不更新**——保持上一个有效值。这是硬性下限，防止在全市场数据中断时仅凭一两个存活来源定价。

权重被设为 0 的交易所（例如该交易对已下架）永远不会被请求。

## 发布机制

合成后的 `oracle_px` **每个区块发布一次**，时间戳来自共识区块时间（而非本地时钟），并由活跃验证者集合中的预言机验证者签名。由于中位数计算、过期/异常过滤以及时间戳均来自共识，每个诚实验证者对同一区块会计算出**完全相同的**预言机快照。

## 与标记价格和资金费率的关系

- **标记价格。** 预言机是标记价格的 **C1 锚点**：`C1 = oracle + EMA(book_mid − oracle)`。若内部订单簿和外部永续合约数据均缺失，标记价格将完全退化为预言机价格。详见[标记价格](./mark-prices.md)。
- **资金费率。** 资金费率 = **冲击价格**（深度加权订单簿价格）与**预言机价格**之差，并以**预言机价格**结算。尤为重要的是：当某市场的预言机价格过期或不可信时，该市场的资金费率会被**关闭**，并逐渐衰减至 0，而不会用无人信任的价格结算。详见[资金费率](./funding-rates.md#gating-when-the-oracle-is-untrusted)。

## 查询

合成后的 `oracle_px` 以**整 USDC 精度**（如 `"67042.335"`）通过 [`market_info`](../api/rest/info/perpetuals.md#market_info) 接口返回，同时附带 `mark_px`：

```bash
curl -X POST https://devnet-gateway.mtf.exchange/info \
  -H 'content-type: application/json' \
  -d '{"type":"market_info","asset_id":0}'
```

```json
{
  "type": "market_info",
  "data": {
    "asset_id":  0,
    "mark_px":   "67042.335",
    "oracle_px": "67042.335"
  }
}
```

已生效的每市场来源集合可通过 `oracle_sources` 查询（某市场已启用的交易所子集）：

```bash
curl -X POST https://devnet-gateway.mtf.exchange/info \
  -H 'content-type: application/json' \
  -d '{"type":"oracle_sources","asset_id":0}'
```

每个交易所的原始输入及本次 tick 实际使用的权重存储在共识状态中；目前除来源子集外，暂未作为独立 wire 字段对外暴露。

## 边界情况

<details>
<summary>展开边界情况</summary>

- **某交易所价格卡死。** 会被过期过滤器（> 60 秒）或异常值过滤器（偏离跨交易所中位数 > 5 %）中的较早触发者剔除；中位数在存活来源上计算。
- **全市场中断（有效权重 < 50 %）。** 预言机槽位保持上一个有效值。标记价格 C1 继续使用该值，保证保证金和清算有价格锚定，而不会冻结或跌落到薄板打印的价格。
- **长尾市场未在主流交易所上架。** 冷启动时使用默认权重表（大多数情况下解析为空），直到治理为该交易对设置 `SetOracleWeights` 覆盖，指向实际上架该市场的交易所。
- **现货预言机健康但永续合约价格偏离。** 这是正常现象——永续合约可以长期对现货溢价或折价交易。预言机（现货）保持不变；标记价格通过 C2/C3 和 C1 基差 EMA 跟随永续合约价格变化。详见[标记价格与预言机价格的差异](./mark-prices.md#mark-vs-oracle--why-they-diverge)。

</details>

## 常见问题

<details>
<summary>展开常见问题</summary>

**Q：预言机价格和标记价格是同一个东西吗？**
A：不是。预言机价格是纯粹的外部现货参考价。标记价格是一个抗操纵的*合成价格*，将预言机价格、MetaFlux 订单簿和外部永续合约中间价混合计算。当永续合约跟踪现货时两者一致，当永续合约带有基差时两者出现偏差。详见[标记价格](./mark-prices.md)。

**Q：预言机运营方能移动我的清算价格吗？**
A：如果标记价格纯粹由预言机决定，则可以。这正是标记价格采用"三者取中位数"设计的原因：预言机仅是三个分量之一，单一来源被篡改会被另外两个分量压制，除非订单簿和外部永续合约同步配合。

**Q：某个市场的价格来自哪些交易所？**
A：默认使用 10 个交易所的权重表，除非治理设置了针对该交易对的覆盖配置。通过 `oracle_sources` 查询指定市场已生效的来源子集。

</details>
