# 预言机价格

:::tip
**稳定版本。**
:::

## 快速总结

**预言机价格**是协议中资产的参考价格，每个块组成一次，是**外部现货交易所的加权中位数**。它是外部锚点，[标记价格](./mark-prices.md)（其 C1 分量）和[融资利率](./funding-rates.md)（结算参考）都建立在其基础上。预言机故意是*现货衍生且推送速度缓慢的*——它不是 MetaFlux 账簿价格，也不是最后交易价格。

两个不同的数据源集为协议提供支持，它们容易混淆：

| 数据源集 | 交易所 | 聚合方式 | 驱动 |
|----------|--------|-------------|--------|
| **现货预言机** | 最多 **10 个现货交易所** | 加权中位数 | `oracle_px`；标记 C1 锚点；融资利率结算名义价值 |
| **外部合约中价** | **5 个合约交易所**（币安、OKX、Bybit、Gate、MEXC） | 中位数（≥ 2 个存在） | 标记 **C3** 分量 |

## 为什么需要预言机（而不是账簿）？

保证金、清算和融资利率都需要一个对手方**无法**通过在薄弱的 MetaFlux 账簿上进行单一交易来推动的价格。许多深度外部现货交易所的市场范围加权中位数正是这样的价格：要移动它，你必须同时在许多交易所推动现货价格，这既昂贵又会自动套利。内部账簿*确实*为标记价格贡献数据（通过 C2 和 C1-basis 条款）——但总是针对这个外部锚点进行混合。

## 构成

`oracle_px` 是该资产现有现货交易所的**加权中位数**。

### 默认现货权重表（总和 = 15）

| 交易所 | 权重 | | 交易所 | 权重 |
|-------|-------:|-|-------|-------:|
| 币安 | 3 | | Kraken | 1 |
| OKX | 2 | | KuCoin | 1 |
| Bybit | 2 | | Gate | 1 |
| Coinbase | 2 | | MEXC | 1 |
| Bitget | 1 | | MetaFlux 现货 | 1 |

使用**加权中位数**（而不是加权平均数）是为了确保单个交易所打出垃圾数据不能拖累结果——它只会改变哪个样本位于加权中点。

### 按符号治理权重

默认表是一个后备选项。仅治理 `SetOracleWeights { asset_id, weights }` 操作（`ActionId 148`）**替换**（不是合并）一个资产的表——这是必要的，因为长尾市场和无需许可（[MIP-3](../mip/mip-3.md)）市场通常不在币安/Coinbase 上市，所以默认权重会解析为不可用的内容。市场部署者**无法**设置自己的权重（选择预言机源 = 选择自己的标记）；新市场从默认表开始冷启动，只有治理可以覆盖。

已提交的每个市场源集是可查询的——参见 [`oracle_sources`](#querying)——作为场地列表的子集掩码。

## 可靠性规则

聚合器设计用于降级而非失真。每个 tick，按顺序：

- **每个数据源新鲜度。** 在 `feed_staleness_ms`（默认 **60 秒**）内未产生新打印的交易所在该 tick 中被视为不存在。
- **跨交易所异常值拒绝。** 距离跨交易所中位数超过 `feed_deviation_pct`（默认 **5 %**）的交易所在取加权中位数之前被删除——防御单个卡住/零/胖手指打印。
- **对幸存者重新规范化。** 不存在的交易所被视为权重 0，剩余权重被重新规范化。
- **最小覆盖保持。** 如果在一个 tick 中存在的总配置权重**少于 50 %**，预言机插槽**不更新**——前一个好值持续。这是硬下限，阻止一个或两个幸存交易所在市场范围内数据源中断期间定义价格。

权重设置为 0 的交易所（例如该符号被下架）根本不会被请求。

## 发布

组成的 `oracle_px` 每个块发布**一次**，从共识块时间戳导出（从不从墙上时钟），并由活跃集中的预言机验证器签署。因为中位数、新鲜度/异常值过滤器和时间戳都是共识衍生的，每个诚实验证器为该块计算**字节相同的**预言机快照。

## 与标记和融资利率的关系

- **标记。** 预言机是标记的 **C1 锚点**：`C1 = oracle + EMA(book_mid − oracle)`。没有内部账簿且没有外部合约时，标记完全降级到预言机。参见 [标记价格](./mark-prices.md)。
- **融资利率。** 融资利率是**冲击价格**（深度加权账簿价格）和**预言机**之间的差距，它**针对预言机结算**。至关重要的是，当市场的预言机陈旧或不可信时，该市场的融资利率*被阻止*并衰减至 0，而不是针对没人信任的价格结算。参见 [融资利率](./funding-rates.md#gating-when-the-oracle-is-untrusted)。

## 查询

组成的 `oracle_px` 在**整个 USDC 平面**上报告（例如 `"67042.335"`），由 [`market_info`](../api/rest/info.md#market_info) 读取，以及 `mark_px`：

```bash
curl -X POST https://devnet-gateway.mtf.exchange/info \
  -H 'content-type: application/json' \
  -d '{"type":"market_info","asset_id":0}'
```

```json
{
  "type": "market_info",
  "data": {
    "asset_id":  0,
    "mark_px":   "67042.335",
    "oracle_px": "67042.335"
  }
}
```

已提交的每个市场源集可通过 `oracle_sources` 查询（市场启用的交易所子集）：

```bash
curl -X POST https://devnet-gateway.mtf.exchange/info \
  -H 'content-type: application/json' \
  -d '{"type":"oracle_sources","asset_id":0}'
```

每个交易所的原始输入和一个 tick 中使用的确切权重存在于已提交状态中；除了源子集外，它们还没有被分解为连线字段。

## 边界情况

<details>
<summary>显示边界情况</summary>

- **一个交易所卡在陈旧价格。** 被新鲜度过滤器（> 60 秒）或异常值过滤器（> 5 % 离交易所中位数）中的任何一个删除，先触发的哪个；中位数是在幸存者上取的。
- **市场范围内中断（< 50 % 权重存在）。** 预言机插槽保持其最后好值。标记的 C1 继续使用该值，所以保证金/清算保持锚定而不是冻结或快速跳到薄弱打印。
- **长尾市场不在主流交易所。** 冷启动默认表（大多数解析为无）直到治理设置按符号 `SetOracleWeights` 覆盖指向实际列出它的交易所。
- **现货预言机健康但合约分歧。** 正常——合约可以以持续溢价/折扣交易相对于现货。预言机（现货）保持不变；标记通过 C2/C3 和 C1 basis EMA 随合约移动。参见 [标记与预言机](./mark-prices.md#mark-vs-oracle--why-they-diverge)。

</details>

## 常见问题

<details>
<summary>显示常见问题</summary>

**问：预言机与标记价格相同吗？**
答：否。预言机是纯外部现货参考。标记是一个操纵抗性*组成*，将预言机与 MetaFlux 账簿和外部合约中价混合。当合约跟踪现货时它们一致，当合约携带基差时分歧。参见 [标记价格](./mark-prices.md)。

**问：预言机运营者可以移动我的清算价格吗？**
答：纯预言机标记会让他们这样做。这正是标记是三者中位数的原因：预言机只是三个分量之一，所以操纵的数据源被否决除非账簿和外部合约随之移动。

**问：哪些交易所为给定市场定价？**
答：默认 10 交易所表，除非治理设置了按符号覆盖。查询 `oracle_sources` 以获得特定市场的已提交子集。

</details>
