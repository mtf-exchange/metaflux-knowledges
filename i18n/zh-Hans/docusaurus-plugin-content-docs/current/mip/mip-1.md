# MIP-1 — 现货代币标准 + 市场部署

:::info
**已实现。** 作为 `spotDeploy` 操作族发布；请参阅下方编号说明。
:::

MIP-1 是 MetaFlux 的原生现货代币标准，也是通过链上燃气拍卖为代币部署现货市场的机制。它是 [MIP-3](./mip-3.md) 的无权限**永续**部署的现货对应物——在已有链上平台上的类似原始物是与永续提案分开的提案，MetaFlux 也反映了这种划分。

## 为什么存在这个机制

现货上市，就像永续上市一样，是协议的一部分，而不是由策划团队决定。任何人都可以注册代币符号并通过赢得相关燃气拍卖和提供种子参数来建立现货市场——没有允许列表，没有审查委员会。

## 流程

现货部署是 `spotDeploy` 操作，由 `SpotDeployKind` 子变体分派，涵盖完整的交易对生命周期：

1. **`RegisterToken`** — 注册新的现货代币；分配 `AssetId`。
2. **`SetPair`** — 注册 `(base, quote)` 交易对（例如 `(BTC, USDC)`）；
   分配交易对的 `AssetId`。
3. **`SetFee`** — 设置每个交易对的费用等级。
4. **`ActivatePair`** — 将交易对设置为活跃（开放交易）。
5. **`DeactivatePair`** — 将交易对设置为非活跃（关闭新订单）。

赢得部署插槽通过共享燃气拍卖进行：建造者针对 `register_token_gas_auction` 流（以声明代币符号）或 `spot_pair_deploy_gas_auction` 流（以部署交易对）调用 **`submitGasAuctionBid`**。每个出价托管 USDC 金额（如果失败则退款，扣除小额费用）并携带市场规格。拍卖参数（衰减、退款窗口、插槽间隔）由治理配置，并与 MIP-3 机制共享。

## 编号说明

在当前实现中，`spotDeploy` 操作与 `perpDeploy` 操作位于同一模块中，历史上被标记为"MIP-3"。根据 [MIP 注册表](./index.md)，现货部署正确的标签是 **MIP-1**，永续部署是 **MIP-3**（反映已有平台上的现货对永续划分）。行为保持不变；只是标签被重新对齐。
