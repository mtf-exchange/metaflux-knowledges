# MIP — Market Improvement Proposals

:::info
**状态。** MIP-1 已实现 · MIP-2 进行中 · MIP-3 已实现 · MIP-4 计划中（V2） · MIP-5（Earn）计划中 · MIP-6 延期（V3）。
:::

MetaFlux 采用编号改进提案模型（类似于已建立的链上永续合约协议所使用的改进提案方案）用于影响上市市场、原生流动性或核心费用机制的协议级变更。

| MIP | 标题 | 状态 |
|-----|-------|--------|
| [MIP-1](./mip-1.md) | 现货代币标准 + 市场部署 | 已实现 |
| [MIP-2](./mip-2.md) | Metaliquidity — 协议流动性金库 | 进行中 |
| [MIP-3](./mip-3.md) | 无权限永续市场部署 | 已实现 |
| [MIP-4](./mip-4.md) | 永续流动性聚合器 / 内部化者 | 计划中（V2） |
| [MIP-5](./mip-5.md) | Earn — 现货借贷池 | 计划中 |
| [MIP-6](./mip-6.md) | Outcomes / 预测市场 | 延期（V3） |

部署提案按照已建立交易所的相同方式将现货与永续分开：**MIP-1** 是无权限现货代币 + 市场部署（`spotDeploy` 操作族），**MIP-3** 是无权限构建者部署的永续市场（`perpDeploy` 操作族 + `SetGlobal` 治理）。两者都采用相同的三个 gas 拍卖流。（当前实现仍将两个操作族捆绑在一个模块中，并将现货路径标记为 "MIP-3"；这正在重新调整 — 行为不变。）**MIP-2（Metaliquidity）** 是协议拥有的原生流动性金库。**MIP-4** 是 MetaFlux 运营的聚合器，承载零售流量，补充无权限市场。**MIP-6**（Outcomes）之前编号为 MIP-4，当 MIP-4 重新定义为聚合器时进行了重新编号。**MIP-5（Earn）** 是借贷池供应方 — 存款人从现货保证金借款人支付的利息中获得收益，重用 MIP-2 NAV/share 模型。

## V1 范围

V1 涵盖 MIP-1、MIP-2 和 MIP-3。MIP-4（聚合器）针对 V2；MIP-6（Outcomes）延期至 V3。MIP-5（Earn）已计划，将在其构建的现货保证金工作之后进行。
