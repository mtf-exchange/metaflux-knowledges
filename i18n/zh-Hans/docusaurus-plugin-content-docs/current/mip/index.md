# MIP — 市场改进提案

:::info
**状态说明。** MIP-1 已实施 · MIP-2 进行中 · MIP-3 已实施 · MIP-4 已规划（V2）· MIP-5（Earn）已规划 · MIP-6 推迟（V3）。
:::

MetaFlux 采用编号式改进提案机制（类似于其他成熟链上永续合约协议所使用的改进提案体系），用于对上架市场、原生流动性或核心手续费机制产生影响的协议级变更。

| MIP | 标题 | 状态 |
|-----|-------|--------|
| [MIP-1](./mip-1.md) | 现货代币标准 + 市场部署 | 已实施 |
| [MIP-2](./mip-2.md) | Metaliquidity — 协议流动性金库 | 进行中 |
| [MIP-3](./mip-3.md) | 无许可永续合约市场部署 | 已实施 |
| [MIP-4](./mip-4.md) | 永续合约流动性聚合器 / 内化器 | 已规划（V2）|
| [MIP-5](./mip-5.md) | Earn — 现货借贷池 | 已规划 |
| [MIP-6](./mip-6.md) | 结果市场 / 预测市场 | 推迟（V3）|

这两类部署提案以与成熟交易所相同的方式将现货与永续合约分开：**MIP-1** 是无许可现货代币 + 市场部署（`spotDeploy` 操作族），**MIP-3** 是无许可构建者主导的永续合约市场部署（`perpDeploy` 操作族 + 治理）。两者共用同一套三条 Gas 竞价流。（当前实现仍将两个操作族打包在同一模块中，并将现货路径标注为"MIP-3"；这一命名正在重新对齐，行为不变。）**MIP-2（Metaliquidity）** 是协议自有原生流动性金库。**MIP-4** 是 MetaFlux 运营的聚合器，承接散户订单流，与无许可市场互为补充。**MIP-6**（结果市场）此前编号为 MIP-4，在 MIP-4 被重新定义为聚合器后改为现有编号。**MIP-5（Earn）** 是借贷池的供给侧——存款人从现货保证金借款人支付的利息中获得收益，复用 MIP-2 的 NAV/份额模型。

## V1 范围

V1 涵盖 MIP-1、MIP-2 和 MIP-3。MIP-4（聚合器）计划在 V2 中推出；MIP-6（结果市场）推迟至 V3。MIP-5（Earn）已进入规划阶段，依赖其所基于的现货保证金工作先行完成。
