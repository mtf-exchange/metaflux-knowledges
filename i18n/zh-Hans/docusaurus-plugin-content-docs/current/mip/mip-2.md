# MIP-2 — Metaliquidity

:::info
**进行中。** 链上保险库登陆；做市策略链下运行。
:::

Metaliquidity 是 MetaFlux 协议/社区的**流动性提供者保险库** — 原生流动性主干，类似于已建立链上场所的自动化流动性提供提案。流动性提供者将 USDC 存入保险库并分享在订单簿上报价的做市策略的收益和损失。它提供 taker 交易对手的静止流动性，因此市场不会从第一天起仅依赖外部做市商。

## 链上与链下的部分

刻意的分割使共识表面保持小规模：

- **链上 — 仅保险库。** 汇总的 LP 资本、份额会计、NAV（根据预言机标记对市场）、提现锁定和认可提供者地址白名单。
- **链下 — 策略。** 做市逻辑（报价、库存管理）作为普通的 MTF-native 客户端运行：白名单化策略密钥**代表保险库账户**签署订单并通过正常的已签署订单路径提交。没有策略逻辑被烤进共识中。

## 对于流动性提供者

- **存入** USDC，无需许可，并以当前 NAV 价格（`现金 + 保险库开仓头寸的标记对市场价值`）接收保险库份额。
- **提现** 通过赎回份额以获取其 NAV 份额，受限于从最近一次存款起**7 天**的提现锁定。
- 您的份额随策略的已实现和未实现收益和损失而增长或贬值 — 存在市场风险；这不是收益保证。

## 提供者白名单

认可的 Metaliquidity 提供者地址是一份**列表**，在创世阶段初始化，可由治理修改。只有白名单地址可以运营 Metaliquidity 保险库并获权交易其汇总资本；存款保持对任何人开放。

## 状态与历史

Metaliquidity 提供协议拥有的提供者应该引导的原生订单簿流动性。HLP-等效协议保险库最初在启动后推迟（到 V2），以支持外部做市商；它被提前推进，因为原生静止流动性需要比该计划假设更早。链上保险库现在登陆；链下策略和提供者白名单种子与其一起登陆。
