# 现货保证金

:::info
**已在 Devnet 上线（预览版）。** 由 [Earn](../concepts/earn.md) 借贷资金池支持的杠杆现货交易。[普通现货](./spot.md)仅使用自有余额（无杠杆）；现货保证金是在此基础上叠加借款与杠杆的扩展模式。完整的存入 → 借款 → 杠杆买入 → 平仓流程以及自动[强制清算](#liquidation)，**目前已在 Devnet 上端到端运行**（详见下方[操作接口](#action-surface)）。请将其视为**预览版**：各交易对的**维持保证金比例仍是正在标定的治理参数**。杠杆功能在 Devnet 上可用，请勿将其视为生产环境下的安全保障。
:::

## 概述

现货保证金让你**以抵押品借入 USDC，再用杠杆买入现货资产**，而无需全额出资。借入的 USDC 来自 [Earn](../concepts/earn.md) 资金池，借款期间你需支付**利息**，同时该仓位与永续合约一样设有**维持保证金**和**清算价格**。

首版上线时，现货保证金采用**逐对隔离**模式——每个杠杆现货仓位单独存入保证金，独立清算，与永续合约的全仓账户完全分离。

## 运作机制

```
1. 为该交易对存入抵押品（USDC）—— 作为纯粹的亏损缓冲。
2. 从 Earn 资金池借入报价资产；借款 100% 用于买入。
3. 用借入的报价资产在现货订单薄上发起 IOC 买单，买入基础资产。
   买入的基础资产以隔离方式持有在保证金账户中。
4. 贷款存续期间持续支付借款利息。
5. 平仓：卖出基础资产，偿还本金及累计利息，剩余部分归还账户。
```

抵押品**不**用于买入——买入资金来自借款。抵押品是使仓位满足超额抵押要求的亏损缓冲，因此杠杆倍数 ≈ `名义价值 / 抵押品`。买入的基础资产以**隔离**方式托管在保证金账户中，与你的现货可用余额完全隔开，因此平仓（或后续清算）仅作用于该仓位本身。首版仅允许**每个 `(账户, 交易对)` 持有一个开放仓位**（不可加仓）；开仓的 IOC 单执行后会**立即偿还未动用的借款**，因此未偿贷款仅等于买单实际使用的金额。

### 操作接口

六个 [`/exchange`](../api/rest/exchange.md#spot-margin--earn) 操作（均需发送方授权）驱动完整流程。可通过 [`/info` `spot_margin_state`](../api/rest/info.md#spot_margin_state) 查询已确认的链上状态。

| 操作 | 效果 |
|---|---|
| [`spot_margin_deposit`](../api/rest/exchange.md#spot_margin_deposit) | 为该交易对存入报价资产抵押品（需先启用保证金） |
| [`spot_margin_open`](../api/rest/exchange.md#spot_margin_open) | 借款 + IOC 买入基础资产，受初始保证金要求门控 |
| [`spot_margin_close`](../api/rest/exchange.md#spot_margin_close) | IOC 卖出持有的基础资产，偿还本金及利息，返还剩余部分 |
| [`spot_margin_withdraw`](../api/rest/exchange.md#spot_margin_withdraw) | 提取空闲抵押品（空仓时可全额提取；有仓位时受初始保证金门控） |

### 保证金

```
position_value   = base_held × mark_px
debt             = borrowed + accrued_interest
equity           = position_value − debt
init_required    = position_value × spot_margin_initial_bps / 10000
maint_required   = position_value × spot_margin_maintenance_bps / 10000
health           = equity / maint_required
```

若开仓后会导致 `equity < init_required`，则开仓请求将被拒绝。当 `health < 1`（权益跌至维持保证金下限）时，仓位将被清算。

现货的**维持保证金比例是逐对参数，设置较为保守**——通常高于流动性相当的永续合约。原因在于机制本身：现货保证金清算**在现货订单薄上卖出基础资产**，因此维持保证金缓冲必须覆盖在清算阈值处平仓所产生的**滑点**，否则亏损将由借贷资金池承担。订单薄越薄（长尾资产），滑点越大，对应比例也越高。每个交易对的具体数值**依据该交易对的订单薄深度和波动率，以目标清算滑点上限为约束进行标定**——这是治理设定的风控参数，而非固定常数；在比例未完成标定前，该交易对不会开放现货保证金功能。**Devnet 上各交易对的比例仍在标定中**——尚未完成风控参数标定的交易对会拒绝所有现货保证金操作（`spot margin not enabled for pair`）。

### 利息

借入的 USDC 按逐对费率（`spot_borrow_rate_bps`，年化，每区块计提）持续产生利息。利息流入 [Earn](../concepts/earn.md) 资金池，提升其每份额价值——这即是出借方的收益。首版利率为**固定利率**；基于使用率的动态利率曲线将在后续版本中推出。

### 清算

**Devnet 上已上线。** 每个区块，链上都会按该交易对现货标记价格（订单薄最新成交价）重新估值所有保证金账户，并强制平仓权益跌破维持下限的账户：

```
liquidate when   collateral + base_held × mark − debt  <  base_held × mark × maint_bps / 10⁴
```

强制平仓走的是**与主动平仓完全相同的结算路径**——持有的基础资产在现货订单薄上以 IOC 方式卖出，Earn 资金池收回本金及利息，剩余部分（扣除小额**清算费**，该费用注入协议保险基金）归还账户，仓位随即关闭。两项防级联崩溃机制与[永续合约强制平仓](../concepts/tiered-liquidation.md#how-a-forced-close-executes-the-price-floor)的设计一致：

- **价格下限。** 强制卖单为限价单，下限为 `mark × (1 − floor)`（默认值：维持保证金比例的一半，逐对可配置）。订单薄极薄时不会被一扫而空——价格无法高于下限的部分保留持有，下一个区块重新评估。
- **部分成交保持账户开放。** 已实现收益立即偿还债务；未售出的基础资产待流动性恢复后继续尝试卖出。

爆仓损失由该仓位**自身的隔离抵押品**和 Earn 资金池承担——不会波及你的永续合约全仓账户。

**亏空处理。** 当完全平仓所得无法覆盖债务时（所得与抵押品合计不足），全部贷款本金仍从资金池借出账中核销，**亏空由 Earn 供应方共同承担**——资金池的供应总量将被相应扣减（最低归零），从而拉低份额价值。保守的逐对维持保证金比例和自动清算机制的设计目标正是让亏空情形极少发生。

## 费用

一个现货保证金仓位涉及三类不同收费：

| 费用类型 | 时机 | 费率 |
|---|---|---|
| **交易手续费** | 开仓和平仓 IOC 成交时 | 该交易对的[现货挂单/吃单费率](./spot.md#matching-fills-and-fees)（现货保证金走现货订单薄） |
| **借款利息** | 持续计提，按 USDC 未偿借款计算 | `spot_borrow_rate_bps`——逐对、年化，每区块计提；作为出借方收益流入 [Earn](../concepts/earn.md) 资金池 |
| **清算费** | 仅在强制平仓时收取 | 逐对小额费率，用于充实协议保险基金 |

开仓和平仓均为普通现货 IOC 成交，适用**现货**费率体系，不适用永续合约分级费率。借款利息是现货保证金特有的成本——它与 [Earn](../concepts/earn.md) 供应方所获收益完全一致。所有费率均为逐对治理参数，可通过 [`/info spot_margin_state`](../api/rest/info.md#spot_margin_state) 和现货 [`fee_schedule`](../api/rest/info.md#fee_schedule) 查询。

## 抵押品范围

| 版本 | 抵押品 | 清算波及范围 |
|---|---|---|
| V1 | **逐对隔离**——每个仓位单独存入 USDC | 仅该仓位 |
| 后续版本 | 支持全仓（共享账户抵押品） | 全账户 |

逐对隔离使首版风险可控：杠杆现货爆仓不会影响你的永续合约全仓余额。

## 与 Earn 的关系

现货保证金借款人是**需求方**；[Earn](../concepts/earn.md) 存款人是**供给方**。现货保证金交易者支付的借款利息，恰好是 Earn 存款人获得的收益。收益计算方式详见 [Earn](../concepts/earn.md)。

## 常见问题

<details>
<summary>展开常见问题</summary>

**Q：普通（无杠杆）现货交易会受影响吗？**
A：不会。用自有余额 100% 买入现货与以前完全一致——现货保证金是可选的叠加模式。

**Q：现货保证金的亏损会影响我的永续合约账户吗？**
A：首版不会——现货保证金采用逐对隔离模式。全仓抵押品是后续的可选升级功能。

**Q：借入的 USDC 从哪里来？**
A：来自 [Earn](../concepts/earn.md) 借贷资金池。借款上限为资金池的可用（未借出）流动性。

**Q：借款利率是多少？**
A：首版为固定的逐对年化利率，每区块计提。基于使用率的动态定价将在后续版本推出。

</details>
