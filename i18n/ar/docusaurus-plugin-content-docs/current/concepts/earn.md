# الكسب

:::info
**متاح على devnet (معاينة).** مجمع إقراض USDC يُدرّ عائداً من مقترضي [الهامش الفوري](../products/spot-margin.md). العرض، وتسعير الحصص، والاسترداد المقيّد بالسيولة الخاملة، إضافةً إلى المُصفّي التلقائي للهامش الفوري الذي يحمي المجمع — كلها تعمل من البداية إلى النهاية على **devnet اليوم** (راجع [واجهة الإجراءات](#deposit--withdraw) أدناه). اعتبره **معاينة مبكرة**: نسب الصيانة لكل زوج لا تزال قيد المعايرة.
:::

## ملخص سريع

أودع USDC في **مجمع الكسب** واجنِ عائداً. يُقرض المجمع USDC لمقترضي [الهامش الفوري](../products/spot-margin.md) الذين يدفعون فائدة؛ تتراكم هذه الفائدة في المجمع وترفع قيمة **حصصك**. **لا توجد خطوة مطالبة** — يتركّب العائد بصورة مستمرة في قيمة حصصك، وتُحقّقه عند السحب.

## آلية العمل — نموذج الحصة / صافي قيمة الأصول

عند الإيداع تحصل على **حصص** مسعّرة وفق صافي قيمة الأصول الحالية للمجمع لكل حصة (NAV). الفائدة التي يدفعها المقترضون ترفع القيمة الإجمالية للمجمع، فتصبح كل حصة تساوي تدريجياً USDC أكثر.

```
share_price        = pool_value / total_shares           # NAV per share
deposit D USDC  →  mint  D / share_price  shares
withdraw S shares → receive  S × share_price  USDC
```

- `pool_value` يبدأ مساوياً لإجمالي الودائع و**يزيد كل بلوك** مع تراكم فائدة الاقتراض فيه.
- `total_shares` يتغير فقط عند الإيداع (إصدار) والسحب (إلغاء).
- يُعيّن الإيداع الأول `share_price = 1.0` (حصة واحدة = 1 USDC).

بما أن الفائدة تُضخم `pool_value` (لا عدد الحصص)، يرتفع `share_price` بصورة متصاعدة طوال مدة أداء القروض — تزداد قيمة حصص كل مالك بالمعدل ذاته، دون سباق مطالبة ودون محاسبة لكل مستخدم على حدة.

## حساب العائد

عائدك هو ارتفاع قيمة حصصك بين الإيداع والسحب:

```
your_yield = your_shares × (share_price_now − share_price_at_deposit)
```

في كل بلوك، يزداد المجمع بالفائدة المستحقة على القروض القائمة:

```
interest_this_block = total_borrowed × borrow_rate_per_ms × Δms
pool_value         += interest_this_block
share_price         = pool_value / total_shares          # recomputed
```

### العائد السنوي الفعلي (APY)

لا يُقرض كل USDC المودع دفعةً واحدة — فقط الجزء **المُستخدَم** هو الذي يُدرّ معدل الاقتراض. لذا فإن العائد الذي يراه المودع هو معدل الاقتراض مضروباً في نسبة الاستخدام:

```
utilisation     = total_borrowed / pool_value            # 0 … 1
depositor_APY  ≈ borrow_APR × utilisation × (1 − protocol_fee)
```

| | القيمة |
|---|---|
| `borrow_APR` | معدل الاقتراض الثابت للهامش الفوري (لكل زوج) |
| `utilisation` | نسبة المجمع المُقرضة حالياً |
| `protocol_fee` | حصة البروتوكول الاختيارية من الفائدة، إن كانت مُفعّلة |

مثال: معدل اقتراض سنوي 12% عند استخدام 50% وبدون رسوم بروتوكول → عائد المودع السنوي ≈ 6%. كل العمليات الحسابية بدقة ثابتة (`Decimal`)، لا فاصلة عائمة.

## الإيداع / السحب

كلا الإجراءين مُخوَّل من المُرسِل على مسار
[`/exchange`](../api/rest/exchange.md#spot-margin--earn) العام؛ `asset` هو
**معرّف أصل الاقتباس القابل للإقراض** (مفتاح المجمع — الاقتباس لزوج فوري مسجّل)،
و`amount` / `shares` أعداد عشرية تُرسَل كسلاسل JSON. **يُنشأ المجمع تلقائياً عند أول إيداع**
لأي أصل قابل للإقراض. تحقق من الحصص المُصدَرة / المتبقية وإجماليات المجمع عبر
[`/info` `earn_state`](../api/rest/info/spot.md#earn_state).

```json
// supply 5,000 USDC into the Earn pool for asset 100
{ "type": "earn_deposit", "params": { "asset": 100, "amount": "5000" } }
```

```json
// redeem shares (receive shares × share_value), idle-bounded
{ "type": "earn_withdraw", "params": { "asset": 100, "shares": "1234.5" } }
```

| الإجراء | التأثير |
|---|---|
| [`earn_deposit`](../api/rest/exchange.md#earn_deposit) | إيداع عملة الاقتباس → حصص في المجمع (1:1 في مجمع جديد، وإلا مسعّرة وفق صافي قيمة الأصول) |
| [`earn_withdraw`](../api/rest/exchange.md#earn_withdraw) | استرداد الحصص → عملة الاقتباس، **مقيّد بالسيولة الخاملة** |

**قيد السيولة الخاملة.** السحب فوري لكنه **مقيّد بالسيولة الخاملة**
(`total_supplied − total_borrowed`): الاسترداد الأكبر من الخامل يدفع الخامل تماماً
ويحرق حصصاً أقل بالتناسب، ومجمع **ذو سيولة خاملة صفرية** (مُقرَض بالكامل) يرفض
السحب حتى يسدّد المقترضون. هذا يضمن للمورّد الخروج دائماً حتى ما لم يُقرَض، ودون
أن تبقى دفتر الاقتراض غير مضمون بكفاية.

## المخاطر

الكسب **ليس خالياً من المخاطر**. إذا أُغلق مركز [هامش فوري](../products/spot-margin.md)
بخسارة لا يستطيع ضمان المقترض تغطيتها، **يُوزَّع العجز على الموردين**:
يُخفَّض `total_supplied` للمجمع (مع حد أدنى صفر)، مما يخفض `share_value`. حماية
المجمع هي **المُصفّي التلقائي** (يعمل على devnet): في كل بلوك، تُغلَق حسابات الهامش
الغارقة [قسراً](../products/spot-margin.md#liquidation) عند حد الصيانة، فيُصفَّى المركز
في حين تكفي قيمته عادةً لسداد القرض. تُحجم هذه المخزن نسبة الصيانة المحافظة لكل
زوج (لا تزال قيد المعايرة)؛ وشلال احتياطي التأمين أمام الموردين مخطط له لكنه لم
يُنفَّذ بعد. ثمة أيضاً **مخاطر السيولة**: الاستردادات مقيّدة بالسيولة الخاملة،
لذا لا يمكن الخروج من مجمع مستخدَم بالكامل حتى يسدّد المقترضون.

**س: لماذا لا أستطيع سحب رصيدي الكامل الآن؟**
ج: الاستردادات مقيّدة بـ**السيولة الخاملة** (`supplied − borrowed`). إذا كان المجمع مُقرَضاً بالكامل لمقترضي الهامش الفوري، يمكنك السحب حتى المبلغ الخامل فقط؛ والباقي يُتاح مع سداد المقترضين.

**س: ما الفرق بين الكسب وخزينة Metaliquidity؟**
ج: الكسب هو مجمع **إقراض** USDC سلبي (العائد = فائدة الاقتراض). أما [الخزينة](./vaults.md) فهي رأس مال LP **يُتداول** (العائد/الخسارة = ربح وخسارة الاستراتيجية). ملفات مخاطر مختلفة.
