# وضع التحوط (مراكز ثنائية الاتجاه)

:::info
**مُفعَّل.** خيار التفعيل الاختياري، وتوجيه الأوامر الصريح لكل جانب، و**الهامش المستقل لكل ضلع**، و**الإبلاغ عن مراكز ضلعين مزدوجين** — كل ذلك متاح الآن: يمكن للحساب التبديل إلى وضع التحوط (في حالة رصيد مغلق)، وتوجيه كل أمر إلى ضلع محدد عبر `position_side`، ويُرسي كل ضلع هامشه الخاص ويُبلَّغ عنه ككائن مركز مستقل. **تصفية كل ضلع بشكل منفصل** مُنجزة جزئياً: عند الإشارة إلى حساب للإغلاق القسري، يُفحص كلا الضلعين ويُقيَّمان، ويُحسب ترتيب الإغلاق الحتمي (الضلع ذو الهامش الأكبر أولاً) بشكل متطابق عبر جميع المدققين — أما إصدار الإغلاق الفعلي لكل ضلع في مقابل دفتر الأوامر فلا يزال قيد الطرح التدريجي. يظل السلوك الافتراضي والموصى به هو وضع الاتجاه الواحد (مركز صافٍ واحد لكل سوق).
:::

## ملخص سريع

افتراضياً يحتفظ الحساب بـ**مركز صافٍ واحد لكل سوق** (أحادي الاتجاه): فالشراء أثناء الاحتفاظ بمركز بيع يُقلل من المركز، ثم يقلبه. أما **وضع التحوط** فيُتيح للحساب الاحتفاظ بـ**ضلع شراء وضلع بيع في السوق ذاتها في الوقت نفسه**، مع تتبع كليهما بشكل منفصل.

وضع التحوط **اختياري على مستوى الحساب** و**لا يشترط حداً أدنى للرصيد** — يمكن لأي حساب تفعيله. ولا يمكن التبديل إليه إلا عندما يكون الحساب **خالياً من المراكز في جميع الأسواق**.

## أحادي الاتجاه مقابل التحوط

| | أحادي الاتجاه (الافتراضي) | التحوط |
|---|---|---|
| المراكز لكل سوق | مركز صافٍ واحد (بإشارة) | حتى 2 (ضلع شراء + ضلع بيع) |
| "الشراء أثناء مركز البيع" | يُقلل المركز الصافي ثم يقلبه | يُقلل ضلع **البيع** فقط (أو يفتح/يمدد ضلع **الشراء** — بحسب اختيارك) |
| تحديد جانب الأمر | مستنتج من الشراء/البيع | **صريح** عبر `position_side` (`long` / `short`) إلزامياً |
| الهامش | متطلب صافٍ واحد | كل ضلع مُهمَّش باستقلالية |
| التصفية | سعر تصفية واحد | كل ضلع مُقيَّم بهامش صيانته الخاص؛ ترتيب الإغلاق الحتمي مُنجز، وإصدار الإغلاق لكل ضلع قيد الطرح التدريجي |
| الإبلاغ | كائن مركز صافٍ واحد | كائن لكل ضلع غير صفري (كل منهما مُوسوم بـ `position_side`) |

خيار التبديل، وتوجيه الأوامر لكل جانب، والهامش المستقل لكل ضلع، والإبلاغ عن الضلعين المزدوجين — كل ذلك مُفعَّل؛ أما اختيار تصفية كل ضلع فمُنجز، وإصدار الإغلاق لكل ضلع لا يزال قيد الطرح التدريجي (راجع ملاحظة الحالة أعلاه).

## التفعيل

التبديل في وضع المركز إجراء موقَّع؛ ولا يكون قانونياً إلا عندما يكون الحساب **خالياً من المراكز في جميع الأسواق**.

```json
// تفعيل وضع التحوط (قانوني فقط عند عدم وجود مراكز في جميع الأسواق)
{ "type": "set_position_mode", "params": { "hedge": true } }
```

```json
// العودة إلى أحادي الاتجاه (أيضاً فقط عند عدم وجود مراكز)
{ "type": "set_position_mode", "params": { "hedge": false } }
```

`hedge` قيمة منطقية: `true` = تحوط (ثنائي الاتجاه)، `false` = أحادي الاتجاه (الافتراضي).
شرط **خلو جميع الأسواق من المراكز** قاعدة أمان — يُرفض التبديل أثناء الاحتفاظ بأي مركز مفتوح (عملية لا تُنفَّذ ولا تُغيِّر شيئاً)، وذلك حتى لا يُعاد تفسير المركز الصافي خفيةً كضلع معلَّق. أما ضبط الوضع على قيمته الحالية مع خلو الحساب من المراكز فعملية نجاح لا أثر لها.

راجع [`set_position_mode`](../api/rest/exchange.md#set_position_mode) في مرجع `/exchange` للاطلاع على تفاصيل الطلب والاستجابة.

## تقديم الأوامر في وضع التحوط

في وضع التحوط، يجب أن يحمل كل أمر **`position_side` صريحاً** (`long` / `short`). يجب أن **لا** يُرسل حساب أحادي الاتجاه `position_side`؛ بينما يجب على حساب التحوط إرساله. الحقل موجود في جسم أمر `submit_order` جانباً `side` و`reduce_only` وغيرهما.

```json
{
  "type": "submit_order",
  "order": {
    "owner": "0x...aa", "market": 0, "side": "bid",
    "kind": "limit", "size": 100000000, "limit_px": 5000000000,
    "tif": "gtc", "stp_mode": "cancel_oldest",
    "reduce_only": false,
    "position_side": "long"
  }
}
```

| النية | `side` | `position_side` | `reduce_only` |
|---|---|---|---|
| فتح ضلع شراء / إضافة إليه | `bid` | `long` | false |
| تقليص ضلع الشراء / إغلاقه | `ask` | `long` | true |
| فتح ضلع بيع / إضافة إليه | `ask` | `short` | false |
| تقليص ضلع البيع / إغلاقه | `bid` | `short` | true |

`position_side` **صريح وغير مستنتج أبداً** — وإلا قد يُنشئ أمر شراء مُعدّ لـ*تقليص مركز بيع* أو يُوسّع ضلع شراء عن طريق الخطأ. يُقيَّم `reduce_only` **في مقابل الضلع المُسمَّى فقط**: لا يستطيع أمر `reduce_only` على ضلع `short` لمس ضلع `long` بأي حال.

لا يوجد **"قلب"** في وضع التحوط. إغلاق ضلع الشراء لا يفتح مركز بيع أبداً — ذلك أمر منفصل على ضلع البيع.

## الهامش

يُهمَّش كل ضلع **باستقلالية تامة** — يُرسي ضلع الشراء وضلع البيع كل منهما هامش الدخول وهامش الصيانة الخاص به، تُجمع في متطلب الحساب الإجمالي:

```
required_margin = init_margin(long_leg) + init_margin(short_leg)
```

هذا النهج متحفظ بقصد: فمركز الشراء والبيع في سوق واحدة محايدٌ في الدلتا من حيث السعر، لكن كل ضلع يشغل هامشاً. (قد يُتيح تحديث مستقبلي خصماً لتعويض الأضلاع المتقابلة في إطار [هامش المحفظة](./portfolio-margin.md)؛ وحتى ذلك الحين، لا تحتفظ بكلا الضلعين إلا إذا أردت تعريضات منفصلة فعلاً، كأسعار دخول مختلفة تعتزم إدارتها باستقلالية.)

يحتفظ كل ضلع بـ[وضع الهامش](./margin-modes.md) الخاص به — يمكنك مثلاً تشغيل ضلع الشراء بهامش معزول وضلع البيع بهامش متقاطع.

## التصفية

عند الإشارة إلى حساب للإغلاق القسري، يُفحص كلا الضلعين ويُقيَّمان بحسب مساهمة هامش الصيانة الخاص بكل منهما، ويُحسب ترتيب الإغلاق الحتمي — **الضلع ذو الهامش الأكبر أولاً** (عند التساوي: الشراء قبل البيع)، متطابقاً عبر جميع المدققين — عبر سُلَّم [التصفية المتدرجة](./tiered-liquidation.md) القياسي. أما إصدار الإغلاق الفعلي لكل ضلع في مقابل دفتر الأوامر فلا يزال قيد الطرح التدريجي؛ وحين يُطرح، لن تمس تصفية أحد الضلعين الضلعَ الآخر.

## الإبلاغ

عند تفعيل وضع التحوط، يُعيد حالة الحساب وقراءات مراكز `/info` **كائن مركز واحداً لكل ضلع غير صفري** في سوق يحمل كلا الضلعين، مُوسَماً كل منهما بـ`position_side` (`"long"` / `"short"`). أما حساب أحادي الاتجاه فيُعيد مركز صافياً واحداً **بلا** حقل `position_side`، تماماً كما هو الحال اليوم. يظل الاهتمام المفتوح على مستوى السوق رقماً صافياً واحداً. راجع [مراكز `account_state`](../api/rest/info.md#account_state).
