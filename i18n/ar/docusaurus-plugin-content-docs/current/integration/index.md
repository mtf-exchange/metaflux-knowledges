## حزم SDK

| اللغة | الحالة | الحزمة |
|----------|--------|---------|
| TypeScript / JavaScript | معاينة | [`@metaflux/sdk`](./typescript-sdk.md) |
| Rust | معاينة | [`metaflux-client`](./rust-sdk.md) |

بالنسبة للغات الأخرى (Python، Go، Java، C++ …)، نفّذ توقيع EIP-712 المكتوب وفقًا لـ [توقيع البيانات المكتوبة](./typed-data-signing.md) — كل خطوة موثقة بأمثلة مفصّلة. البروتوكول بسيط بما يكفي لجعل العميل المكتوب يدويًا الخيار الصحيح للبيئات المتخصصة.

## نقاط نهاية الشبكة

راجع [الشبكات](../networks.md) للمرجع الكامل لكل شبكة.

البوابة (`https://<net>-gateway.mtf.exchange`) هي نقطة الدخول العامة الوحيدة.

| المسار | يخدم | الغرض |
|------|--------|---------|
| `POST /info` · `POST /exchange` · `GET /ws` | MTF-native (افتراضي) | الواجهة الأصلية snake_case |
| `POST /hl/info` · `POST /hl/exchange` · `GET /hl/ws` | HL-compat | شكل بروتوكول HL |
| `/ccxt/*` | CCXT-compat | طرق CCXT REST |
| `POST /evm` | EVM JSON-RPC | RPC سلسلة EVM الجانبية |
| `POST /faucet` | Faucet | صنبور اختبار devnet/testnet |

تنتهي TLS في عمليات النشر الإنتاجية عند البوابة ويقف أمامها CDN؛ العقدة عمدًا غير مكشوفة على الإنترنت — إذ تعمل خلف البوابة. عند تشغيل العقدة بنفسك، تُقدَّم الواجهة الأصلية مباشرةً على `http://localhost:8080` (مع EVM RPC الخام على `http://localhost:8545`).

## الأنماط الشائعة

- **بوت صانع السوق** — موقّع بالوكيل، تسعير مستمر، سايدكار مراقب المخاطر، أوامر ALO لضمان مستوى صانع السوق
- **مراقب التصفية** — مشترك WS في `marginEvents` + `userEvents` (`yellowCard`)؛ يُطلق عمليات التجديد قبل T1
- **غلاف TWAP** — يرسل `TwapOrder`، يراقب `twapEvents` لتتبع الشرائح، مع إمكانية إلغاء يدوي أثناء التشغيل
- **مدير الخزينة** — `VaultDeploy` مرة واحدة، ثم أوامر موقّعة بالوكيل لعنوان الخزينة عند إعادة التوازن
- **حضانة مؤسسية** — مفتاح رئيسي متعدد التوقيعات + وكلاء لكل مضيف + تغليف متعدد التوقيعات للتدفقات عالية القيمة
