# مرجع واجهة برمجة التطبيقات (API)

ثلاث عائلات بروتوكول، تخدمها جميعاً نفس البوابة الموحدة
(`https://<net>-gateway.mtf.exchange`) — والاختيار بين أشكال الاتصال ليس سوى
اختيار المسار.

| العائلة | الموقع | متى تستخدمها |
|--------|-------|----------|
| **MTF-native** | المسار **الافتراضي** للبوابة: `POST /exchange`، `POST /info`، `GET /ws`، `POST /faucet` | للعملاء الجدد. شكل بيانات مدمج بصيغة snake_case. يتيح الوصول إلى كل الميزات، بما فيها ميزات التمييز الخاصة بـ MTF (RFQ، FBA، تسجيل PM، التكامل متعدد السلاسل). |
| **HL-compat** | البوابة تحت `/hl/*`: `POST /hl/exchange`، `POST /hl/info`، `GET /hl/ws` | لنقل عميل HL موجود دون تغييرات. تتطابق أشكال JSON مع HL تماماً. صفر تغييرات في الكود لـ `order` و`cancel` (المزيد من المتغيرات قيد الإطلاق تدريجياً). |
| **CCXT-compat** | البوابة تحت `/ccxt/*` | لأطر العمل الكمية التي تعمل بـ CCXT بالفعل. مجموعة REST أساسية متاحة؛ CCXT Pro WS قادم قريباً. |

> البوابة هي نقطة الدخول الموحدة — MTF-native هو المسار الافتراضي
> (`/info`، `/exchange`)، وHL-compat منظّم تحت `/hl/*`، وCCXT تحت
> `/ccxt/*`. هل تشغّل العقدة بنفسك؟ تُقدّم نفس السطح الأصلي
> مباشرةً على `http://localhost:8080`.
