# `POST /faucet` — أموال الاختبار على devnet/testnet

:::warning
**متاح على Devnet / testnet فقط.** يقوم الصنبور بسكّ ضمانات مجانية ورموز spot من لا شيء. وهو **مرفوض هيكليًّا على الشبكة الرئيسية** (معرّف السلسلة `8964`): لا يُثبَّت المسار هناك أصلًا. لا تعتمد عليه أبدًا في بيئة الإنتاج.
:::

## ملخّص سريع

طلب `POST /faucet` واحد يمنح **3000 USDC** ضمانًا متقاطعًا **و10 MTF** رموز spot (معرّف الرمز `104`) لأي عنوان. **مرة واحدة فقط لكل عنوان.** الاستجابة هي `"queued"` — تُضاف الأرصدة بعد نحو بلوك واحد (تُحقن عبر إجراءات نظام المدقّق، وليس بشكل متزامن). يُخدَّم بوصفه `POST /faucet` على البوابة الرئيسية، جنبًا إلى جنب مع المسارَين الافتراضيَّين `/info` و`/exchange`.

## عنوان URL

```
POST  https://<net>-gateway.mtf.exchange/faucet
```

عند تشغيل العقدة محليًّا، يُخدَّم المسار `/faucet` ذاته مباشرةً على:
`http://localhost:8080`.

| البيئة | متاح؟ |
|--------|--------|
| Devnet (`31337`) / testnet (`114514`)، الصنبور مُفعَّل | نعم |
| الشبكة الرئيسية (`8964`) | **لا** — المسار غير مثبَّت أصلًا؛ أي طلب خاطئ يُعيد `403` من معالج الحماية |
| الصنبور معطَّل في إعدادات العقدة | لا |

يُدمج المسار في الموجّه الرئيسي للـ API فقط حين يكون إعداد الصنبور في العقدة **مُفعَّلًا وليس على الشبكة الرئيسية**. يحمل المسار حالة معالجه الخاصة وهو غير قابل للوصول هيكليًّا من شجرة معالجات `/exchange`.

## الطلب

```json
{ "address": "0x00000000000000000000000000000000000ca11e" }
```

| الحقل | النوع | مطلوب | الوصف |
|-------|-------|--------|-------|
| `address` | عنوان hex بصيغة `0x` بطول 20 بايت | نعم | العنوان المستقبِل. يقبل 40 أو 42 حرفًا (البادئة `0x` اختيارية). يُرفض العنوان الصفري. |
| `amount` | uint64 (USDC بالوحدات الصحيحة) | لا | منحة USDC اختيارية؛ **تُخفَّض** إلى الحد الأقصى المُعيَّن (3000) — أي قيمة أعلى تُقيَّد إلى 3000 ولا تتجاوزه أبدًا. القيمة `0` مرفوضة. كمية MTF (10) ثابتة بصرف النظر عن هذا الحقل. |

```bash
curl -s -X POST https://devnet-gateway.mtf.exchange/faucet \
  -H 'content-type: application/json' \
  -d '{"address":"0x00000000000000000000000000000000000ca11e"}'
```

## الاستجابة

### `200 OK` — في قائمة الانتظار

```json
{
  "address": "0x00000000000000000000000000000000000ca11e",
  "usdc":    3000,
  "mtf":     10,
  "status":  "queued"
}
```

| الحقل | النوع | الوصف |
|-------|-------|-------|
| `address` | سلسلة hex بصيغة `0x` | العنوان المستقبِل مُعادًا تطبيعه بأحرف صغيرة |
| `usdc` | uint64 | كمية USDC الممنوحة (بالوحدات الصحيحة، بعد أي تخفيض) |
| `mtf` | uint64 | رموز MTF spot الممنوحة (بالوحدات الصحيحة، ثابتة عند 10) |
| `status` | `"queued"` | الرصيد **مُجدوَل للبلوك التالي**، لم يُثبَّت بعد |

القيمة `"queued"` حرفية: المنحة عبارة عن إجراءَي نظام يحقنهما المدقّق
(`SystemUserModify{AdjustCrossAccountValue}` لـ USDC + `SystemSpotSend` لـ MTF)
تُضافان في مقدمة البلوك المقترح التالي. استعلم عن [`account_state`](./info.md#account_state)
(أو [`spot_clearinghouse_state`](./info.md#spot_clearinghouse_state)) بعد نحو بلوك واحد
للتحقق من الرصيد:

```json
// account_state after the credit commits:
{ "account_value": "3000", "balances": { "usdc": "3000", "spot": { "MTF": "10" } }, ... }
```

### الأخطاء

| HTTP | جسم الاستجابة | السبب |
|------|--------------|-------|
| 400 | `{"error":"invalid address: <detail>"}` | `address` ليس بصيغة hex `0x` صالحة (مثلًا طول خاطئ) |
| 400 | `{"error":"zero address not allowed"}` | العنوان المستقبِل هو العنوان الصفري |
| 400 | `{"error":"amount must be positive"}` | قيمة `amount` الصريحة هي `0` |
| 429 | `{"error":"address already funded"}` | هذا العنوان طالب بالمنحة من قبل (**مرة واحدة فقط**، دائمة طوال عمر العقدة) |
| 429 | `{"error":"rate limit: this IP requested too recently"}` | عنوان IP المصدر طلب ضمن فترة التهدئة لكل IP (الافتراضي: مرة/دقيقة/IP) |
| 403 | `{"error":"faucet disabled on this network"}` | معالج الحماية الدفاعي (لا ينبغي الوصول إليه — الشبكة الرئيسية لا تُثبّت المسار أصلًا) |
| 503 | `{"error":"faucet backlog full; retry shortly"}` | قائمة الحقن مشبعة (ضغط مؤقت؛ أعد المحاولة) |

```json
// second claim for the same address:
{ "error": "address already funded" }   // HTTP 429
```

## القيود

- **مرة واحدة فقط لكل عنوان.** يُتتبَّع في مجموعة داخل الذاكرة (تُصفَّر عند إعادة تشغيل العقدة؛ devnet مؤقتة). أي طلب ثانٍ لنفس العنوان — حتى من IP مختلف أو في وقت متأخر — يُعيد `429 address already funded`. الطلبات *المرفوضة* لا تستهلك فرصة المرة الواحدة.
- **تقييد لكل IP.** الافتراضي: طلب واحد / دقيقة / IP المصدر. العناوين المختلفة من نفس IP ضمن النافذة الزمنية تحصل على `429 rate limit`.
- **حد USDC.** القيمة الاختيارية `amount` تُخفَّض فقط للأسفل؛ لا يمكن الحصول على أكثر من 3000 USDC المُعيَّنة.

## لماذا هذا المسار ليس على `/exchange`

رصيدا الصنبور هما **إجراءات نظام / مميّزة**
(`SystemUserModify`، `SystemSpotSend`) — سكّ ضمانات ورموز spot من لا شيء. هذه الإجراءات تقع ضمن نطاق معرّفات إجراءات النظام وهي **ليست** ضمن القائمة المسموح بها لإجراءات المستخدم على `/exchange`. يُدرج الصنبور هذه الإجراءات في **قائمة حقن مخصصة للمدقّق فقط** (وليس في mempool العام)؛ يُفرّغ وقت التشغيل هذه القائمة في حمولة البلوك تمامًا كتغذية Oracle، مع عنوان المدقّق الخاص بالعقدة بوصفه مُرسِلًا كي يجتاز التحقق `require_system_authority`.
لا يوجد مسار كودي من mempool المستخدم العام إلى هذه القائمة. انظر
[never expose system actions on /exchange](./exchange.md#non-bridged-actions).

## حدود الحتمية

كل ما في طرف HTTP للصنبور غير حتمي (تقييد IP بحسب الساعة الجدارية، مجموعة العناوين المُطالَب بها محليًّا). القيم الوحيدة التي تنتقل إلى الإجماع هي العنوان المستقبِل والكميات في إجراءَي النظام، اللذان يمران عبر المعالجات الحتمية دون تغيير. حالة تقييد المعدل والمجموعة المحلية للعناوين لا تُدرج أبدًا في AppHash.
