# `POST /info` — استعلامات العقود الدائمة

استعلامات القراءة لأسواق **العقود الدائمة**. نفس نقطة نهاية `POST /info` والغلاف والاصطلاحات المستخدمة في [الصفحة الرئيسية](../info.md) — هذه هي أنواع `type` الخاصة بأسواق العقود الدائمة. (قراءات سجل الأوامر / الصفقات / الشمعات تخدم أيضاً أزواج السبوت عبر معرّف `pair`.)

## أنواع استعلامات العقود الدائمة

### `market_info`

البيانات الوصفية لكل سوق على حدة.

```json
{ "type": "market_info", "asset_id": 0 }
```

أو بالاسم:

```json
{ "type": "market_info", "coin": "BTC" }
```

الاستجابة:

```json
{
  "type": "market_info",
  "data": {
    "asset_id":        0,
    "name":            "BTC",
    "kind":            "perp",
    "sz_decimals":     5,
    "mark_px":         "67079.265",
    "oracle_px":       "67073.35",
    "mid_px":          "67079.27",
    "premium":         "0.0015",
    "tick_size":       "1000000",
    "step_size":       "1",
    "min_order":       "1",
    "max_leverage":    50,
    "maint_margin_ratio": "300",
    "init_margin_ratio":  "200",
    "funding": {
      "rate_per_hr":  "0",
      "cap_per_hr":   "400",
      "interval_ms":     3600000,
      "next_payment_ts": 0
    },
    "mark_source": "MedianOfOraclesAndMid",
    "fba_enabled": false,
    "open_interest": "0"
  }
}
```

:::info
**مستوى الإبلاغ عن الأسعار.** في هذه القراءة، يكون كلٌّ من `mark_px` و`oracle_px` في **مستوى عشري USDC كامل** (الدولار البشري — `"67079.265"` / `"67073.35"`)، وهو نفس الوحدة المستخدمة في وسم مراكز الحسابات. `mark_px` هو سعر الوسم المدرج في الكتاب، مُقلَّصاً من التمثيل الداخلي للمحرك بالفاصلة العائمة الثابتة 1e8، مع الرجوع إلى سعر الأوراكل عند غياب الوسم من الكتاب؛ أما `oracle_px` فهو آخر سعر مؤشر مُلتزَم به. أيٌّ منهما يكون `"0"` عند عدم التعيين. لاحظ أن **مستوى تقديم الأوامر/الكتاب يبقى في النقطة العائمة الثابتة 1e8** — أسعار مستويات `l2_book` وأسعار `limit_px` للأوامر ليست بوحدات USDC الكاملة؛ تحتفظ MTF بهذين المستويين منفصلَين، ولا تُبلَّغ الأسعار بوحدات USDC الكاملة إلا في القراءات الموجَّهة للمستخدم (`market_info`، و`markets`، والمراكز). دلالات الحقول لبقية السجل موجودة في جدول [`markets`](#markets) أدناه.
:::

:::info
**دقة السعر مقارنةً بـ`sz_decimals`.** يُقرَّب `mark_px` و`oracle_px` إلى **أقرب نقطة سعر في السوق** (`tick_size`، بالبتر نحو الصفر)، بحيث لا تظهر القراءة أي ضوضاء دون مستوى النقطة — عند نقطة سعر `$0.01` (`tick_size: "1000000"` في المستوى 1e8) يُبلَّغ `66735.255` بوصفه `"66735.25"`. لاحظ أن `sz_decimals` يمثل دقة **الحجم** (دقة كمية الأوامر — `5` ⇒ `0.00001` وحدة)، ولا **يتحكم** في خانات السعر العشرية؛ نقطة السعر هي التي تتحكم في ذلك. وكلاهما محوران مستقلان (نفس التقسيم الذي يستخدمه HL).
:::

### `markets`

كل أسواق العقود الدائمة MIP-3 المسجَّلة، في استدعاء واحد. لا معاملات.

```json
{ "type": "markets" }
```

حمولة `data` هي **مصفوفة** من نفس سجلات السوق الغنية التي يُرجعها [`market_info`](#market_info) لأصل واحد. تُرتَّب السجلات بصورة محددة تصاعدياً حسب `asset_id` (تكرار العقدة على `mip3_market_specs` `BTreeMap`). يُرجع الكون الفارغ `"data": []`.

الاستجابة:

```json
{
  "type": "markets",
  "data": [
    {
      "asset_id":        0,
      "name":            "BTC",
      "kind":            "perp",
      "sz_decimals":     5,
      "mark_px":         "67042.335",
      "oracle_px":       "67042.335",
      "mid_px":          "67042.33",
      "premium":         "0.0015",
      "tick_size":       "1000000",
      "step_size":       "1",
      "min_order":       "1",
      "max_leverage":    50,
      "maint_margin_ratio": "300",
      "init_margin_ratio":  "200",
      "funding": {
        "rate_per_hr":  "0",
        "cap_per_hr":   "400",
        "interval_ms":     3600000,
        "next_payment_ts": 0
      },
      "mark_source": "MedianOfOraclesAndMid",
      "fba_enabled": false,
      "open_interest": "0"
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `asset_id` | uint32 | معرّف الأصل الأساسي (مفتاح الترتيب) |
| `name` | string | رمز السوق، مثل `"BTC"` |
| `kind` | `"perp"` | نوع السوق (بأحرف صغيرة) |
| `sz_decimals` | uint8 | خانات العرض العشرية للحجم (من سجل رموز السبوت الأساسية؛ `0` إذا لم تتوفر مواصفات رمز) |
| `mark_px` | Decimal string | وسم الكتاب، **مستوى USDC الكامل** (وسم الكتاب مُقلَّص من 1e8، مع الرجوع للأوراكل؛ `"0"` إذا لم يُعيَّن) |
| `oracle_px` | Decimal string | سعر المؤشر، **مستوى USDC الكامل** (`"0"` إذا لم يُعيَّن) |
| `mid_px` | Decimal string \| null | منتصف سجل الأوامر الفعلي `(best_bid + best_ask) / 2`، **مستوى USDC الكامل** (مُقرَّب للنقطة)؛ `null` عندما يكون الكتاب ذا اتجاه واحد / فارغاً |
| `premium` | Decimal string \| null | آخر عيّنة علاوة تمويل مُلتزَم بها (موقَّعة)؛ `null` عند غياب أي عيّنة |
| `tick_size` | i128 string | الحد الأدنى لزيادة السعر، **نقطة عائمة ثابتة 1e8** (مستوى تقديم الأوامر/الكتاب) |
| `step_size` | u128 string | الحد الأدنى لزيادة الحجم (حجم الدُفعة)، نقطة عائمة ثابتة |
| `min_order` | u128 string | الحد الأدنى لحجم الأمر |
| `max_leverage` | uint8 | الحد الأقصى للرافعة المالية |
| `maint_margin_ratio` | bps string | نسبة هامش الصيانة، نقاط أساس عشرية |
| `init_margin_ratio` | bps string | نسبة الهامش الأولي (`1 / max_leverage`)، نقاط أساس عشرية |
| `funding.rate_per_hr` | bps string | آخر عيّنة علاوة تمويل، نقاط أساس عشرية |
| `funding.cap_per_hr` | bps string | سقف معدل التمويل في الساعة، نقاط أساس عشرية |
| `funding.interval_ms` | uint64 | دورية التمويل (ساعة واحدة = `3600000`) |
| `funding.next_payment_ts` | uint64 | طابع زمني لدفعة التمويل التالية (`0` حتى توجد عيّنة) |
| `mark_source` | string | واصف سعر الوسم (`"MedianOfOraclesAndMid"`) |
| `fba_enabled` | bool | تفعيل المزاد الدُفعي المتكرر لهذا السوق |
| `open_interest` | u128 string | المركز المفتوح الحالي، نقطة عائمة ثابتة |

كل عنصر مطابق بالبايت للحقل `data` في استجابة `market_info` للأصل الواحد المقابل — كلاهما مبنيٌّ من نفس منشئ سجلات السوق، لذا لا يتباعد شكل الاستجابة الفردية والجماعية أبداً. راجع [`market_info`](#market_info) لدلالات الحقول والملاحظات المتعلقة بوكيل FLAGGED (`mark_source`، `next_payment_ts`).

### `l2_book`

مستويات العروض والطلبات المجمَّعة في نطاق السوق.

```json
{ "type": "l2_book", "market_id": 0 }
```

| Arg | Type | Required |
|-----|------|----------|
| `market_id` | uint32 | نعم |

الاستجابة:

```json
{
  "type": "l2_book",
  "data": {
    "market_id": 0,
    "bids": [ { "px": "99000", "size": "700", "n_orders": 1 } ],
    "asks": [ { "px": "101000", "size": "750", "n_orders": 2 } ]
  }
}
```

تُرتَّب عروض الشراء من الأفضل أولاً (تنازلياً حسب السعر)، وعروض البيع تصاعدياً. كل مستوى يجمع `size` الإجمالية وعدد `n_orders` للأوامر الراسية. يُرجع السوق غير المعروف / الفارغ مصفوفتَي `bids` / `asks` فارغتَين.

| Field | Type | Description |
|-------|------|-------------|
| `market_id` | uint32 | معرّف السوق المُعاد |
| `bids[*].px` / `asks[*].px` | i128 string | سعر المستوى، سلسلة عشرية بنقطة عائمة ثابتة |
| `bids[*].size` / `asks[*].size` | u128 string | الحجم الإجمالي عند المستوى |
| `bids[*].n_orders` / `asks[*].n_orders` | uint64 | الأوامر الراسية عند المستوى |

### `recent_trades`

شريط الصفقات العام في نطاق السوق، يُقدَّم مباشرةً من الحالة المُلتزَم بها في العقدة (حلقة صفقات محدودة لكل سوق مطوية في AppHash — لا فهرسة خارجية).

```json
{ "type": "recent_trades", "market_id": 0 }
```

| Arg | Type | Required | Description |
|-----|------|----------|-------------|
| `market_id` | uint32 | نعم | معرّف الأصل / السوق |
| `limit` | uint32 | لا | تحديد عدد **أحدث** السجلات المُرجَعة؛ غائب / `0` ⇒ الحلقة الكاملة |

الاستجابة:

```json
{
  "type": "recent_trades",
  "data": {
    "market_id":      0,
    "last_trade_ms":  1700000000555,
    "trades": [
      {
        "coin":  0,
        "side":  "B",
        "px":    "67042.50",
        "sz":    "0.125",
        "time":  1700000000555,
        "tid":   90123,
        "block": 562,
        "hash":  "0x2315b79b9e82c2deb279a59448bf7841f3767d30d874e5b544d75bb9fd1e9b0c"
      }
    ]
  }
}
```

تُرتَّب السجلات من الأقدم أولاً (الأحدث أخيراً). الحلقة محدودة، لذا هذه نافذة حديثة وليست كل السجل. يُرجع السوق غير المعروف / الذي لم يُتداوَل فيه `"trades": []` و`last_trade_ms: 0`.

| Field | Type | Description |
|-------|------|-------------|
| `market_id` | uint32 | معرّف السوق المُعاد |
| `last_trade_ms` | uint64 | طابع زمني لآخر صفقة (`0` إذا لم توجد) |
| `trades[*].coin` | uint32 | معرّف الأصل / السوق الذي نُفِّذت فيه الصفقة |
| `trades[*].side` | `"B"` / `"A"` | رمز جانب الطرف المُهاجِم (المتقدم) — `"B"` = شراء، `"A"` = بيع |
| `trades[*].px` | Decimal string | سعر التنفيذ، **USDC عشري** (قابل للقراءة البشرية) |
| `trades[*].sz` | Decimal string | الحجم المُنفَّذ، **وحدات القاعدة** (وحدات كاملة) |
| `trades[*].time` | uint64 | الطابع الزمني للصفقة (بالمللي ثانية حسب التوافق) |
| `trades[*].tid` | uint64 | معرّف صفقة حتمي (مشترك بين كلا طرفَي الطباعة) |
| `trades[*].block` | uint64 | ارتفاع الكتلة المُلتزَم التي تسوَّت فيها الصفقة (محدد موقع على السلسلة) |
| `trades[*].hash` | hex string | تجزئة معاملة الأمر الأصلي، سداسي عشري بادئته `0x` — تتيح تتبع الطباعة على السلسلة |

### `candle`

أشرطة OHLCV التاريخية لـ`(coin, interval)` على نافذة زمنية. المرافق REST لقناة WS الحية [`candles`](../../ws/subscriptions.md#candles) — تدفع قناة WS الشريط الجاري تشكُّله مع تراكم الصفقات، بينما تُرجع هذه القراءة السجل المغلق.

```json
{ "type": "candle", "coin": "BTC", "interval": "1m" }
```

| Arg | Type | Required | Description |
|-----|------|----------|-------------|
| `coin` | string | نعم | رمز السوق، مثل `"BTC"` |
| `interval` | string | نعم | رمز الدلو — أحد القيم: `1m`, `5m`, `15m`, `1h`, `4h`, `1d` |
| `start_time` | uint64 | لا | بداية النافذة (بالمللي ثانية)؛ يُصفِّي حسب وقت فتح الشريط. الافتراضي `0` |
| `end_time` | uint64 | لا | نهاية النافذة (بالمللي ثانية)؛ يُصفِّي حسب وقت فتح الشريط. الافتراضي غير محدود |

يمكن تمرير المعاملات مسطَّحةً (كما أعلاه) أو متداخلةً تحت كائن `req`؛ يقبل `start_time` / `end_time` أيضاً الصياغة camelCase `startTime` / `endTime`. غياب `coin` أو `interval` ← `400 {"error":"missing field <name>"}`.

الاستجابة:

```json
{
  "type": "candle",
  "data": [
    {
      "t": 1700000040000,
      "T": 1700000099999,
      "s": "BTC",
      "i": "1m",
      "o": "67000.00",
      "c": "67042.50",
      "h": "67080.00",
      "l": "66990.00",
      "v": "12.5",
      "q": "837843.75",
      "n": 37
    }
  ]
}
```

تُرتَّب الأشرطة من الأقدم أولاً حسب `t` (وقت الفتح)؛ العنصر الأحدث هو الشريط الجاري تشكُّله. المصفوفة الفارغة هي الإجابة الصادقة لرمز `interval` غير مدعوم، أو سوق بلا صفقات مُفهرَسة، أو نشر بلا فهرسة متصلة.

| Field | Type | Description |
|-------|------|-------------|
| `t` | uint64 | الطابع الزمني لـ**فتح** الشريط (بالمللي ثانية، محاذٍ للدلو) |
| `T` | uint64 | الطابع الزمني لـ**إغلاق** الشريط (بالمللي ثانية) — `t + interval − 1` |
| `s` | string | رمز العملة / السوق |
| `i` | string | رمز دلو الفاصل الزمني |
| `o` / `c` / `h` / `l` | Decimal string | سعر الـ**فتح** / الـ**إغلاق** / الـ**أعلى** / الـ**أدنى**، **USDC عشري** (دولار بشري، مثل `"67042.50"`) |
| `v` | Decimal string | **حجم الأصل القاعدي** — مجموع الحجم المُتداوَل في الشريط (حجم العملة، ليس القيمة الاسمية) |
| `q` | Decimal string | **حجم التداول بالدولار (عرض الأسعار)** — `Σ السعر × الحجم` على صفقات الشريط |
| `n` | uint64 | عدد الصفقات (التعبئات) في الشريط |

:::info
**السلسلة بلا ثغرات.** الفاصل الزمني الذي **لا توجد فيه صفقات** لا يزال يُصدر شريطاً ثابتاً يحمل إغلاق الشريط السابق إلى الأمام: `o = h = l = c = الإغلاق السابق`، و`v = q = 0`، `n = 0`. يحصل المستهلكون على سلسلة شرائط متواصلة — شريط لكل فاصل زمني — دون ثغرات تحتاج إلى استيفاء. **لا يُصدَر أي شريط قبل أول صفقة في السوق** — تبدأ السلسلة من دلو الطباعة الأولى، لذا المصفوفة الفارغة تعني أن السوق لم يُتداوَل فيه قط (أو لا يوجد سجل متصل)، وليس أن الدلاء المبكرة سقطت.
:::

:::info
**هذا النوع تُقدِّمه البوابة، لا العقدة.** الشمعات هي بيانات عرض مشتقة مطوية من دفق الصفقات العام — إنها **ليست** حالة سلسلة مُلتزَماً بها، ولا تلمس app-hash، ولا تحمل أي ضمان توافق. تجيب البوابة على `candle` من مخزنها المتجدد الخاص؛ عقدة مُستعلَمة مباشرةً تُرجع `unknown info type: candle`. إجابة فارغة صادقة (`"data": []`) عندما لا يوجد لدى البوابة سجل صفقات للسوق بعد.
:::

### `funding_history`

عينات علاوة التمويل على مستوى السوق.

```json
{ "type": "funding_history", "market_id": 0 }
```

| Arg | Type | Required |
|-----|------|----------|
| `market_id` | uint32 | yes |

الاستجابة:

```json
{
  "type": "funding_history",
  "data": {
    "market_id": 0,
    "samples": [
      { "ts_ms": 1700000000000, "premium": "0.0015", "funding_rate": "0.0015" },
      { "ts_ms": 1700000008000, "premium": "-0.0007", "funding_rate": "-0.0007" }
    ]
  }
}
```

العينات هي الحلقة المرتبة من لقطات العلاوة الصادرة من متتبع التمويل.
`premium` هو قيمة `Decimal` الدقيقة قبل التقليص، مُعرضةً كسلسلة نصية (بإشارة، بدقة كاملة)؛
أما `funding_rate` فهو العلاوة ذاتها بعد تطبيق حد التمويل لكل أصل
(`±funding_rate_cap`، أو تجاوز المخاطر الديناميكية، أو الأساس الافتراضي `0.04`/ساعة)
— أي المعدل المُحقَّق الذي سيُطبَّق فعليًا. عندما تكون العلاوة ضمن الحد يكون
`funding_rate == premium`؛ وعند تجاوزها يُقلَّص `funding_rate` إلى الحد الأقصى بإشارته.
السوق غير المعروف أو الفارغ يُعيد `"samples": []`.

| Field | Type | Description |
|-------|------|-------------|
| `market_id` | uint32 | معرف السوق المُعاد صدى |
| `samples[*].ts_ms` | uint64 | طابع زمني للعينة (مللي ثانية الإجماع) |
| `samples[*].premium` | decimal string | عينة علاوة التمويل الخام، قبل التقليص (بإشارة) |
| `samples[*].funding_rate` | decimal string | المعدل المُحقَّق = `premium` مُقلَّصًا إلى حد الأصل (بإشارة) |

### `predicted_fundings`

معدل التمويل المتوقع + وقت الدفعة التالية لكل سوق، عبر جميع أسواق العقود الدائمة المسجَّلة.
لا توجد معاملات.

```json
{ "type": "predicted_fundings" }
```

حمولة `data` هي **مصفوفة** مرتبة بصورة حتمية تصاعديًا حسب `asset`
(يتكرر العقدة على `BTreeMap` مواصفات السوق). تُعيد الكون الفارغ `"data": []`.

الاستجابة:

```json
{
  "type": "predicted_fundings",
  "data": [
    { "asset": 0, "predicted_rate": "0.0015", "next_funding_time": 1700003600000 }
  ]
}
```

`predicted_rate` هو أحدث عينة علاوة (وكيل المعدل بالساعة، سلسلة عشرية)
— `"0"` قبل العينة الأولى. `next_funding_time` هو الطابع الزمني المشتق للدفعة التالية
(`last_sample_ts + 1h`)، و`0` قبل العينة الأولى.

| Field | Type | Description |
|-------|------|-------------|
| `asset` | uint32 | معرف الأصل / السوق |
| `predicted_rate` | decimal string | أحدث عينة علاوة (وكيل المعدل بالساعة)؛ `"0"` قبل العينة |
| `next_funding_time` | uint64 | طابع زمني لدفعة التمويل التالية (مللي ثانية الإجماع)؛ `0` قبل العينة |

### `mip3_active_bids`

لقطة مزاد غاز نشر العقود الدائمة غير المصرَّح به وفق MIP-3. لا توجد معاملات.

```json
{ "type": "mip3_active_bids" }
```

الاستجابة:

```json
{
  "type": "mip3_active_bids",
  "data": {
    "auction_round":   2,
    "current_bid":     "12345",
    "current_winner":  "0x<bidder>",
    "auction_end_ms":  1700086400000,
    "started_at_ms":   1700000000000,
    "bids": [
      {
        "bidder":          "0x<bidder>",
        "amount":          "12345",
        "submitted_at_ms": 1700000000500,
        "tag":             "ETH-PERP"
      }
    ]
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `auction_round` | uint64 | جولة المزاد الحالية |
| `current_bid` | decimal string | مبلغ العرض الرائد |
| `current_winner` | hex address \| null | المتقدم الفائز حاليًا، `null` إذا لم يكن هناك أحد |
| `auction_end_ms` | uint64 | طابع زمني لإغلاق المزاد (مللي ثانية الإجماع) |
| `started_at_ms` | uint64 | طابع زمني لبدء المزاد (مللي ثانية الإجماع) |
| `bids[*].bidder` | hex address | عنوان المتقدم |
| `bids[*].amount` | decimal string | مبلغ العرض |
| `bids[*].submitted_at_ms` | uint64 | طابع زمني لتقديم العرض (مللي ثانية الإجماع) |
| `bids[*].tag` | string | وسم العرض (مثلًا اسم السوق المقترح) |

### `liquidatable`

الحسابات المُصنَّفة حاليًا للتصفية. لا توجد معاملات.

```json
{ "type": "liquidatable" }
```

الاستجابة:

```json
{
  "type": "liquidatable",
  "data": { "accounts": [ { "address": "0x<addr>", "tier": "PartialMarket50" } ] }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `accounts[*].address` | hex address | الحساب الذي يستدعي إجراءً |
| `accounts[*].tier` | `"YellowCard" \| "PartialMarket50" \| "FullMarket" \| "BackstopTakeover"` | مستوى BOLE |

مصدر الحالة: `Exchange.bole_index.tier` (فهرس BOLE للإجراءات المطلوبة — **وليس** إعادة مسح كاملة للحساب).

> **ملاحظة.** `bole_index` يحمل `#[serde(skip)]` وهو حالة مشتقة غير قانونية، يُعاد بناؤها بمسح كامل عند الاستخدام الأول / بعد تحميل اللقطة. في لقطة منشورة حديثًا تكون فارغة حتى تُنفِّذ وحدة التشغيل تمريرة BOLE مرة واحدة على الأقل.

### `active_asset_data`

الرافعة المالية / وضع الهامش / الحجم الأقصى للتداول لكل أصل للمستخدم. المطلوب: `address` (سداسي عشري بـ 0x) + `asset_id` (u32).

```json
{ "type": "active_asset_data", "address": "0x<addr>", "asset_id": 0 }
```

الاستجابة:

```json
{
  "type": "active_asset_data",
  "data": {
    "address": "0x<addr>", "asset_id": 0, "leverage": 7,
    "margin_mode": "isolated", "max_trade_size": "5000000000", "has_position": true
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `leverage` | uint32 | رافعة الوضعية إن كانت مفتوحة، وإلا الافتراضي للحساب، وإلا الحد الأقصى للسوق |
| `margin_mode` | `"cross" \| "isolated" \| "strict_iso"` | وضع الهامش الفعّال |
| `max_trade_size` | decimal string | سقف الأمر الأقصى لكل أصل (انظر `max_market_order_ntls`) |
| `has_position` | bool | ما إذا كان للمستخدم وضعية غير صفرية على هذا الأصل |

مصدر الحالة: `locus.clearinghouses[asset].positions[addr]`، `locus.user_account_configs[addr]`، مواصفات السوق / المخاطر الديناميكية.

### `max_market_order_ntls`

القيمة الافتراضية القصوى لأوامر السوق لكل أصل. لا توجد معاملات.

```json
{ "type": "max_market_order_ntls" }
```

الاستجابة:

```json
{
  "type": "max_market_order_ntls",
  "data": { "ntls": [ { "asset_id": 0, "max_market_order_ntl": "5000000000" } ] }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `ntls[*].asset_id` | uint32 | معرف الأصل |
| `ntls[*].max_market_order_ntl` | decimal string | سقف الحجم المشتق من حد الفائدة المفتوحة |

مصدر الحالة: `PerpAnnotation.oi_cap` لكل سوق، وإلا `default_mip3_limits.max_oi_per_market`.

> **ملاحظة.** لا يوجد حقل مخصص لـ"الحد الأقصى لقيمة أمر السوق" لكل أصل في الحالة المُثبَّتة؛ حد الفائدة المفتوحة هو أقرب سقف مخاطر مُثبَّت، ويُبلَّغ عنه بوحدات **الحجم** (تحوّله طبقة المطابقة إلى قيمة افتراضية بسعر العلامة الحي).

### `perps_at_open_interest_cap`

الأصول التي وصلت فائدتها المفتوحة إلى الحد الأقصى أو تجاوزته. لا توجد معاملات.

```json
{ "type": "perps_at_open_interest_cap" }
```

الاستجابة:

```json
{ "type": "perps_at_open_interest_cap", "data": { "assets": [0] } }
```

| Field | Type | Description |
|-------|------|-------------|
| `assets` | uint32[] | معرفات الأصول عند حد `oi_cap` أو فوقه، تصاعديًا |

مصدر الحالة: `open_interest` لكل دفتر مقارنةً بـ `PerpAnnotation.oi_cap` (تُتجاهل الدفاتر التي لا تمتلك حدًا موجبًا).

### `margin_table`

جدول مستويات الهامش (الرافعة → نسب الصيانة / البدء). لا توجد معاملات.

```json
{ "type": "margin_table" }
```

الاستجابة:

```json
{
  "type": "margin_table",
  "data": { "tiers": [ { "asset_id": 0, "max_leverage": 50, "maint_margin_ratio": "300", "init_margin_ratio": "200" } ] }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `tiers[*].asset_id` | uint32 | معرف الأصل |
| `tiers[*].max_leverage` | uint8 | الحد الأقصى الفعّال للرافعة (تجاوز أو ثابت) |
| `tiers[*].maint_margin_ratio` | bps string | نسبة هامش الصيانة (تجاوز أو الحد الأدنى الثابت 3%) |
| `tiers[*].init_margin_ratio` | bps string | `1 / max_leverage` |

مصدر الحالة: `dynamic_risk_overrides[asset]` وإلا الخط الأساسي الثابت.

> **ملاحظة.** تخزّن الحالة المُثبَّتة مستوى مخاطرة فعّالًا واحدًا لكل سوق (تجاوز أو ثابت)، وليس سلّم الرافعات متعدد الصفوف الذي يقدّمه HL. الوكيل هو مستوى واحد لكل سوق — الصف الذي تُطبّقه المحرّكة اليوم.
