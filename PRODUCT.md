# Product

## Register

product

## Users

External developers and bot operators integrating with the MetaFlux derivatives exchange — quants, market makers, and engineers wiring up trading clients. Their context: in a task, looking up a specific wire shape, error code, signing step, or concept while writing code. They arrive from search or a deep link, scan for the exact field/endpoint, copy a payload, and leave. Many are migrating an existing HL-compatible bot.

## Product Purpose

The public integration reference for MetaFlux: REST/WS API surface, signing, SDKs, and the core trading/risk concepts (margin, liquidation, funding, vaults). Success = a developer finds the precise, copy-paste-safe answer fast and trusts it. It is documentation that serves the task, not a marketing surface.

## Brand Personality

Technical, precise, confident, quietly welcoming. Three words: **exact, grounded, unflashy**. It should read like infrastructure docs written by people who run the system — not a hype site. Warmth comes from clarity and good typography, not decoration.

## Anti-references

- Generic crypto dashboards: neon gradients, glassmorphism, glowing cards, "cyber" fonts.
- SaaS-cream landing slop: warm off-white bg, tracked uppercase eyebrows, hero-metric templates, identical icon-card grids.
- Over-animated docs that make you wait for choreography.

## Design Principles

1. **Readability is the product.** Type, measure, contrast, and scannability beat decoration. If a flourish costs legibility, cut it.
2. **Copy-paste confidence.** Code, tables, and field references must be unambiguous and easy to lift — the densest content gets the most care.
3. **Quiet brand, loud clarity.** Brand blue/pink appear as precise accents (links, state, selection), never as wallpaper.
4. **Earned familiarity.** Standard docs affordances (sidebar, TOC, breadcrumbs, admonitions) behave exactly as a fluent reader expects; surprise is a bug.
5. **Dark-first, light-equal.** Both themes are first-class and both clear WCAG AA.

## Accessibility & Inclusion

WCAG AA across both themes (body ≥4.5:1, large ≥3:1, verified). Visible keyboard focus rings. `prefers-reduced-motion` honored on every animation. Color is never the sole signal (admonitions pair color with icon + label).
